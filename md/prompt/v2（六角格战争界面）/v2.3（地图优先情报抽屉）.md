# v2.3 地图优先情报抽屉

## 版本号与分配依据

- 版本号：`v2.3`。
- 分配依据：v2.2 已完成顶栏、支援区和地图格降噪，并由 `origin/main` run `29589038142` 的 iPad 截图验收。新截图显示常驻 Inspector 仍占据约三分之一首屏宽度，因此沿用 v2 小版本继续收敛六角格战争界面。

## 背景

《Glory of Generals 3》参考图以地图、单位和边缘命令为第一视觉层级，详细规则信息按需出现。v2.2 截图中单位与地貌已更清楚，但右侧主操作、战线态势和规则说明仍像常驻调试面板，削弱大地图战场感。

## 目标

- 让地图默认占满顶栏以下的可用工作区。
- 将 Inspector 改为默认收起、从屏幕边缘按需展开的战场情报抽屉。
- 保留单位详情、地形详情、战线态势、结果、敌情、反制和战报的全部入口。
- 保持当前战役/战术/细节缩放和地图滚动语义不变。

## 非目标

- 不改变 `GameState`、规则模型、AI、命令执行或输入语义。
- 不删除 Inspector 内任何功能，不重排核心规则优先级。
- 不新增图片资产、第三方库、战斗动画或战役数据。
- 不修改 Xcode project 或用户未暂存文件。

## 当前架构依据

- `ContentView.swift` 负责根响应式编排和 Inspector；抽屉展开仅是本地 SwiftUI 状态。
- `BattlefieldChrome.swift` 和 `BattlefieldMap.swift` 继续负责地图尺寸、滚动与当前动作 HUD。
- 所有按钮继续调用既有 `GameState` 方法，禁止在 View 中新增规则。

## 实现步骤

1. 在 `ContentView` 持有 Inspector 展开状态，地图工作区始终使用完整可用宽高。
2. 新增表现层 `BattlefieldWorkspace`：regular 宽度使用右缘覆盖抽屉，compact 宽度使用底部覆盖抽屉。
3. 提供稳定 44x44 的边缘按钮、VoiceOver 标签、hover help 和展开状态值。
4. Reduce Motion 开启时取消抽屉位移动画。
5. 提高抽屉背景不透明度，确保覆盖地图后文字仍可读。

## 验收标准

- 首屏 Inspector 默认收起，地图不再为常驻侧栏预留固定宽度。
- 右缘按钮可展开/收起全部 Inspector 内容，点击区至少 44x44。
- regular 与 compact 布局均不挤压地图；抽屉展开后内容可滚动。
- Reduce Motion、VoiceOver 标签和原有地图/Inspector 命令入口保留。
- 最新 `origin/main` run 成功，截图为 WW2Tactics 战场而非黑屏或 SpringBoard。
- 新截图中地图宽度、六角地貌和单位占比明显高于 v2.2，默认不再显示密集右侧面板。

## 风险和禁止项

- 覆盖抽屉可能遮住地图内容，必须提供始终可见的收起按钮。
- 不允许用删除详情功能换取地图面积。
- 不允许让抽屉状态进入 `GameState`。
- 不允许提交 `AGENT.md` 删除或 `project.pbxproj` 的用户改动。
- 不允许用本地输出或旧 artifact 冒充本轮验收。
