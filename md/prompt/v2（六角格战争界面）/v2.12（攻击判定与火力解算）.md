# v2.12 攻击判定与火力解算

## 版本号与分配依据

- 版本号：`v2.12`
- 分配依据：v2.11 已由 `origin/main` commit `0c3d1f8e9cf96de5b3dc0af0b3c33414888c17ae`、GitHub Actions run `30071383570`、attempt `1` 和 artifact `ww2tactics-ci-v2.11-main-0c3d1f8-run30071383570-attempt1` 完成云端验收。选中/POS 状态和战术标记密度已可控，本轮继续递增 v2 小版本，补齐真实 ATK 状态与战争化火力判定界面。

## 背景

官方 EasyTech 战场截图在选中敌军时会让攻击目标、伤害关系和行动入口成为明确焦点。WW2Tactics 当前 POS 截图已能显示接敌路线，但 `InlineMapCommandPreview` 仍主要是一行说明文字，攻击时缺少双方耐久、预计战果和反击关系的结构化视觉，也没有实际云端 ATK 截图证据。

## 目标

1. CI 截图使用真实输入链让 M10 从 q7,r6 移入攻击位并聚焦 q9,r6 敌方装甲侦察营，停在攻击执行前。
2. ATK 状态下地图动作 HUD 显示独立火力解算带：攻击方、目标、双方当前/预计耐久、主伤害、反击和击毁/存活结论。
3. 执行入口使用明确的 scope/开火按钮，保持至少 44pt 点击高度并继续调用 `executeFocusedCommand()`。
4. 非 ATK 的 MOVE/POS/查看状态继续使用紧凑命令预览，不改变既有行为。
5. 正常 app 启动不执行 CI 移动；旧 POS 参数仍可保留为独立诊断入口。

## 非目标

- 不修改伤害公式、克制、地形加成、反击、经验、士气或攻击执行链。
- 不在 View 中重新计算战斗数值，不直接写 `selectedUnitID`、位置或 focusedCoordinate。
- 不自动执行第二次攻击，不生成伪造战果或动画。
- 不修改地图坐标、命中、单位模型、地貌或 Xcode project。
- 不引入第三方库、位图资产或网络依赖。
- 不修改或提交用户现有的 `AGENT.md` 删除和 `project.pbxproj` 修改。

## 当前架构依据

- `handleSecondaryAction` 对 `.approachAttack` 会复用现有移动规则进入攻击位，随后保持选中单位并把焦点放回敌军，不自动攻击。
- `focusedCommandPreview` 随后成为 `.attack`，包含 attacker/defender、damage、counterDamage、defenderHPAfterAttack 与 willDestroy。
- `combatPreviewAgainstFocusedTarget()` 提供 attackerHPAfterCounter、克制和地形等补充只读数据。
- `MapActionHUD` 已承载单位摘要、指标、快捷命令和 `InlineMapCommandPreview`，本轮只重构该表现层。

## 实现步骤

1. `WW2TacticsApp` 增加 `--ci-selected-attack-preview`：先 `handleTap(q7,r6)`，再 `handleSecondaryAction(q9,r6)`；保留旧 approach 参数作为 fallback。
2. workflow 把截图场景改为 `selected-attack-preview` 并传入新参数；500 KB 黑屏门禁继续生效。
3. `InlineMapCommandPreview` 对 `.attack` 提取专用 `InlineAttackFireSolution`，其余 preview 保留紧凑布局。
4. 火力解算带只消费 `.attack` 关联值、`selectedUnit`、`focusedUnit` 和 `combatPreviewAgainstFocusedTarget()`；禁止复制公式。
5. 使用稳定的双栏耐久行和三项指标：HIT、RET、OUT。内容变化不能改变按钮尺寸或遮挡地图。
6. 使用图标、标签和数值共同区分攻击/反击/击毁，不能只依赖颜色；执行按钮保留 VoiceOver 标签。
7. 不增加持续动画；所有显示保持只读，只有开火按钮调用既有命令。
8. 同步 README、flow、flowchart、test、prompt README 和 update log。

## 验收标准

1. 云端截图明确处于 ATK 而非 POS：HUD 的 ATK 可用、敌方目标使用红色准星，消息说明已进入攻击位。
2. 火力解算带可辨认攻击方 M10、目标装甲侦察营、预计主伤害、反击、双方预计耐久和战果结论。
3. 开火按钮清晰且不自动触发；截图仍是攻击前状态。
4. 地图、单位、HP、短战术标记、Inspector、编队条和支援甲板入口不被遮挡。
5. 正常启动和旧 POS 参数边界明确，代码审查确认不直接写规则字段。
6. 最新 SHA/run/attempt 与 artifact manifest 完全一致，所有云端门禁成功，PNG 大于 500 KB 且实际画面非黑屏。

## 风险和禁止项

- 火力解算带不能扩张为遮挡大片战场的侧栏或模态窗口。
- 不得使用超小字体塞入全部规则解释；只显示决策所需高信号值。
- 不得把攻击预测写成已发生战果。
- 不得在 CI 启动场景中直接改单位位置或 HP。
- 不得触碰规则层、用户改动、分支或 PR 流程。
