# v2.9 地貌纵深与战区质感

## 版本号与分配依据

- 版本号：`v2.9`
- 分配依据：v2.8 已由 `origin/main` commit `99de0e88cdda06c6d613542ab11e2171d7240779` 对应的 GitHub Actions run `30066551656`、attempt `1` 和 artifact `ww2tactics-ci-v2.8-main-99de0e8-run30066551656-attempt1` 完成云端验收。地图单位已从徽章化棋子升级为透明落地军械模型，本轮继续在 v2 阶段递增小版本，处理云端截图中地貌仍像平铺色块的问题。

## 背景

Agent X 已核对 Google Play 官方《将军的荣耀3》雪地、沙漠和海战截图。官方战场的核心不是高饱和六角色块，而是低对比地表纹理、山体明暗面、树林前后层、城市体块投影和雪地冷色凹陷共同形成的连续地理层次。WW2Tactics v2.8 的单位模型明显改善，但平原、森林、城市、山地和雪地仍主要依赖单一渐变与细线图标，地图远看偏平。

当前会话已按 `imagegen` 技能检查位图生成路径，但没有可调用的内置图像生成工具；技能规则禁止静默切换到需要 API key 的 CLI。因此本轮使用可审查、无外部依赖的 SwiftUI 确定性地貌绘制，不引入来源不明资产，也不修改用户正在编辑的 Xcode project。

## 目标

1. 平原增加坐标稳定的田块明暗面、田垄和地表阴影，减少单色六角感。
2. 森林增加树冠底影、前后大小变化和亮暗分层，形成树林体积。
3. 城市增加道路底槽、建筑投影、屋顶面和边缘高光，形成可辨认的聚落体块。
4. 山地从单一折线升级为有亮面、暗面和脊线的多面山体。
5. 雪地增加冷色凹陷、积雪高光和漂移边缘，保持低对比而非纯白。
6. 河流增加深浅水层与高光，道路增加暗边和虚线中心，但仍严格沿既有连接方向连续。
7. 所有变体只由 `TerrainTile.coordinate` 派生，不引入随机状态或持久化。

## 非目标

- 不修改 `TerrainType` 规则语义、移动消耗、地形加成或关卡数据。
- 不改变 tile 的 86x74 frame、`position(for:)`、六角命中路径或 `HexInputReader`。
- 不改变河流/道路邻接计算、marker 数据源、排序、槽位或边框优先级。
- 不修改 `GameState`、单位模型、HUD、Inspector、支援甲板或战斗结算。
- 不引入位图、第三方库、网络依赖、随机数或持续动画。
- 不修改或提交用户现有的 `AGENT.md` 删除和 `project.pbxproj` 修改。

## 当前架构依据

- `HexTileView` 先绘制 `tile.terrain.mapGradient`，再叠加 `TerrainTexture`，之后才是路线、marker、单位和准星。
- `TerrainTexture` 已接收完整 `TerrainTile` 与同类邻接方向，并以 q/r 坐标产生稳定 seed。
- `TerrainType.showsMapConnections`、`connectionColor` 和 `connectionWidth` 继续决定河流、道路、林带等连续层。
- 本轮只扩展表现层绘制，不新增模型或规则状态。

## 实现步骤

1. 在 `TerrainTexture` 的地形分支中强化现有绘制函数，不改变调用顺序和裁剪方式。
2. 平原使用 3 个稳定位置与角度的半透明田块，再叠加田垄线；田块颜色在暖高光和冷暗面之间交替。
3. 森林每个树簇先绘制椭圆落地阴影，再绘制大小不同的深浅树冠；树冠位置与大小继续由 seed 派生。
4. 城市道路先画深色底槽，再画浅色路面；建筑先画偏移阴影，再画屋顶与高光边缘。
5. 山地构建两个闭合山体面：暗面与亮面分别填充，并在上层保留脊线；位置与峰高使用既有 seed。
6. 雪地先绘制冷蓝凹陷面，再绘制两条不同透明度的雪脊高光。
7. 河流/道路用双层或三层 stroke 增加深浅和边缘，但 endpoint 与 connectionDirections 不变。
8. 保持所有装饰 `.allowsHitTesting(false)`、`.accessibilityHidden(true)`，不新增动画。
9. 同步 README、flow、flowchart、test、prompt README 和 update log，并把 v2.8 等待记录替换为实际成功记录。

## 验收标准

1. 云端截图中平原能看到柔和田块深浅，森林具有明显前后层，城市具有屋顶与投影，山地不再只是折线，雪地不再只是白色六角。
2. 河流和道路仍跨相邻六角连续，没有断裂、突变或异常加粗。
3. 单位模型、Commander、HP、目标旗、NEXT 指挥坞、Inspector、目标跳转条和支援甲板入口仍可见。
4. tile frame、坐标、命中、marker、边框优先级和 `GameState` 未改变。
5. 最新 `origin/main`、Actions run 和 artifact manifest 的 SHA/run/attempt 完全一致，所有云端门禁成功。

## 风险和禁止项

- 地貌阴影不能把单位、HP、marker 或地形边界压暗到不可辨认。
- 不得让每个六角形成厚重独立卡片；纵深必须服务连续战区而非强化棋盘拼块感。
- 不能使用运行时随机数、Timer 或动画制造“自然感”。
- 不得触碰规则层、Xcode project、用户改动、分支或 PR 流程。
- 不得把 v2.8 的旧截图或 artifact 冒充 v2.9 验收结果。
