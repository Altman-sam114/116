# v2.15 攻击预判与结算一致性

## 版本号与分配依据

- 版本号：`v2.15`
- 分配依据：v2.14 已由 `origin/main` commit `51dad562c78dcffa26f9246fd721b7ec38acf71e`、GitHub Actions run `30076575519`、attempt `1` 和 artifact `ww2tactics-ci-v2.14-main-51dad56-run30076575519-attempt1` 完成云端验收。攻击后地图反馈成立，但 v2.12 攻击前截图预测 `RET -11`，v2.14 真实结算显示 `RET -7`，因此先修复玩家决策信息与真实规则不一致。

## 背景

`combatPreview` 当前在防守方受击前调用 `counterDamageValue(defender:attacker:)`，使用防守方原始 HP、士气和攻击方原始防御姿态。真实 `attack` 则先扣除防守方 HP、清除双方防御姿态、给防守方士气 -10，再调用 `counterAttackIfPossible`。反击伤害的 `damageValue` 包含攻击者 HP 比例、士气倍率和目标防御姿态，所以预判与结算必然可能分叉。EasyTech 风格战棋的火力解算必须能被玩家信任。

## 目标

1. `combatPreview` 按真实攻击顺序预测“攻击后、反击前”的双方状态。
2. 防守方反击预测使用扣除主伤害后的 HP、遭受攻击后的士气和已清除的防御姿态。
3. 攻击方作为反击目标时按真实流程清除防御姿态。
4. 同一未击毁普通攻击中，`CombatPreview.counterDamage` 与 `CombatResultSummary.counterDamage` 完全一致。
5. v2.15 云端场景继续显示真实攻击后 `RET -7`，测试证明攻击前预测也是 7。

## 非目标

- 不修改主伤害、反击伤害公式、经验、晋升、士气、夹击、地形、补给、指挥官或 AI 规则。
- 不改变 `attack` 的真实状态变更顺序。
- 不在 SwiftUI 或字符串层修正数值，不保存第二套 UI 预测状态。
- 不新增动画、视觉装饰或第三方依赖。
- 不修改 Xcode project，不触碰用户现有 `AGENT.md` 删除与 `project.pbxproj` 修改。

## 当前架构依据

- `combatPreview` 与真实攻击都必须复用 `counterDamageValue`，避免第二套伤害公式。
- `BattleUnit` 是值类型，可在预判中复制并应用确定性的攻击后状态，不写回 `scenario.units`。
- 真实未击毁链为：主伤害 -> 清除姿态 -> 攻击方经验/士气 -> 防守方士气 -10 -> 反击。
- 反击伤害依赖反击方 HP/最大 HP、士气、补给、兵种、地形、协同和目标防御姿态；预判至少必须模拟在反击前已经发生且会影响公式的字段。

## 实现步骤

1. 在 `GameState` 增加私有纯 helper，从原始 attacker/defender 与 `defenderHPAfterAttack` 生成反击预判输入。
2. 防守方副本设置结算后 HP、士气减 10（限制 0...100）、`isEntrenched = false`；攻击方副本设置 `isEntrenched = false`。
3. `combatPreview` 仅在目标存活且射程满足时，用上述副本调用既有 `counterDamageValue`；其他返回逻辑不变。
4. 不复制 `damageValue`、`counterDamageValue` 或士气阈值公式，不调用会写日志/状态的 `adjustMorale`。
5. XCTest 新增或强化断言：先取得 preview，再执行同一攻击，确认主伤害、反击伤害、双方 ending HP 与 preview 完全一致；至少覆盖 v2.14 的 M10 对装甲侦察营场景，以及防御姿态/士气阈值会影响反击的场景。
6. RulesSmokeTest 增加 preview/result 反击一致性断言。
7. 同步 README、flow、flowchart、test、prompt README 和 update log，记录 v2.14 精确验收证据及 v2.15 等待状态。

## 验收标准

1. M10 对装甲侦察营的攻击前 preview 反击值为 7，执行后 result 反击值也为 7。
2. `preview.defenderHPAfterAttack == result.defender.endingHP`，`preview.attackerHPAfterCounter == result.attacker.endingHP`。
3. 目标被击毁或无法反击时仍预测 0，不引入虚假反击。
4. 防守方受击后跨入低士气阈值时，preview 使用低士气反击倍率；攻击方原有防御姿态不会错误降低反击预判。
5. 预判不改变 scenario、日志、选择、焦点、行动标记或任何 latest result。
6. 云端 static、smoke、build 和截图全部成功；最新 artifact manifest 与 `origin/main` SHA/run/attempt 完全一致。
