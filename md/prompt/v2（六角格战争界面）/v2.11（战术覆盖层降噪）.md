# v2.11 战术覆盖层降噪

## 版本号与分配依据

- 版本号：`v2.11`
- 分配依据：v2.10 已由 `origin/main` commit `e01348e1b227dad26593526c49502e00f0686151` 对应 GitHub Actions run `30068638485`、attempt `1` 和 artifact `ww2tactics-ci-v2.10-main-e01348e-run30068638485-attempt1` 完成云端验收。选中与 POS 状态真实可见，但截图中 29 个 MOVE、9 个 THR 和多枚 POS 文字胶囊同时覆盖战区，因此继续递增 v2 小版本，专门收敛战术覆盖层信息密度。

## 背景

v2.10 的 2064x2752 iPad 云端截图确认蓝色选中环、四角框、橙色 POS 准星与动作 HUD 正常工作，同时暴露出地图在选中单位后被大量 `M5`、`THR`、`POS3+` 胶囊占据。EasyTech 参考画面会保留可达范围和关键目标，但不会让每个普通候选格都承担同等文字权重。本轮以焦点优先的方式降噪，让军械、地貌和关键目标重新成为第一视觉层。

## 目标

1. 当前聚焦的 MOVE/ATK/POS 格继续显示完整胶囊与数值。
2. 非焦点 MOVE 格改为小型成本圆点，不再重复 `M` 前缀和宽胶囊。
3. 非焦点 ATK/POS 格改为紧凑 target/scope 图标，不重复伤害或 `POS` 文本。
4. THR 从文字胶囊改为小型警告三角，风险仍由红色边界、控制区和 HUD 汇总表达。
5. 保留地图格 VoiceOver 摘要、边框、可达/攻击/接敌数量和全部输入语义。

## 非目标

- 不修改 `GameState.mapActionHint`、移动代价、威胁判定、攻击预览、路线或单位状态。
- 不减少可移动、可攻击、可接敌或受威胁格的实际数量。
- 不修改地貌、单位模型、HUD 布局、CI 启动场景或战役数据。
- 不新增动画、资产、依赖或规则状态。
- 不修改或提交用户现有的 `AGENT.md` 删除和 `project.pbxproj` 修改。

## 当前架构依据

- `HexTileView` 已同时持有 `isFocused` 与 `actionHint`，可把焦点布尔值只读传给 `ActionMarker`。
- `ActionMarker` 当前自行从 `MapActionHint` 派生图标、标签和颜色，本轮只调整它的表现分支。
- `ThreatenedMoveMarker` 只读表示 `threatenedReachableTiles` 成员，本轮只改变图形，不改变集合。
- `HexTileView.accessibilityLabel` 继续从完整 tile/unit/action 数据生成，无需依赖可见胶囊文字。

## 实现步骤

1. 给 `ActionMarker` 增加 `isFocused` 参数，并在 `HexTileView` 调用处传入现有焦点状态。
2. 焦点态保留现有完整胶囊、icon、label、颜色与高对比描边。
3. 非焦点 `.move` 显示 17-18pt 青色圆点和单个成本数字；控制区额外消耗用小 `+` 保留。
4. 非焦点 `.attack` 与 `.approachAttack` 显示 17-18pt 红/橙圆点和 target/scope 图标，不显示长文本。
5. `ThreatenedMoveMarker` 改为约 14pt 红色警告三角，不绘制 `THR` 文本、胶囊或额外白边。
6. 所有标记继续 `.allowsHitTesting(false)`、`.accessibilityHidden(true)`，tile 的 VoiceOver 主摘要不变。
7. 同步 README、flow、flowchart、test、prompt README 和 update log。

## 验收标准

1. v2.11 截图中 MOVE、THR、POS 的重复文字数量显著少于 v2.10，地图军械和地貌更容易辨认。
2. 当前敌方 POS 焦点仍显示橙色四角准星和完整 `POS 接敌位置` HUD；当前格的关键数值可读取。
3. 普通 MOVE 成本仍可从紧凑圆点读取，THR/ATK/POS 类型仍能按颜色和图标区分。
4. selected HUD 的 MOVE/ATK/THR/SUP 数量与 v2.10 一致，不修改规则集合或交互。
5. 最新 SHA/run/attempt 与 artifact manifest 完全一致，云端门禁全部成功。

## 风险和禁止项

- 紧凑标记不能小到无法在 iPad 截图中辨认，也不能遮挡 HP 条或单位主体。
- 焦点目标必须保留完整语义，不能为了降噪把关键执行信息全部隐藏。
- 不得删除 VoiceOver 信息或命中区域。
- 不得把 v2.10 artifact 冒充 v2.11 验收结果。
- 不得触碰规则层、用户改动、分支或 PR 流程。
