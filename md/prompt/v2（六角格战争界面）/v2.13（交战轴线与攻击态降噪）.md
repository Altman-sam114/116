# v2.13 交战轴线与攻击态降噪

## 版本号与分配依据

- 版本号：`v2.13`
- 分配依据：v2.12 已由 `origin/main` commit `aa394f17fa9b180b939dfd115f5ed72aec9ade51`、GitHub Actions run `30072585832`、attempt `1` 和 artifact `ww2tactics-ci-v2.12-main-aa394f1-run30072585832-attempt1` 完成云端验收。真实 ATK 火力解算已成立，但截图显示全图控制区、射程覆盖与非目标命令标记仍和当前交战争夺视觉焦点，因此继续递增 v2 小版本做攻击态专项降噪。

## 背景

v2.12 云端截图清楚显示 M10 与装甲侦察营的攻击前数值、红色准星和开火入口，但攻击目标周围仍铺满红色控制区圆点/边线、橙色覆盖点与其他命令标记。EasyTech 风格的攻击判定需要把玩家注意力锁在攻击方、目标和两者关系上，而不是让整个威胁图层继续保持同等强度。

## 目标

1. `.attack` 预览存在时进入纯视觉攻击聚焦模式。
2. 保留选中攻击方、当前焦点目标、目标 ATK 标记、双方模型/HP、据点与地形。
3. 暂时隐藏非焦点控制区图标/边线、射程覆盖图标/边线、非焦点 command marker 和次要态势 chips。
4. 在攻击方与目标间绘制不拦截输入的交战轴线，表达火力方向。
5. 离开 `.attack` 预览后，所有覆盖层恢复既有显示。

## 非目标

- 不修改控制区、射程、攻击目标、伤害、反击、移动或 AI 规则。
- 不清理或改写任何 `GameState` 集合，不新增 `@Published` 状态。
- 不改变 tile frame、坐标、滚动、缩放、命中或手势。
- 不改 v2.12 火力解算数值与开火行为。
- 不修改 Xcode project，不引入第三方库或位图资产。
- 不修改或提交用户现有的 `AGENT.md` 删除和 `project.pbxproj` 修改。

## 当前架构依据

- `HexMapView` 已集中读取 `focusedCommandPreview`、selected/focused coordinate 与各覆盖层集合，适合派生单个只读 `isAttackFocusMode`。
- `HexTileView` 已独立控制 `AttackCoverageMarker`、`ControlZoneMarker`、`ActionMarker` 和对应 border 的显示强度。
- `position(for:)` 已把六角坐标转换为地图点位，可用于绘制攻击方到目标的只读轴线。
- `HexInputReader` 位于 tile overlay，新增视觉必须 `.allowsHitTesting(false)`，不能改变输入链。

## 实现步骤

1. `HexMapView` 从 `focusedCommandPreview` 是否为 `.attack` 派生攻击聚焦模式，并把它传给 `HexTileView`。
2. 聚焦模式下，只隐藏非焦点的 `AttackCoverageMarker`、`ControlZoneMarker`、command marker 与次要态势 chips；焦点目标的 ATK marker 必须保留。
3. `borderColor`/`borderWidth` 同步忽略聚焦模式下的非焦点 attack coverage 与 enemy control zone，避免只隐藏图标却仍保留全图强边线。
4. 在地图 tile 层之上绘制攻击方到目标的 `EngagementAxisOverlay`，使用暗色底线、橙红色短划线和目标方向符号；端点留白，避免穿过单位主体。
5. 轴线隐藏无障碍并禁止 hit testing；地图格 accessibility 仍保留规则信息。
6. 同步 README、flow、flowchart、test、prompt README 与 update log，记录 v2.12 验收证据和 v2.13 等待状态。

## 验收标准

1. 云端截图仍显示 v2.12 的 ATK 火力解算、红色目标准星、双方 HP 和开火按钮。
2. 攻击方与目标之间存在清晰但不遮挡模型的交战轴线。
3. 与 v2.12 对比，全图红色控制区圆点/边线、橙色射程覆盖点和非焦点命令标记显著减少。
4. 当前焦点目标的 ATK 伤害/反击标记仍可见；双方单位、HP、地形和据点仍清楚。
5. 代码审查确认只过滤视觉显示，底层集合、accessibility 文本、输入和规则完全保留。
6. 最新 SHA/run/attempt 与 artifact manifest 完全一致，所有云端门禁成功，PNG 至少 500 KB 且实际画面非黑屏。

## 风险和禁止项

- 不能隐藏当前焦点 ATK marker、准星、目标模型或 HP。
- 不能让交战轴线穿透整块 HUD、拦截输入或改变 scroll content size。
- 不能用全屏暗幕遮掉地图和其他单位；本轮只压低规则覆盖层。
- 不能通过改 `GameState` 集合伪造降噪。
- 不得触碰用户改动、分支或 PR 流程。
