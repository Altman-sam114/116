# v2.8 地图单位落地模型

## 版本号与分配依据

- 版本号：`v2.8`
- 分配依据：v2.7 已由 `origin/main` commit `512c819d06da3721c5a3638e7f74010f00f35622` 对应的 GitHub Actions run `30065985230`、attempt `1` 和 artifact `ww2tactics-ci-v2.7-main-512c819-run30065985230-attempt1` 完成云端验收。空闲 HUD 已收敛且地图保持满幅，本轮继续在 v2 阶段递增小版本，处理云端截图中地图单位仍像圆形徽章的问题。

## 背景

《将军的荣耀3》官方战斗截图中的坦克、火炮和步兵模型直接落在地表，阵营环与状态标记只作为辅助识别。WW2Tactics v2.7 云端截图显示，四类军械内部结构已经存在，但地图 `UnitCounter` 仍复用侧栏 `UnitShapeBadge` 的大面积蓝/红底座，远看更像圆形图标，削弱模型和地貌的关系。

## 目标

1. 为地图 `UnitCounter` 提供专用透明落地棋子，不再复用大面积填色的 `UnitShapeBadge`。
2. 放大军械模型，使坦克履带、炮塔、火炮轮组、步兵队形和侦察车轮廓在地图上更直接可见。
3. 用椭圆地面环、AL/AX 短码和实线/虚线同时表达阵营，保留非纯颜色识别。
4. 完整保留 Commander、rank、CUT、战术状态、spent 和五段 HP。
5. 侧栏、HUD、编队条和其他信息密集区域继续使用现有 `UnitShapeBadge`。

## 非目标

- 不改变 `BattleUnit`、`UnitKind`、`Faction` 或任何规则数据。
- 不改变单位坐标、tile frame、命中、选择、移动、攻击或 marker。
- 不替换真实图片资产，不引入第三方库或网络依赖。
- 不重构侧栏、HUD、编队条、将领系统或战斗结算。
- 不增加持续动画、粒子、音效或镜头效果。

## 当前架构依据

- `BattlefieldMap.swift` 只通过 `UnitCounter(unit:)` 放置地图单位，调用边界无需改变。
- `BattlefieldUnitViews.swift` 已包含 `UnitModelView`、四类 `UnitMarkerShape`、内部结构线、阵营军械色板、Commander 与 HP 组件。
- `UnitShapeBadge` 被多个非地图界面复用，本轮不能全局修改其视觉语义。

## 实现步骤

1. 在 `BattlefieldUnitViews.swift` 新增地图专用 `MapUnitPiece`，由 `UnitCounter` 使用。
2. `MapUnitPiece` 使用透明背景和椭圆落地环：盟军为蓝色实线，轴心为红色虚线，并加入 `AL`/`AX` 短码。
3. 在地面环上方直接绘制放大的 `UnitModelView`，保留军械阴影、渐变、边缘和内部结构线。
4. 把 rank、CUT 与战术状态保留在稳定角落，避免遮挡模型主体；Commander 仍由 `UnitCounter` 外层叠加。
5. 保持 `UnitCounter` 外接宽高适配现有 86x74 tile，HP 条仍在模型下方且使用原比例。
6. 不修改 `UnitShapeBadge` 和所有非地图调用点。
7. 同步 README、flow、flowchart、test、prompt README 和 update log，并记录 v2.7 实际云端结果。
