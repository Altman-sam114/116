# v2.4 连续地貌自然化

## 版本号与分配依据

- 版本号：`v2.4`。
- 分配依据：v2.3 已由 `origin/main` run `29596560289` 的真实 iPad 截图验收，地图占满主工作区。截图显示当前最大视觉问题转为每格重复的平原斜线、森林树簇、道路辐射和山雪纹理，因此继续 v2 小版本处理连续地貌。

## 背景

《Glory of Generals 3》参考图的六角规则层叠在连续地图上，地貌不会在每格重复同一图案。当前 `TerrainTexture` 对所有同类邻格绘制中心辐射线，并让每个同类地格复用相同纹理坐标，形成明显拼花感。

## 目标

- 降低普通地貌的六角重复纹理和中心辐射线。
- 保留河流与道路跨格连续性，同时让线宽和对比更接近地图地物。
- 使用地格坐标派生稳定纹理变化，避免运行时随机和重绘闪动。
- 保留所有 marker、单位、命中区域、滚动和无障碍语义。

## 非目标

- 不改变地形类型、移动消耗、防御、战役数据或 `GameState`。
- 不引入图片资产、噪声库、第三方依赖或随机状态。
- 不改变单位模型、Inspector 抽屉、HUD 或战斗动画。
- 不修改 Xcode project 和用户未暂存文件。

## 当前架构依据

- `BattlefieldMap.swift` 的 `TerrainTexture` 只读接收 `TerrainTile` 与同类邻接方向。
- 地形规则仍来自 `GameModels` / `GameState`，纹理只改变 SwiftUI 表现。
- `HexTileView` 的 frame、position、Hexagon 命中 path 和 `HexInputReader` 必须保持不变。

## 实现步骤

1. 仅为 river、road 与弱化后的 forest 绘制同类连接线；plains/city/mountain/snow 不再画中心辐射。
2. 降低 road/forest 连接线宽和对比，保留 river 主水道宽度。
3. 从 q/r 坐标计算确定性整数 seed。
4. 用 seed 改变平原线条起点/坡度、树簇位置/尺寸、城市块位置、山脊峰位和雪纹曲线。
5. 降低普通纹理 opacity 和密度，让单位与战术 marker 保持更高层级。
