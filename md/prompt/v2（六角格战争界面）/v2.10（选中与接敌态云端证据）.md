# v2.10 选中与接敌态云端证据

## 版本号与分配依据

- 版本号：`v2.10`
- 分配依据：v2.9 已由 `origin/main` commit `1911ae4104f61433b909804f89c2ade1e2d347b7` 对应的 GitHub Actions run `30067975922`、attempt `1` 和 artifact `ww2tactics-ci-v2.9-main-1911ae4-run30067975922-attempt1` 完成云端验收。v2.7 已实现选中光环和战斗准星，但既有 CI 只截取空闲首屏，本轮继续递增 v2 小版本，为选中与接敌态建立真实云端视觉证据。

## 背景

v2.9 修复版截图已确认地貌、单位、HUD 和边缘指挥坞能够共同显示，早期道路黑色三角和河流黑线问题已经消失。当前缺口不是继续增加视觉装饰，而是 CI 截图尚未覆盖玩家最常见的单位选择和接敌预览，无法从 artifact 判断蓝色选中反馈、POS 准星、动作 HUD 与地貌模型是否发生遮挡。

## 目标

1. 为 CI 截图启动增加专用参数，稳定进入“己方单位已选中、敌方接敌目标已聚焦”的只读预览场景。
2. 通过现有 `GameState` 选择和聚焦入口构造状态，不直接改写规则字段，不执行移动或攻击。
3. 云端截图同屏展示蓝色选中地面环、蓝色四角框、橙色 POS 目标准星、移动/攻击覆盖和已选单位动作 HUD。
4. 正常用户启动不带 CI 参数时继续保持 v2.9 空闲首屏，不改变战役、规则或交互。
5. artifact manifest 和截图日志记录本轮截图场景，避免把空闲首屏误当成选中态证据。

## 非目标

- 不修改移动、攻击、AI、地形、单位数值、坐标或胜负规则。
- 不重新设计 v2.7 已有选中光环、准星或动作 HUD。
- 不执行自动移动、攻击、待命、战术命令或回合推进。
- 不新增 UI 测试 target、第三方库、网络依赖、位图资产或持续动画。
- 不修改或提交用户现有的 `AGENT.md` 删除和 `project.pbxproj` 修改。

## 当前架构依据

- `WW2TacticsApp` 是唯一启动入口，可在创建 `GameState` 后根据进程参数选择是否构造 CI 视觉场景。
- `GameState.handleTap(on:)` 与 `focus(coordinate:)` 已分别承担地图选择和只读聚焦；CI 场景必须复用这两个入口。
- 阿登初始战役的 M10 坦克歼击营位于 `q7,r6`，敌方装甲侦察营位于 `q9,r6`；选择前者并聚焦后者会形成可进入攻击位的 POS 预览，但不会执行命令。
- `HexTileView` 已从 `isSelected`、`isFocused` 和 `actionHint` 派生光环、四角框与红橙准星，`MapActionHUD` 已从 `selectedUnit` 派生动作信息。

## 实现步骤

1. 在 `WW2TacticsApp` 初始化 `GameState` 时识别单一、明确的 `--ci-selected-approach-preview` 进程参数。
2. 参数存在时，先用 `handleTap(on: q7,r6)` 走现有地图选择链，再用 `focus(coordinate: q9,r6)` 走现有敌军预览链；不直接写 `selectedUnitID` 或 `focusedCoordinate`。
3. 参数不存在时不做任何附加动作，保持默认 `GameState()` 行为。
4. 修改 CI 的 `simctl launch`，只在云端截图步骤向 app 传入该参数，并在截图日志、manifest 和 failure summary 中记录场景名。
5. 同步 README、flow、flowchart、test、prompt README 和 update log。

## 验收标准

1. 云端截图中能同时辨认 M10 的蓝色选中地面环/四角框、装甲侦察营的橙色 POS 准星与接敌位置提示。
2. 已选单位动作 HUD 显示单位名、HP、MOVE/ATK/THR/补给和 NEXT/ATK/POS/OBJ 入口，没有遮挡主要战区。
3. 截图日志和 manifest 明确记录 `selected-approach-preview`，且截图不是空闲首屏、SpringBoard、黑屏或崩溃画面。
4. 不带参数的正常启动仍保持默认未选中状态；代码审查确认只在明确 CI 参数存在时应用场景。
5. 最新 `origin/main`、Actions run 和 artifact manifest 的 SHA/run/attempt 完全一致，所有云端门禁成功。

## 风险和禁止项

- CI 场景坐标只服务当前阿登固定基线；若战役初始编队改变，截图应失败或明显退化，而不是静默伪造目标态。
- 不得通过直接写 `@Published` 字段绕过真实输入/聚焦语义。
- 不得让 CI 参数进入普通用户启动路径或持久化到游戏状态。
- 不得把 v2.9 空闲截图或旧 artifact 冒充 v2.10 选中态验收结果。
- 不得触碰用户的 Xcode project 改动、创建分支或 PR。
