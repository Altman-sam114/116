# v2.6 战区满幅构图

## 版本号与分配依据

- 版本号：`v2.6`。
- 分配依据：v2.5 已由 `origin/main` run `29884658051` 的 2064x2752 iPad 截图验收。单位模型层次达到本轮目标，但截图显示 22x14 地图在固定 0.86 缩放下只占工作区中段，上下出现明显空带，因此继续 v2 小版本修正战区构图。

## 背景

EasyTech 风格战棋首屏应由连续六角战区主导，地图边缘可以滚动但不应在可用工作区内显示大块无内容底板。当前 `MapCommandCenter` 的双轴 `ScrollView` 高于缩放后的地图，SwiftUI 将较小内容居中，导致战区上下各留出空白，削弱地图规模感和单位可读性。

## 目标

- 默认战役档让六角地图至少铺满当前地图 viewport 高度。
- 战术/细节档在满幅基线上继续放大，保留清楚的三级缩放差异。
- 初始进入、焦点变化和缩放切档后都让当前焦点保持在可视区中心。
- 保留双轴滚动、地图 HUD、据点跳转、单位、地貌和 marker 层级。

## 非目标

- 不改变地图行列、tile 尺寸、坐标公式、地貌、单位模型或战斗规则。
- 不新增手势、惯性算法、相机状态、动画特效或第三方库。
- 不修改 `GameState`、`GameModels`、Xcode project 或用户未暂存文件。

## 当前架构依据

- `MapCommandCenter` 拥有地图可用 viewport 和缩放 Picker。
- `HexMapView` 拥有基于场景行列与固定 tile 尺寸计算的内容宽高。
- `ScrollViewReader` 已通过 tile id 聚焦 `focusedCoordinate`，无需新增规则或持久相机状态。

## 实现步骤

1. 把 `MapCommandCenter` 的地图区域放入 `GeometryReader`，将 viewport 高度传给 `HexMapView`。
2. `HexMapView` 用未缩放内容高度计算 `fillScale = max(0.86, availableHeight / contentHeight)`。
3. `MapScaleMode` 改为满幅基线上的 1.0 / 1.16 / 1.32 倍率。
4. resolved scale 同时应用于 `scaleEffect` 与外接滚动 frame，保持命中内容和滚动范围一致。
5. 缩放模式改变时复用现有 `scrollToFocusedCoordinate`，并继续尊重 Reduce Motion。
6. 地图内边距从 22 收敛为 8，避免重新引入明显空带。

## 风险和禁止项

- 高竖屏 viewport 会增大默认 scale 并增加横向滚动距离；必须以当前焦点居中保证首屏有效。
- 紧凑或低高度 viewport 不得缩小到 0.86 以下，避免单位和 marker 过小。
- 禁止用固定 iPad 高度或 `UIScreen.main.bounds` 计算缩放。
- 禁止改变地图规则、坐标、tile frame、命中路径或 marker 层级。
- 禁止提交 `AGENT.md` 删除或 `project.pbxproj` 用户改动。
- 禁止使用 v2.5 run 或截图冒充 v2.6 验收。
