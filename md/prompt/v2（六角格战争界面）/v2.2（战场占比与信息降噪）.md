# v2.2 战场占比与信息降噪

## 版本号与分配依据

v2.2。v2.1 已由 run `29426676489`、attempt `1`、commit `b2d97b685448a2aec8d8d0296feb041d8839d61c` 建立云端截图链，artifact `ww2tactics-ci-v2.1-main-b2d97b6-run29426676489-attempt1` 中的 iPad 首屏截图成为本轮权威视觉证据。

## 背景

截图证明 app 正常启动、六角格和单位可辨，但首屏仍偏高密度调试指挥台：TopCommandBar 因横向内容溢出退化为三行，地图内重复战役 HUD 遮挡地形，地图下方编队/战术/增援/图例常驻占用高度，每个普通格都显示地形代码，默认六角边线与地形色差形成明显棋盘墙，单位主体相对地格偏小且地图单位没有传入真实补给状态。

## 目标

1. 顶栏固定为一行：战役标题、战役菜单、可横滚核心状态和 44pt 结束回合按钮。
2. 地图内删除与顶栏/Inspector 重复的战役 HUD，只保留选中单位动作 HUD；普通选中状态不显示冗余命令预览。
3. 编队、战术、增援和图例进入默认收起的 44pt 支援甲板，所有入口仍可展开访问。
4. 普通地格不再常驻 `PLN/RIV/RD` 等代码，仅空地焦点显示；默认六角边线降到背景级，交互/风险/据点边线优先级不变。
5. 调整地貌色阶使平原、森林、道路、河流等保持可辨但更接近连续战区。
6. 放大四类单位主体和 HP 条，地图单位传入真实补给状态，删除重复 READY/DONE/HP 状态条。

## 非目标

- 不修改 `GameState`、`GameModels`、坐标、命中路径、移动/攻击/AI/补给规则。
- 不删除 MOVE/ATK/POS/THR/INT/PRS/RSP/AI 等战术 marker。
- 不新增图片资产、第三方库、3D 引擎或复杂动画。
- 不重写 Inspector 和结果卡。
- 不触碰工作树已有的 `project.pbxproj` 与 `AGENT.md` 改动。

## 架构与实现

- `BattlefieldChrome.swift`：单行 TopCommandBar；CampaignPicker 改为图标菜单；状态条可横滚；BattlefieldView 引入本地 `isSupportDeckExpanded`；MapToolbar 单行化；地图 overlay 只保留 MapActionHUD；命令预览只在可执行时出现。
- `BattlefieldMap.swift`：默认边线和基础描边降噪；地形代码仅焦点空地显示；移除地图专用 UnitStatusPlate；收敛地貌色阶。
- `BattlefieldUnitViews.swift`：地图 UnitCounter 放大、HP 条稳定放大，并传入 `game.supplyState(for:)`。
- 支援甲板只保存表现层展开状态，不复制或改变任何规则状态；Reduce Motion 下不执行展开动画。
