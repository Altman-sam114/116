# v2.7 战斗焦点与空闲 HUD 收敛

## 版本号与分配依据

- 版本号：`v2.7`
- 分配依据：v2.6 已由 `origin/main` commit `bfdaf9780ec0beba8eb8186ecca6e8e906ad253a` 对应的 GitHub Actions run `29885508105`、attempt `1` 和 artifact `ww2tactics-ci-v2.6-main-bfdaf97-run29885508105-attempt1` 完成云端验收。地图已按工作区高度满幅显示，本轮继续在 `v2（六角格战争界面）` 内递增小版本，收敛遮挡并强化战斗焦点。

## 背景

Agent X 对照 Google Play 官方《将军的荣耀3》战斗截图后确认，其核心构图是让地图持续占据视觉主体：空闲态不保留大块信息卡，选中单位使用蓝色地面光环和四角框，攻击目标使用红色或橙色准星，命令集中在屏幕边缘。当前 WW2Tactics 的 v2.6 已解决地图上下空带，但未选择单位时仍常驻宽约 268/304pt 的 `MapActionHUD`，且选中/攻击焦点主要依赖六角边框，战斗指向不够明确。

## 目标

1. 未选择单位时，以小型边缘 `NEXT` 指挥坞替代大型空闲 `MapActionHUD`，让地图信息保持可见。
2. 选择单位后保留现有紧凑单位信息、指标、命令和可执行预览，不删除规则入口。
3. 在选中单位下方绘制蓝色地面光环，并在选中地格显示清晰的四角选中框。
4. 当前聚焦的攻击目标显示红色四角准星，当前聚焦的接敌目标显示橙色四角准星。
5. 视觉反馈不只依赖颜色：光环、四角几何和既有 ATK/POS 文本标记共同表达状态。

## 非目标

- 不重写 `GameState`，不新增规则状态、动画状态或持久化状态。
- 不改变移动、攻击、AI、补给、据点、胜负、威胁覆盖或战术命令规则。
- 不改变 tile 尺寸、六角坐标、`position(for:)`、命中路径或 `HexInputReader` 输入链。
- 不改变 marker 的数据源、排序、槽位、短码和溢出规则。
- 不引入图片资产、第三方库、网络依赖或复杂资产管线。
- 本轮不实现战斗结算动画、镜头震动、粒子、音效或新战役。

## 当前架构依据

- `WW2Tactics/WW2Tactics/BattlefieldChrome.swift` 的 `MapCommandCenter` 负责地图覆盖层编排，`MapActionHUD` 只转发 `GameState` 的既有只读状态和命令入口。
- `WW2Tactics/WW2Tactics/BattlefieldMap.swift` 的 `HexTileView` 已接收 `unit`、`isSelected`、`isFocused` 和 `actionHint`，足以只读派生选中光环与攻击/接敌准星。
- `WW2Tactics/WW2Tactics/GameState.swift` 继续是唯一核心规则状态机；本轮不得将规则写入 View。

## 实现步骤

1. 在 `MapCommandCenter` 中按 `game.selectedUnit` 切换覆盖层：选中时显示 `MapActionHUD`，未选中时显示独立的小型 `NEXT` 指挥坞。
2. 删除 `MapActionHUD` 内原有的大型空闲态文案分支，使其只承担选中单位的紧凑操作信息。
3. 小型指挥坞保持至少 44pt 点击区、明确的 `NEXT` 文本和 VoiceOver 标签，并继续调用 `game.selectNextReadyUnitFromMap()`。
4. 在 `HexTileView` 的单位层下方加入蓝色椭圆地面光环，仅在 `isSelected && unit != nil` 时显示。
5. 在 tile 内容上层加入四角准星：选中态使用蓝白高对比；`isFocused && actionHint.isAttack` 使用红色；`isFocused && actionHint.isApproachAttack` 使用橙色。保留既有六角 border 优先级和 marker。
6. 准星采用静态轮廓，天然满足 Reduce Motion；不得用持续动画导致每个六角格维护独立动画状态。
7. 同步 README、flow、flowchart、test、prompt README 和 update log，并把 v2.6 等待验收记录改为实际成功记录。

1. 云端首屏截图中原右上大型空闲信息卡消失，替换为明显更小的边缘 `NEXT` 指挥坞。
