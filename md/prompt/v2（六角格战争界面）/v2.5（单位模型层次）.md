# v2.5 单位模型层次

## 版本号与分配依据

- 版本号：`v2.5`。
- 分配依据：v2.4 已由 `origin/main` run `29604249640` 的 iPad 截图验收，地图占比与地貌拼花得到收敛。最新截图中单位仍以单色合并剪影呈现，内部军械结构不清晰，因此继续 v2 小版本强化单位模型。

## 背景

《Glory of Generals 3》参考图中的单位是地图第一视觉层级：不同阵营使用不同军械色调，坦克履带/炮塔、火炮轮组/炮架、侦察车轮毂/车体、步兵队形和枪线清楚。当前 `UnitShapeBadge` 虽有四类外轮廓，但统一浅灰渐变把重叠部件合并为单块，远看仍像扁平符号。

## 目标

- 为盟军与轴心军械建立不同但克制的模型色板，不只依赖蓝/红底座识别阵营。
- 在现有四类剪影内增加稳定结构线，强化兵种辨识。
- 增加模型落地阴影、边缘高光和暗部，形成轻量立体兵棋感。
- 将地图 HP 条改为稳定五段刻度，保留现有 HP 比例和红/绿阈值。
- 保留将领、军衔、补给、战术状态和 spent 状态的现有信息层级。

## 非目标

- 不引入图片、3D、动画、粒子、第三方库或新资产管线。
- 不改变 `BattleUnit`、`UnitKind`、战斗数值、规则状态或 AI。
- 不扩大六角格、改变单位 frame、地图命中区域、marker 槽位或输入事件。
- 不修改 Inspector、地貌、HUD、Xcode project 和用户未暂存文件。

## 当前架构依据

- `BattlefieldUnitViews.swift` 负责地图单位、单位徽标、阵营底座、HP 和将领徽章。
- `UnitShapeBadge` 被地图、编队条、HUD 和 Inspector 复用；升级必须适配 34x18 到 58x37 的现有尺寸。
- `GameState` 只提供现有单位/补给状态；新层次全部由 `UnitKind`、`Faction` 和 `isSpent` 只读派生。

## 实现步骤

1. 把 `UnitShapeBadge` 中的单层 `UnitMarkerShape` 替换为独立 `UnitModelView`。
2. `UnitModelView` 叠加落地阴影、阵营军械渐变、外轮廓和结构线。
3. 新增 `UnitModelDetailShape`：分别绘制步兵队形/枪线、坦克履带/负重轮/炮塔、火炮轮毂/炮架、侦察车轮毂/车窗/天线。
4. 在 `Faction` 的纯视觉扩展中提供盟军橄榄色和轴心灰褐色模型色板。
5. `MiniHealthBar` 统一负责地图 HP，并增加五段刻度和细边框。
6. 保持 `UnitCounter` 尺寸、CommanderBadge 位置、rank/supply/tactical 标签与无障碍摘要不变。

- 最新 iPad 截图中坦克、步兵、火炮、侦察不依赖文字即可区分。
- 盟军与轴心军械主体色调不同，同时保留蓝/红阵营底座和非颜色形状差异。
- 坦克履带/炮塔、火炮轮组/炮架、侦察车车体/轮毂、步兵队形/枪线清楚但不盖住 marker。

- 禁止用阵营颜色作为唯一差异；底座形状、轮廓和现有标签必须保留。
- 禁止新增可变 UI/规则状态或随机绘制。
- 禁止修改 `UnitMarkerShape` 外接尺寸、地图 tile/input 和 marker 布局。
- 禁止提交 `AGENT.md` 删除或 `project.pbxproj` 用户改动。
- 禁止使用旧 run 或旧截图验收本轮。
