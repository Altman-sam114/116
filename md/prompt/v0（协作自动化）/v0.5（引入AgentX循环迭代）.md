# v0.5 / 引入 Agent X 循环迭代文档基线

## 版本号

v0.5

## 版本分配依据

人工指定本轮版本号为 v0.5，标题为“引入 Agent X 循环迭代文档基线”。本轮只改协作文档，为未来 `agentx:` 主控循环做准备。

## 背景

项目已有 Agent A / Agent B / Agent C 三角色流程：

- Agent A：分析目标并写版本化实现提示词。
- Agent B：实现、轻量检查、提交并 `git push origin main`。
- Agent C：下载 GitHub Actions artifact，核对 manifest、JUnit、日志和结果包。

需要新增未来角色 Agent X：围绕人工总目标 X 拆分多轮任务，并按 Agent A -> Agent B -> Agent C 循环调度。

## 目标

- 在文档中新增 Agent X 的召唤规则、职责、循环边界、停止条件和禁止项。
- 将现有 A/B/C 云端验证流程扩展为可被 Agent X 多轮调用。
- 更新 flow、flowchart、test、prompt README、README 和 update_log。
- 明确本轮只做文档准备，不启动真实 Agent X 循环。

## 非目标

- 不修改 Swift 源码。
- 不修改 Xcode project。
- 不修改 GitHub Actions workflow。
- 不新增测试 target。
- 不创建 PR。
- 不切换分支。
- 不 push，除非人工另行明确要求。

## 当前架构依据

- `AGENTS.md` 是入口规则和 Agent 协作方式。
- `md/flow/flow.md` 和 `md/flow/flowchart.md` 描述当前真实核心逻辑和协作流。
- `md/test/test.md` 定义本地轻量检查、云端 artifact 和 Agent C 验收规则。
- `md/prompt/README.md` 定义 Agent A 提示词目录和版本规则。
- `WW2Tactics/README.md` 面向使用者简述协作入口。

## 实现步骤

1. 更新 `AGENTS.md`，加入 `agentx` / `x:` / `X:` 召唤规则、Agent X 职责、循环边界、停止条件和禁止项。
2. 更新 `md/flow/flow.md`，加入 Agent X 主控循环阶段。
3. 更新 `md/flow/flowchart.md`，用 Mermaid 表达人工总目标、Agent X 拆轮次、A/B/C、GitHub Actions artifact、Agent C 验收和 Agent X 判断。
4. 更新 `md/test/test.md`，明确 Agent X 不得跳过 Agent C artifact 验收，并补充小数据量和下载容量限制。
5. 更新 `md/prompt/README.md`，说明 Agent X 可要求 Agent A 为每轮生成版本化提示词。
6. 更新 `WW2Tactics/README.md`，简短说明 `agentx:` 是主控循环入口。
7. 更新 `update_log.md`，记录本轮 v0.5 文档基线。

## 测试要求

- 本轮是文档-only 修改，至少运行 `git diff --check`。
- 若 Mermaid 图被修改，检查 Markdown 代码块闭合。
- 不运行 Swift parse、Xcode build 或 XCTest，除非误改 Swift / Xcode / workflow 文件。

## 文档更新要求

- 所有新增 Agent X 规则必须与现有 `main` 直推、GitHub Actions artifact、Agent C 下载验收流程一致。
- README 只写简短协作入口，不堆流程细节。
- update_log 必须记录本轮只做文档准备，不启动真实自动循环。

## 验收标准

- 文档明确支持未来 `agentx:` 主控循环。
- Agent X 不直接替代 A/B/C，不跳过 Agent C artifact 验收。
- 失败时不能继续下一轮伪装成功。
- `git diff --check` 通过。

## 风险和禁止项

- 禁止启动真实 Agent X 循环。
- 禁止修改 Swift 源码、Xcode project 或 workflow。
- 禁止 push，除非人工明确要求。
- 禁止把旧 run、旧 artifact 或本地输出冒充最新云端结果。
- 禁止默认下载大体积 artifact、模型、数据集或无关产物。
