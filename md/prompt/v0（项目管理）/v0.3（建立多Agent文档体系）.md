# v0.3（建立多 Agent 文档体系）

## 版本号

v0.3

## 版本分配依据

人工要求建立长期迭代的 Agent 协作系统、记忆文档、测试规范、核心流程文档和提示词目录。当前项目已有 SwiftUI 战棋原型和 README，但缺少标准化多 Agent 文档体系，因此归入 v0 项目管理阶段的小版本。

## 背景

`WW2Tactics` 是 SwiftUI iOS 二战回合制战棋原型，目标是持续逼近 EasyTech 风格的大地图战棋体验。当前源码集中在 `GameModels.swift`、`GameState.swift`、`ContentView.swift`，测试包括 XCTest 和命令行 smoke test。

## 目标

- 创建或更新 `AGENTS.md`。
- 创建 `update_log.md`。
- 创建 `md/prompt/README.md`。
- 创建 `md/test/test.md`。
- 创建 `md/flow/flow.md`。
- 创建 `md/flow/flowchart.md`。
- 保留 `AGENT.md` 作为兼容入口，指向 `AGENTS.md`。
- 内容必须贴合当前项目，不写空泛模板。

## 非目标

- 不修改 Swift 源码。
- 不新增功能。
- 不调整 Xcode 工程。
- 不运行完整业务测试。

## 当前架构依据

- `GameModels.swift`：模型和战役数据。
- `GameState.swift`：核心规则状态机。
- `ContentView.swift`：SwiftUI 表现和地图交互。
- `GameStateTests.swift`：XCTest。
- `RulesSmokeTest.swift`：规则 smoke test。
- `README.md`：当前功能与验证命令。

## 实现步骤

1. 阅读 pasted text 中的文档结构要求。
2. 阅读 README、现有 Agent 入口、源码摘要和测试入口。
3. 将入口规则整理为 `AGENTS.md`。
4. 将历史、决策、遗留事项写入 `update_log.md`。
5. 将测试命令和触发条件写入 `md/test/test.md`。
6. 将当前真实架构和执行流写入 `md/flow/flow.md`。
7. 将核心逻辑、地图命令和 Agent 循环画入 `md/flow/flowchart.md`。
8. 建立 `md/prompt/README.md` 和本 prompt 文件。
9. 运行 `git diff --check`。

## 测试要求

文档-only 修改至少运行：

```sh
git diff --check
```

不需要运行 Swift smoke/typecheck，除非同时修改 Swift 源码。

## 文档更新要求

- README 增加协作与维护文档入口。
- `update_log.md` 记录 v0.3 文档体系建立。

## 验收标准

- 所有要求的目录和文件存在。
- `AGENTS.md` 明确 Agent A/B/C 工作流。
- `md/test/test.md` 包含测试分层、命令、触发条件、当前基线。
- `md/flow/flow.md` 描述当前真实架构和执行流。
- `md/flow/flowchart.md` 至少包含核心逻辑图和 Agent 迭代流程图。
- `git diff --check` 通过。

## 风险和禁止项

- 不要把 `AGENTS.md` 写成历史流水账。
- 不要把 `flow.md` 写成愿景文档；它只记录当前真实逻辑。
- 不要伪造测试结果。
- 不要删除旧入口导致后续 Agent 找不到规则；保留 `AGENT.md` 兼容跳转。
