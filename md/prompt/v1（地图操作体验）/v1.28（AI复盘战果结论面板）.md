# v1.28 AI复盘战果结论面板

## 版本号

v1.28

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.27（AI复盘自动播放控制）。`origin/main` 最新验收 commit `9b11c5c3c49f84f5a9eaf4fd2159ce7bffef372b` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.27-main-9b11c5c-run28768394430-attempt1` 核对。v1.27 已能播放、暂停和调速浏览敌方行动；下一步应让玩家在复盘前后快速理解“敌方这一回合造成了什么战果”，因此本轮递增为 v1.28。

## 背景

当前 AI 回合摘要已有行动、伤害、承伤、后勤、占点、指令点和时间线，但玩家仍需要自己从指标和列表中归纳重点。EasyTech 风格战棋的回合反馈通常会给出明确战果结论，例如“敌军夺取据点”“火炮压制造成高伤害”“主要是部署整补”。本轮新增只读的 AI 复盘战果结论：从既有 `AIPhaseSummary.timeline` 和汇总字段派生结论类型、核心指标和关键事件，不重新计算 AI、不改变任何单位或据点状态。

## 目标

1. 在模型层新增 AI 复盘结论：
   - 新增 `AIPhaseReplayConclusion` 或等价模型，包含结论类型、标题、说明、指标列表和关键事件列表。
   - 结论类型至少区分：夺点突破、火力压制、后勤整备、机动推进、低强度回合。
   - 指标至少覆盖伤害/承伤、占点、后勤、指令点变化等既有 summary 字段。
   - 关键事件从既有 `AIPhaseTimelineEvent` 派生，优先展示占点、击毁、高伤害、整补/部署等事件，最多 3 条。
2. 保持只读边界：
   - 结论只从 `AIPhaseSummary` 和其 `timeline` 派生。
   - 不调用移动、攻击、战术命令、部署、整补、AI、回合推进或任何写状态方法。
   - 不改变 `latestAIPhaseSummary`、timeline、地图 marker、播放状态、单位、据点、指令点、战报或反制建议。
3. 强化 SwiftUI 展示：
   - 在 `AIPhaseSummaryView` 的基础指标之后、行动时间线之前显示“战果结论”区块。
   - 展示结论标题、说明、最多 4 个紧凑指标和最多 3 条关键事件。
   - UI 只展示模型字段，不自行判断结论类型、排序关键事件或重新计算指标。
   - 提供 VoiceOver 可读标签和 hint；不要新增动画、音效或真实美术资产。
4. 补充 XCTest 和规则 smoke test：
   - 覆盖移动后火炮弹幕场景应归类为火力压制，并包含伤害、指令、关键战术事件。
   - 覆盖直取据点场景应归类为夺点突破，并包含占点指标和占点关键事件。
   - 覆盖后勤整补/部署场景应归类或至少输出后勤指标与关键事件。
   - 覆盖空 timeline / 低强度 summary 仍生成稳定结论，不崩溃、不生成虚假关键事件。
5. 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不改变 AI 决策、行动顺序、时间线事件生成、地图复盘 marker 生成规则或播放控制语义。
- 不新增历史 AI 回合列表、逐帧动画、镜头动画、时间轴滑块、音效、真实美术资产或第三方库。
- 不让 SwiftUI 计算结论分类、指标或关键事件排序。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励、胜负或反制建议语义。

## 当前架构依据

- `AIPhaseSummary` 已集中记录 AI 回合后的动作计数、伤害、占点、歼灭、指令点变化和 timeline。
- `AIPhaseTimelineEvent` 已记录单条真实 AI 成功动作的类型、伤害、恢复、指令点、击毁和占点信息，适合作为结论关键事件来源。
- `AIPhaseSummaryView` 已展示汇总指标、时间线、播放/暂停和速度控制；新结论区块应插入其内部并只展示模型字段。
- `GameState` 只负责生成 `latestAIPhaseSummary`；本轮优先把结论做成 `AIPhaseSummary` 的纯派生属性，避免新增可过期状态。

## 实现步骤

1. 修改 `GameModels.swift`：
   - 增加结论类型枚举、指标类型枚举、指标模型和 `AIPhaseReplayConclusion`。

   - 在现有 AI post-move barrage、immediate objective、deployment/reinforcement smoke 中加入结论断言。
5. 文档同步：
   - README AI、地图交互、战斗辅助和 smoke 描述加入“战果结论”。
   - `md/flow/flow.md` AI 回合摘要段落加入结论派生规则。
   - `md/flow/flowchart.md` AI summary 到 UI 链路加入战果结论节点。
   - `md/test/test.md` smoke / XCTest 基线加入 AI 复盘战果结论。
   - `md/prompt/README.md` 当前建议递进到 v1.28 后续方向。
   - `update_log.md` 追加 v1.28 记录，并修正顶部当前状态。

## 验收标准

1. AI 回合摘要会派生稳定 `replayConclusion`，包含类型、标题、说明、指标和关键事件。
2. 火力压制、夺点突破、后勤整备、机动推进和低强度回合至少能按既有 summary/timeline 字段稳定区分。
3. 关键事件最多 3 条，优先占点、击毁、高伤害和后勤动作，顺序稳定。
4. 结论派生不改变单位、据点、指令点、summary、timeline、AI map marker、播放状态或反制建议。
5. SwiftUI 只展示模型字段，不计算结论分类或关键事件排序。
6. 本地轻量检查通过。
7. Agent B 在 `main` 提交并 `git push origin main`。
8. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
9. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：若 View 自行分类或排序关键事件，会与规则层失步；必须由模型派生。
- 风险：结论名称容易暗示真实 AI 归因；本轮只解释当前 summary/timeline 已记录事实，不模拟未发生事件。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负、播放控制和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘结论状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
