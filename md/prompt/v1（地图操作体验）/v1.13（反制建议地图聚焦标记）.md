# v1.13（反制建议地图聚焦标记）

## 版本号

v1.13

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新版本为 v1.12（反制建议点选聚焦）。本轮继续沿着 v1 地图操作体验推进，不开启新大版本，因此递增为 v1.13。

## 背景

v1.11 已生成敌方意图反制建议，v1.12 已允许玩家点击建议行来聚焦执行单位、敌军、撤退目的格、据点防守格或整补单位当前位置。当前地图会因为 `focusedCoordinate`、`guidedObjectiveCoordinate` 和已有路线/OBJ/INT 标记发生变化，但玩家仍需要从侧栏文字反推“这条建议的威胁来源、执行单位、目标/目的地分别在哪里”。

为了更接近 EasyTech 式大地图态势阅读，本轮要把已聚焦的反制建议投射成清晰地图标记，让玩家在地图上直接看出当前参谋建议的源点和目标点，同时保持建议仍是只读预览。

## 目标

1. 在 `GameState` 提供当前聚焦反制建议的地图标记派生信息，至少能区分：
   - 执行单位/受威胁单位坐标。
   - 威胁来源敌军坐标。
   - 建议目的格或被威胁据点坐标。
   - 当前聚焦建议类型。
2. `ContentView` 在地图格上渲染当前聚焦反制建议标记，例如：
   - 执行/受威胁单位格显示 `ACT` 或等价短码。
   - 威胁来源格显示 `SRC` 或等价短码。
   - 建议目的格/被威胁据点显示 `CTR` 或等价短码。
3. 标记只在某条反制建议已被点选且当前状态仍能匹配该建议时显示；普通聚焦、切换单位、执行移动/攻击、回合变化、战役重开后不得残留过期标记。
4. 地图图例和无障碍描述同步说明反制建议标记。
5. 保持 UI 只消费 `GameState` 派生状态，不在 `ContentView` 重新计算路线、战斗、整补或建议有效性。

## 非目标

- 不自动执行移动、攻击、整补、部署或战术命令。
- 不改变敌方意图生成、反制建议生成、排序或 limit 策略。
- 不改变 AI 优先级、战斗数值、移动规则、整补成本、胜负条件或回合流程。
- 不引入第三方库、真实美术资产、动画、音效或复杂资产管线。
- 不把复杂规则写进 SwiftUI View。

## 当前架构依据

- `GameModels.swift` 定义 `EnemyThreatCountermeasurePreview` 与相关枚举。
- `GameState.swift` 是核心状态机，已经有 `focusEnemyThreatCountermeasure(_:)` 和 `isEnemyThreatCountermeasureFocused(_:)`。
- `ContentView.swift` 只负责渲染地图、侧栏、图例和转发点击事件，不能在 View 中二次计算核心规则。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已覆盖反制建议生成、点选聚焦和无副作用，本轮应扩展这些测试覆盖地图标记派生状态。

## 实现步骤

1. 在 `GameModels.swift` 或 `GameState.swift` 选择符合现有风格的位置定义轻量地图标记结构。若结构会被 UI 和测试共同使用，优先放在 `GameModels.swift`；若只作为 `GameState` 内部派生类型，保持局部实现即可。
2. 在 `GameState` 增加只读属性或方法，例如 `focusedEnemyThreatCountermeasureMapMarkers`，基于当前 `visibleEnemyThreatCountermeasurePreviews` 找到 `isEnemyThreatCountermeasureFocused(_:) == true` 的建议，再重新输出 ACT/SRC/CTR 坐标集合。
3. 标记生成必须复用 v1.12 的有效性边界：
   - 只有仍能匹配当前聚焦建议时才返回标记。
   - 过期建议、普通聚焦、切换单位、执行行动或战役切换后不能显示旧标记。
   - 查询标记不能产生副作用，不能修改 `selectedUnitID`、`focusedCoordinate`、`guidedObjectiveCoordinate`、HP、位置、行动状态、指令点、战报或 `latest*Result`。
4. 更新 `HexMapView` 和 `HexTileView`，把标记布尔值或短码传入地图格渲染：
   - 保持现有布局稳定，不让文字撑大格子。
   - 标记不能遮挡 MOVE/ATK/POS/NEXT/OBJ/INT 等关键反馈；必要时放在已有角标层。
   - 对重叠标记使用稳定优先级或短码组合，避免同一格多层文字不可读。
5. 更新 `MapLegendView`，加入反制建议标记说明。
6. 更新 `HexTileView` 的 accessibility 文本，让 VoiceOver 能读出当前格是否为反制建议执行点、威胁来源或建议目标。
7. 扩展 XCTest：
   - 点选抢先打击后能得到执行单位、威胁来源和建议目标标记。
   - 点选撤出危险区后能得到执行单位、威胁来源和撤退目的格标记。
   - 点选据点防守后能得到执行单位、威胁来源、目的格和被威胁据点/目标标记。
   - 点选整补支撑后能得到受威胁单位和建议目标标记。
   - 普通聚焦或过期建议不会保留旧标记。
   - 查询标记无副作用。
8. 扩展 `RulesSmokeTest.swift` 的反制建议聚焦段，加入同类轻量断言。
9. 同步更新：
   - `WW2Tactics/README.md`：用户可见地图交互和战斗辅助说明。
   - `md/flow/flow.md`：当前核心逻辑中反制建议聚焦与地图标记。
   - `md/flow/flowchart.md`：必要时更新地图命令/规则图节点。
   - `md/test/test.md`：当前基线增加反制建议地图标记覆盖。
   - `update_log.md`：新增 v1.13 记录。

## 验收标准

1. 点击任一当前有效反制建议后，地图能显示该建议的执行点、威胁来源和建议目标/目的地标记。
2. 这些标记来自 `GameState` 派生状态，`ContentView` 不重新计算规则。
3. 查询或显示标记不消耗行动、不移动单位、不攻击、不整补、不改 HP/指令点/战报/结果摘要。
4. 过期建议、普通聚焦、执行动作、重开或切战役后不会残留旧反制标记。
5. XCTest、规则 smoke、SwiftUI typecheck 和测试源码 typecheck 通过。
6. `main` 已推送，GitHub Actions 最新 run 成功。
7. Agent C 已下载最新 artifact，核对 manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与 `origin/main` 一致，并检查 failure summary、JUnit、rules smoke、xcodebuild log 和 `.xcresult`。

## 风险和禁止项

- 禁止让 `ContentView` 根据文字字段或坐标自行判断建议是否有效。
- 禁止新增会执行命令的“反制建议按钮”。
- 禁止用静态缓存保存标记导致过期状态残留。
- 禁止为了显示标记破坏 MOVE/ATK/POS/NEXT/OBJ/INT/CAP 等现有反馈。
- 禁止扩大范围到 AI、数值、战役数据、真实资产或动画。
- 禁止跳过 `main` 直推和 Agent C artifact 验收。
