## 2. 版本分配依据

- 当前 `md/prompt/` 只有 `v0（项目管理）` 阶段提示词，`update_log.md` 最新功能/流程状态为 v0.5。
- v0.x 已主要完成协作制度、测试规范和云端结果包流程，不再继续扩展项目管理主题。
- `md/prompt/README.md` 已建议下一轮从 `v1（地图操作体验）` 开始。
- 人工本轮要求是“总结目前进展，设计下一步优化目标，向游戏继续推进”，应回到地图可玩性、操作反馈和可测试规则反馈。
- 因此本轮新开 `v1（地图操作体验）`，首个任务为 `v1.0（路线与战斗预判强化）`。

## 3. 背景和目前进展总结

当前项目已经具备以下基础：

- SwiftUI iOS 原型，目标体验是 EasyTech 风格二战回合制战棋。
- 已有阿登 22x14 大地图、14 个据点、诺曼底战役、战役切换、编队条、侧栏、HUD、图例和战报。
- 规则层已有移动、攻击、反击、地形、兵种克制、补给、士气、经验、将领、待命防御、夹击、机动追击、据点奖励、据点休整、指令点、战术命令、增援、简易 AI、胜负和三星目标。
- 地图交互已有左键/点按聚焦、右键/执行按钮执行 MOVE/ATK/POS、移动后 NEXT 聚焦、OBJ 据点引导和 THR 敌火覆盖提示。
- 测试层已有 `GameStateTests.swift` 和 `Tools/RulesSmokeTest.swift`，覆盖主规则链、地图命令预览、OBJ/THR、AI、战术命令和非法操作不消耗行动。
- 协作流程已切换为 `main` 直推、GitHub Actions 云端重验证、Agent C 下载未加密 CI artifact 验收。

当前最明显的体验缺口：

- 地图已经能显示路径高亮，但路线信息仍偏粗：玩家只能看到路径格和总消耗，不容易判断每一步为什么贵、哪里进入控制区、哪里暴露在敌火下。
- 移动后攻击已有目标名提示，但缺少“从目的地发起攻击后大概造成多少伤害、是否会被反击、能否击毁”的决策信息。
- 现有 `THR` 是危险移动格总提示，尚未和当前聚焦路线绑定成一条可读的行军风险摘要。

本轮应优先强化“移动前决策质量”，让玩家在执行移动前就能读懂路线成本、风险和移动后火力窗口。

## 4. 目标

### 4.1 规则/状态层目标

在 `GameState` 中增加可测试的路线情报和移动后攻击预判能力，供 SwiftUI 展示使用：

- 为当前聚焦 MOVE/POS 路线提供逐步路线预览：
  - 步序号。
  - 坐标。
  - 进入该格的地形移动消耗。
  - 进入敌方控制区产生的额外消耗。
  - 是否暴露在敌方火力下。
  - 威胁来源名称，至少保留前 2 个，数量更多时可显示总数。
  - 是否为终点。
- 为当前聚焦 MOVE 路线提供移动后攻击预判：
  - 目标名称。
  - 预计伤害。
  - 反击伤害。
  - 目标攻击后剩余耐久。
  - 是否预计击毁。
  - 结果排序应稳定，优先击毁，其次伤害高，其次目标耐久低或名称稳定排序。
- 暴露给 UI 的新数据必须来自 `GameState` 或模型层，不得在 `ContentView` 里重新实现规则。
- 不改变实际移动、攻击、AI 和胜负结算数值；本轮只增强预览和可读反馈。

建议命名，可按现有风格微调：

- 在 `GameModels.swift` 增加 `RouteStepPreview: Equatable, Identifiable`。
- 在 `GameModels.swift` 增加 `PostMoveAttackPreview: Equatable, Identifiable`。
- 在 `GameState.swift` 增加：
  - `func routeStepPreviews(for unit: BattleUnit, route: MovementRoute) -> [RouteStepPreview]`
  - `var focusedRouteStepPreviews: [RouteStepPreview]`
  - `func postMoveAttackPreviews(for unit: BattleUnit, to coordinate: HexCoordinate) -> [PostMoveAttackPreview]`
  - `var focusedPostMoveAttackPreviews: [PostMoveAttackPreview]`

实现提示：

- `RouteStepPreview` 应基于 `MovementRoute.coordinates.dropFirst()`，起点不算进入步骤。
- 每一步的移动消耗可复用 `movementCostPreview(for:entering:)`，控制区额外消耗可复用 `enemyControlZonePenalty(for:entering:)`。
- 每一步威胁来源可复用 `threateningEnemies(against:at:)`。
- 路线步骤的总移动消耗之和应等于 `MovementRoute.totalCost`；步骤控制区额外消耗之和应等于 `MovementRoute.controlZonePenalty`。
- 移动后攻击预判可以创建一份临时 attacker，将 `position` 改为目的地后调用现有 `combatPreview(attacker:defender:)`，这样将领协同、地形、补给和夹击仍由现有规则计算。
- 只对合法 `movementRoute` 目的地返回移动后攻击预判，且选中单位必须还能攻击。

### 4.2 UI 目标

在 `ContentView.swift` 中增强现有地图和预览展示：

- 当前聚焦路线上的格子继续高亮，但路线标记应能显示步序号和/或该步消耗。
- 如果路线步骤暴露在敌火下，标记需要有非纯颜色差异，例如警告图标或 `THR` 小标，不只靠红色区分。
- `InlineMapCommandPreview` 和 `FocusedCommandPreviewPanel` 中的 MOVE/POS 详情应加入：
  - 路线步数。
  - 总消耗。
  - 控制区额外消耗。
  - 受威胁步骤数量。
  - 威胁来源摘要。
- MOVE 预览若存在移动后可攻击目标，应显示最佳目标的预计伤害、反击、是否击毁；没有目标时保留现有“移动后暂无射程内目标”语义。
- POS 接敌预览应显示到攻击位的路线风险，并说明移动后目标进入射程；不直接执行攻击。
- 地图格的 `accessibilityLabel` 应包含路线步序、消耗、控制区、敌火风险等新信息。
- 不新增第三方库，不引入真实美术资产，不重做整体布局。

### 4.3 测试目标

新增或修改测试，锁住本轮新增的状态层行为：

- `GameStateTests.swift`
  - 增加路线步骤预览测试：步骤数量、起终点、每步成本之和、控制区额外消耗之和、威胁来源。
  - 增加移动后攻击预判测试：聚焦 MOVE 目的地后，最佳目标名称、伤害、反击、击毁标记与 `combatPreview` 一致。
  - 增加 POS 路线风险测试：聚焦射程外敌军时，`focusedRouteStepPreviews` 对应推荐攻击位路线。
- `Tools/RulesSmokeTest.swift`
  - 在现有 post-move / zone-of-control / THR 片段中追加少量断言，确认路线步骤预览和移动后攻击预判可用。

测试不要依赖 UI 文案的完整句子，除非该文案是用户关键反馈。优先断言结构化数据。

## 5. 非目标

- 不改变移动可达算法的结果。
- 不改变攻击、反击、伤害、士气、补给、经验、AI、胜负或据点评价结算。
- 不实现战斗动画、镜头动画、真实地图贴图、单位立绘或将领头像。
- 不重构 `ContentView.swift` 的整体文件结构；只做本轮必要的小范围视图拆分。
- 不新增保存系统、生产队列、科技系统或新战役。
- 不引入第三方库、网络依赖或复杂资产管线。
- 不创建 PR，不创建长期分支，不使用 `develop` / `smalldata_test` / `codeb/...`。

## 6. 当前架构依据

必须遵守当前项目边界：

- `GameModels.swift`：放结构化数据类型，如 `RouteStepPreview`、`PostMoveAttackPreview`。
- `GameState.swift`：放核心规则查询和预览生成，如路线步骤、威胁来源、移动后攻击预判。
- `ContentView.swift`：只渲染 `GameState` 给出的结果并转发输入，不重新计算核心规则。
- `GameStateTests.swift`：优先覆盖状态机和预览数据。
- `RulesSmokeTest.swift`：覆盖主规则链，不替代 XCTest。

SwiftUI 约束：

- 不把业务规则写进 `body`。
- 图标按钮必须有可用的无障碍标签。
- 新增颜色含义时必须同时有文字、图标或形状差异，不能只靠颜色。
- 避免在地图 `body` 内做昂贵重复计算；能由 `GameState` 产出的数据不要在 View 中重新排序和推导。

## 11. 验收标准

功能验收：

- 选中单位并聚焦可移动格时，地图显示路线步序/成本，终点清晰。
- 聚焦路线中若有控制区或敌火风险，HUD/侧栏和地图都能读出风险。
- 聚焦 MOVE 目的地且移动后可攻击时，预览显示最佳攻击目标和伤害/反击/击毁信息。
- 聚焦射程外敌军时，POS 推荐攻击位仍可执行，且路线风险可读。
- 左键/点按聚焦不消耗行动，右键/执行按钮才执行的既有铁律不变。
- 实际移动、攻击、AI、胜负结算不因本轮变化而改变。

测试验收：

- `GameStateTests.swift` 新增结构化断言覆盖路线步骤、风险和移动后攻击预判。
- `RulesSmokeTest.swift` 覆盖主链路中的新预览数据。
- 本地 `git diff --check` 通过。
- 云端 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 匹配。
- 云端 smoke 和 Xcode build-for-testing 通过，或失败时 Agent C 能从 artifact 明确指出失败点。

## 12. 风险和禁止项

风险：

- `ContentView.swift` 已很大，新增 UI 时容易继续堆复杂逻辑；必须把规则判断留在 `GameState`。
- `MapCommandPreview` 已被大量测试等值断言覆盖，扩展 case 会扩大修改面；优先新增旁路预览数据。
- `movementRoutes(for:)` 当前会频繁被 UI 查询，新增预览不要在多个 View 里重复做昂贵计算。
- 移动后攻击预判使用临时 attacker 时，要确认不会写回 `scenario.units`。

禁止项：

- 禁止把路线成本、威胁来源、移动后伤害计算写进 View。
- 禁止改变实际规则结算来迎合 UI 展示。
- 禁止新增不可测试的核心玩法状态。
- 禁止只做装饰而不增加决策信息。
- 禁止引入第三方库、网络依赖或复杂资产管线。
- 禁止绕过 `main` 直推和云端结果包验收流程。
- 禁止把旧 artifact、旧 output 或 checkout 自带报告冒充本轮云端结果。
