# v1.58（攻击预览与战斗结果反馈卡视觉统一）

## 版本号

v1.58

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.57。v1.56 已建立战场界面视觉基线，v1.57 已升级地图格、地形标签、据点名牌和单位棋子。本轮继续在 `v1（地图操作体验）` 下递增，聚焦战斗前后的读数反馈。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》类大地图战棋体验。当前地图外壳和棋盘质感已完成初步重构，但侧栏中的攻击目标、战斗预测和战斗结果卡仍偏普通列表样式，和新战区指挥台视觉基线不完全一致。战斗读数是玩家判断移动、攻击、反击、击毁和追击的核心反馈，需要比一般信息卡更突出。

## 目标

- 统一 `AttackTargetsView`、`AttackTargetButton`、`CombatForecastView`、`CombatResultSummaryView` 和 `CombatantResultLine` 的战斗反馈视觉语言。
- 让攻击目标列表更像可执行战术目标牌：明确聚焦态、伤害、反击、击毁风险和 HP 信息。
- 让战斗预测卡更突出攻防双方、伤害/目标剩余/反击三项核心读数，以及地形、兵种、将领、夹击、防御姿态等影响来源。
- 让普通攻击结果卡更像战报结果牌：顶部结论、攻防双方 HP 变化、核心指标和后续效果层级更清楚。
- 复用已有 `BattlefieldTheme`、`tacticalSurface` 和现有颜色 helper，不引入第三方库、图片资产或新规则模型。

## 非目标

- 不修改 `GameModels.swift`、`GameState.swift`、攻击、反击、伤害、士气、经验、追击或据点规则。
- 不合并 `InlineMapCommandPreview` 与 `FocusedCommandPreviewPanel`，该重复结构后续单独处理。
- 不重构战线态势面板、AI 复盘时间线、反制建议、后勤结果、据点占领结果或战术命令结果。
- 不改 `CombatResultMetric` 的全局表现，避免影响其他复用面板。
- 不新增动画、音效、网络依赖、复杂资产管线或不可测试状态。

## 当前架构依据

- `ContentView.swift` 只负责 SwiftUI 表现层，所有攻击预览和结果数据都来自 `GameState` / `GameModels`。
- `AttackTargetsView` 通过 `game.attackableUnits(for:)` 读取可攻击目标；按钮只调用 `game.handleTap(on:)` 聚焦目标，不直接执行攻击。
- `CombatForecastView` 消费 `CombatPreview`，只展示预览，不写回状态。
- `CombatResultSummaryView` 消费 `CombatResultSummary`，只展示最近一次真实普通攻击结果。
- `InspectorPanel` 中普通攻击、战术命令、据点占领、部署、整补结果互斥展示，本轮只改普通攻击结果卡表现。

## 实现步骤

1. 在 `ContentView.swift` 中窄范围调整攻击目标列表：
   - 给 `AttackTargetsView` 增加主题化标题、计数徽标、空状态卡。
   - 给 `AttackTargetButton` 增强聚焦态、目标 HP、伤害、反击和击毁提示。
   - 保持按钮动作仍为 `game.handleTap(on:)`。
2. 调整 `CombatForecastView`：
   - 改为更清晰的战斗预测卡结构。
   - 顶部显示攻击方、目标方和结论。
   - 三项核心指标继续使用局部 `ForecastMetric`，不要改全局 `CombatResultMetric`。
   - 保留现有 matchup、terrain、commander、support、entrenchment 文案。
3. 调整 `CombatResultSummaryView` 和 `CombatantResultLine`：
   - 增强顶部结论牌、攻防双方结果行和后续效果列表。
   - 保持 `detailRows` 纯派生展示，不新增状态。
4. 如需新增小型私有 View/helper，只放在 `ContentView.swift` 对应区域，命名清楚，避免跨模块抽象。
5. 同步文档：
   - `WW2Tactics/README.md` 写明 v1.58 战斗预览/结果卡视觉升级。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md` 更新 ContentView 表现职责。
   - `md/test/test.md` 如需补充 UI 战斗反馈卡云端验证口径，写当前真实策略。
   - `md/prompt/README.md` 更新当前建议。
   - `update_log.md` 追加 v1.58 记录。
