# v1.2（战斗结果回放与战损对比）

## 版本号

v1.2

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 已有 `v1.0（路线与战斗预判强化）.md` 和 `v1.1（火力风险与安全接敌建议）.md`。本轮继续服务于地图操作体验，聚焦攻击执行后的可读反馈，因此递增为 `v1.2`。

## 背景

当前项目已能在攻击前显示 `combatPreview`、MOVE/POS 路线风险、移动后攻击预判和火力暴露风险；但普通攻击执行后，玩家主要依赖 `message` 和 `battleLog` 的文字记录理解战斗结果。EasyTech 风格战棋需要更直接的战损对比：谁攻击谁、双方攻击前后耐久、造成伤害、反击伤害、目标是否被击毁、攻击者是否遭反击损伤、经验/士气/晋升等结果。

## 目标

- 在 `GameModels.swift` 新增可测试的战斗结果摘要模型，用结构化字段记录普通攻击执行后的关键前后状态。
- 在 `GameState.swift` 新增最新普通攻击结果状态，并在 `attack(attackerID:targetID:)` 中从真实规则结算前后填充。
- 摘要必须包含攻击方/防守方名称、兵种、阵营、攻击前后 HP、伤害、反击伤害、目标击毁、攻击者是否被反击击毁、夹击加成、机动追击、攻击者/防守者经验和士气变化、攻击者/防守者军衔变化、防御姿态是否被消耗。
- 攻击结果摘要应在新战役、重开、切换战役时清理，避免显示旧战斗。
- 在 `ContentView.swift` 的侧栏战报附近增加紧凑战斗结果回放面板，展示双方 HP 前后对比、伤害/反击、击毁/追击、士气/经验/晋升变化。View 只能读 `GameState` 的结构化摘要，不能从 `battleLog` 文本反推规则。
- 补充 XCTest 和规则 smoke test，覆盖普通攻击、反击、击毁/机动追击、防御姿态消耗、摘要不会被预览写入等行为。
- 同步更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 非目标

- 不新增战斗动画、音效、美术资产或截图。
- 不调整伤害公式、AI 决策、士气/经验数值、机动追击规则或战斗平衡。
- 不把战术命令火炮弹幕/突破突击纳入本轮摘要，除非实现普通攻击摘要时已有低风险复用点；本轮验收只要求普通攻击。
- 不改变 MOVE/POS/ATK 的执行条件，不让聚焦动作消耗行动。
- 不引入第三方库、网络依赖或复杂资源管线。

## 当前架构依据

- `GameModels.swift` 已定义 `CombatPreview`、`PostMoveAttackPreview`、`PostMoveFireExposurePreview` 等结构化预览模型，战斗结果摘要应沿用这种模型化方式。
- `GameState.swift` 是核心规则状态机；普通攻击结算落点是 `private func attack(attackerID:targetID:)`，反击落点是 `counterAttackIfPossible(defenderID:attackerID:)`。
- `ContentView.swift` 只能展示状态和转发用户意图。战斗结果面板必须读取 `GameState.latestCombatResult` 一类结构化状态，不得在 View 内计算核心规则。
- `GameStateTests.swift` 和 `Tools/RulesSmokeTest.swift` 是本轮主要验证入口。

## 实现步骤

1. 在 `GameModels.swift` 增加结构化摘要类型，建议包括：
   - `CombatantResultSnapshot`：单位 ID、名称、兵种、阵营、开始 HP、结束 HP、开始经验、结束经验、开始士气、结束士气、开始军衔、结束军衔。
   - `CombatResultSummary`：攻击方快照、防守方快照、damage、counterDamage、supportDamageBonusPercent、didDestroyDefender、didDestroyAttacker、didTriggerManeuverPursuit、didConsumeDefenderEntrenchment。
   - 可增加便于 UI 展示的派生属性，如 HP delta、经验 delta、士气 delta、是否晋升。
2. 在 `GameState.swift` 增加 `@Published private(set) var latestCombatResult: CombatResultSummary?`。
3. 在 `attack(attackerID:targetID:)` 中：
   - 结算前保存攻击方和防守方起始快照。
   - 使用现有 `damageValue`、`counterDamageValue`、`awardExperience`、`adjustMorale`、机动追击等真实逻辑，不改变公式和副作用顺序。
   - 结算后从 `scenario.units` 读取双方最终状态，写入 `latestCombatResult`。
   - 确保反击造成的攻击方 HP、经验、士气变化进入摘要。
   - 确保防御姿态是否被消耗、夹击加成和机动追击进入摘要。
4. 在重开、切换战役等重置路径清空 `latestCombatResult`。普通聚焦、预览、MOVE/POS 不能写入新的战斗结果摘要。
5. 在 `ContentView.swift` 的 `BattleLogView` 或侧栏战报附近增加 `CombatResultSummaryView`，要求：
   - 首屏/侧栏紧凑，不能遮挡地图。
   - 使用系统图标和 Dynamic Type 字体。
   - 展示攻击方和防守方 HP 前后，如 `耐久 42 -> 29`。
   - 展示伤害、反击、击毁、追击、夹击、士气/经验变化和晋升提示。
6. 在 `GameStateTests.swift` 新增或扩展测试：
   - 普通攻击后 `latestCombatResult` 存在，字段与真实单位 HP/经验/士气/军衔一致。
   - 反击场景记录 `counterDamage` 和攻击者 HP 变化。
   - 击毁场景记录 `didDestroyDefender`，坦克/侦察机动追击场景记录 `didTriggerManeuverPursuit`。
   - 防御姿态被攻击消耗时记录 `didConsumeDefenderEntrenchment`。
   - `combatPreview`、MOVE/POS 预览不会写入或改变 `latestCombatResult`。
7. 在 `Tools/RulesSmokeTest.swift` 增加轻量断言，覆盖一次普通攻击后摘要的 damage、HP 前后、反击或击毁标记。
8. 更新文档：
   - `WW2Tactics/README.md` 已实现列表加入战斗结果回放和战损对比。
   - `md/flow/flow.md` 攻击流程加入 `latestCombatResult`。
   - `md/flow/flowchart.md` 核心执行图或规则状态图加入战斗结果摘要。
   - `update_log.md` 追加 v1.2 记录，先写本地轻量检查，云端 run/artifact 由 Agent C 验收后补充或确认。

## 文档更新要求

- 用户可见行为变化必须更新 `WW2Tactics/README.md`。
- 核心规则状态变化必须更新 `md/flow/flow.md` 和 `md/flow/flowchart.md`。
- 测试策略不变时无需修改 `md/test/test.md`。
- 追加 `update_log.md` 的 v1.2 记录，记录本地轻量检查、commit、push 和 Agent C 云端验收事实。

## 验收标准

- 普通攻击执行后，`latestCombatResult` 准确反映攻击方和防守方前后 HP、经验、士气、军衔、伤害、反击、击毁、追击、夹击和防御姿态消耗。
- `combatPreview`、MOVE/POS 聚焦预览和安全接敌预览不会创建虚假的战斗结果摘要。
- UI 侧栏能在战报附近显示最新战斗结果，不遮挡地图，不把规则写进 View。
- XCTest 和 smoke test 覆盖新增摘要行为。
- 本地轻量检查通过。
- Agent B 确认在 `main` 上提交并推送到 `origin/main`。
- Agent C 下载最新 run 的 artifact，核对 manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与 `origin/main` 一致，并确认静态检查、规则 smoke 和 Xcode build-for-testing 通过。

## 风险和禁止项

- 禁止通过解析 `battleLog` 文本生成战斗结果。
- 禁止改变既有伤害、反击、士气、经验、机动追击和胜负判定规则。
- 禁止把普通攻击摘要写成只能 UI 展示、无法测试的状态。
- 禁止让预览、聚焦、快速定位或据点/敌军列表点击产生战斗结果。
- 禁止扩大到战斗动画、完整战术命令摘要或 AI 重写。
- 禁止创建长期分支、PR 或推送非 `main`。
- 禁止在没有真实云端 artifact 核对时宣称 Agent C 验收通过。
