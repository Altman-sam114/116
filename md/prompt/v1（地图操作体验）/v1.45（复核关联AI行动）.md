# v1.45 复核关联 AI 行动

## 版本分配依据

`md/prompt/v1（地图操作体验）` 已完成到 v1.44。v1.44 细分了据点防守执行后的敌方回合复核，本轮继续补齐“复核结果对应哪些真实 AI 行动”的可读闭环，编号递增为 v1.45。

## 背景

当前反制复核能说明据点是否守住、守点位置是否到位、威胁源是否仍压迫据点，但玩家仍需要到 AI 行动时间线里自行查找相关事件。项目已有 `AIPhaseSummary.timeline`、AI 复盘定位和地图标记，本轮应把复核卡与真实 AI 时间线轻量关联起来，提高 EasyTech 风格战后复盘的可读性。

## 目标

- 在 `EnemyThreatCountermeasureFollowUpSummary` 中新增轻量“关联 AI 行动”字段，只保存真实 AI timeline 的 order、kind、summary、坐标和关系类型。
- 据点防守复核从 `AIPhaseSummary.timeline` 中按真实事件筛选关联行动，优先匹配目标据点坐标、防守单位、原威胁源和受威胁目标。
- UI 在敌方回合复核卡中展示最多 3 条关联 AI 行动，并用按钮复用 `focusAIPhaseTimelineEvent(order:)` 只读定位。
- 测试覆盖关联事件来自最新 AI timeline、order 合法、kind 合法、点击定位只读、不相关时不伪造事件。
- 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不做精确因果归因，不声称某个 AI 行动必然导致复核结果。
- 不模拟“如果没防守会怎样”，不改 AI 行为、排序、移动、攻击、战术命令或时间线记录时机。
- 不新增 `@Published` 状态，不让 UI 反查 timeline 或自行计算归因。
- 不引入第三方库、网络依赖或新资产。

## 架构依据

- `GameModels.swift` 定义 `EnemyThreatCountermeasureFollowUpAIEvent` 和 relation enum，挂到 follow-up summary。
- `GameState.swift` 在 `countermeasureFollowUpSummary(from:aiSummary:)` 中使用同一个 `aiSummary.timeline` 生成关联事件，不调用任何真实执行或预览方法。
- `ContentView.swift` 只展示 `summary.relatedAIEvents`，按钮只转发 event order 到既有 AI 复盘定位入口。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 复用现有 `enemyThreatFollowUpScenario(axisSpent:)` 与 objectiveDefense 成功路径。

## 实现步骤

1. 新增 `EnemyThreatCountermeasureFollowUpAIEventRelation`，包含 `threatenedObjective`、`actingUnit`、`threatEnemy`、`targetUnit`。
2. 新增 `EnemyThreatCountermeasureFollowUpAIEvent`，包含 `order`、`kind`、`title`、`summary`、`coordinate`、`relation`，并从 relation 派生短标题。
3. 扩展 `EnemyThreatCountermeasureFollowUpSummary`，新增 `relatedAIEvents: [EnemyThreatCountermeasureFollowUpAIEvent]`。
4. 在 `GameState` 中新增私有 helper，从真实 `AIPhaseSummary.timeline` 按以下顺序筛选：
   - `event.to == threatTargetCoordinate || event.from == threatTargetCoordinate`。
   - `event.actorUnitID == actingUnitID || event.targetUnitID == actingUnitID`。
   - `event.actorUnitID == threatEnemyUnitID || event.targetUnitID == threatEnemyUnitID`。
   - `targetUnitID` 存在时匹配 actor/target。
   - 去重保留 timeline 原 order，最多保留 3 条。
5. UI 在复核卡守点 detail 后显示“关联AI行动”，每条为 Button + Label/短码/摘要，点击调用 `game.focusAIPhaseTimelineEvent(order:)`。
6. 测试新增或扩展 objectiveDefense follow-up 断言：事件 order 存在于 `latestAIPhaseSummary.timeline`、kind 属于合法集合、关系类型非空、坐标能对应 `event.to ?? event.from`，并验证点击关联事件只定位不改变单位、据点、AI summary 和 follow-up。
7. smoke test 追加最小结构断言：存在时 order 合法、kind 合法；成功守点仍没有敌方占领目标据点事件。
8. 更新文档和日志。

## 验收标准

- 据点防守复核能展示来自同一敌方回合 AI timeline 的最多 3 条关联行动。
- 关联行动只来自真实 `AIPhaseSummary.timeline`，不伪造、不模拟、不改变 AI。
- 点击关联行动只定位 AI 复盘事件，不改变单位 HP/位置、据点归属、指令点、AI summary、follow-up summary 或战报。
- 本地轻量检查通过，云端最新 artifact 与 `origin/main` 最新 commit 完全一致。

## 风险

- 当前只能称为“关联 AI 行动”，不是精确因果归因；文案和文档必须避免过度承诺。
- 现有小场景中的具体 AI 行动可能随规则变化而变化，测试应断结构和合法性，少断中文摘要。
- 如果没有相关事件，字段应为空；不得用 AI 总览或 timeline 第一条冒充归因。

## 禁止项

- 禁止改变 AI、反制建议排序、守点 score、移动、攻击、战术命令或时间线记录时机。
- 禁止绕过 `GameState` 在 View 中实现关联筛选。
- 禁止跳过 Agent C artifact 验收或用旧 run 伪装通过。
