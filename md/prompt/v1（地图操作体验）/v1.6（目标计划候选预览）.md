# v1.6 目标计划候选预览

## 版本号

v1.6

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.5，本轮继续围绕地图操作体验递增为 v1.6。
- v1.5 已完成据点推进计划摘要，但计划列表仍是静态信息；本轮把候选计划变成可点选预览，让玩家比较第二/第三目标路线和风险。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类大地图、多据点、单位移动、攻击判定和目标推进体验。当前 `GameState` 已能生成最多 3 条 `ObjectiveAdvancePreview`，首项与 OBJ 快捷按钮一致，并在侧栏显示目标名、归属、路线消耗、距离变化和终点火力风险。

当前不足是侧栏计划不能交互，玩家只能通过 OBJ 快捷键预览首选目标，无法把第二/第三候选计划投射到地图路线、终点和最终目标标记。v1.6 要让目标计划行可点选，但实际移动仍必须通过既有右键/执行按钮链路进入 `GameState.executeFocusedCommand()`。

## 目标

1. `GameState` 新增规则入口，允许根据某条 `ObjectiveAdvancePreview` 或目标坐标聚焦对应目标计划。
2. 点选任一目标计划后，地图显示该计划路线、路线步序、目的格、终点火力风险和最终目标标记。
3. 点选计划只预览，不消耗移动/攻击，不改变单位位置，不生成据点占领摘要。
4. 第一条计划仍与 OBJ 快捷按钮一致；第二/第三条可独立聚焦并生成可执行 MOVE 预览。
5. 侧栏目标计划行使用 SwiftUI `Button`，保留可访问文本和当前聚焦态。
6. 补充 XCTest 和规则 smoke test，覆盖点选第二/第三计划、远距计划、直达计划、非法/过期计划不执行、以及执行仍通过既有 MOVE 链路。
7. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不改变 OBJ 目标排序算法。
- 不改变移动、控制区、攻击、占领奖励、胜负、AI 或战术命令规则。
- 不新增“一键直接执行第二/第三目标”的捷径；点选计划只设置预览，执行仍走当前右键/执行按钮路径。
- 不新增第三方库、网络依赖、真实美术或复杂资产。
- 不把目标计划筛选、路径或火力风险计算写进 `ContentView`。

## 当前架构依据

- `GameModels.swift` 已有 `ObjectiveAdvancePreview`、`MovementRoute`、`PostMoveFireExposurePreview` 等结构化预览模型。
- `GameState.swift` 是核心规则状态机，已有私有 `ObjectiveAdvancePlan`、`objectiveAdvancePlans(for:)`、`objectiveAdvancePreviews(for:limit:)`、`focusNearestObjectiveTarget()` 和 `guidedObjectiveCoordinate`。
- `ContentView.swift` 只负责渲染目标计划和转发点击意图；不能自行计算路线、排序或风险。
- 地图路线展示已经消费 `focusedMovementRoute`、`focusedRouteStepPreviews`、`focusedFireExposurePreview` 和 `guidedObjectiveCoordinate`；本轮应复用这些链路。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已覆盖 v1.5 的目标计划摘要行为，本轮应扩展到交互聚焦行为。

## 实现步骤

1. 在 `GameState.swift` 新增公开方法，建议命名：
   - `func focusObjectiveAdvancePreview(_ preview: ObjectiveAdvancePreview)`
   - 或 `func focusObjectiveAdvanceTarget(coordinate: HexCoordinate)`
2. 该方法必须：
   - 要求 `winner == nil`。
   - 要求存在当前 `selectedUnit`，否则按现有模式选择下一支可行动单位或返回提示。
   - 基于当前 `selectedUnit` 重新从 `objectiveAdvancePlans(for:)` 查找同一目标坐标，避免使用 UI 持有的过期路线直接写状态。
   - 若目标不存在、已归属己方、被单位占据或当前单位不能移动，则提示不可推进，不消耗行动。
   - 若计划直达，设置 `focusedCoordinate = objective.coordinate`，`guidedObjectiveCoordinate = objective.coordinate`。
   - 若计划远距，设置 `focusedCoordinate = plan.route.destination`，`guidedObjectiveCoordinate = objective.coordinate`。
   - 写入与 OBJ 快捷动作一致风格的 `message`，但文案要能说明这是第 N/目标计划预览更好；不得执行移动。
3. 如有明显重复，可让 `focusNearestObjectiveTarget()` 调用同一私有聚焦 helper，以保证 OBJ 首项和计划行聚焦行为一致。
4. 在 `ContentView.swift`：
   - 将 `ObjectiveAdvancePlanRow` 改为 `Button` 或让 `ObjectiveAdvancePlanPanel` 包一层 Button 调用 `game.focusObjectiveAdvancePreview(preview)`。
   - 第一条保留 `OBJ` 标记；当前聚焦的目标/路线应有视觉状态，例如描边或 “当前” 短码。
   - 按钮必须有文本 label 或 `.accessibilityLabel`，不要使用裸 `onTapGesture`。
5. 地图不应新增独立规则。点选计划后，现有 `focusedMovementRoute`、`focusedRouteStepPreviews`、`focusedFireExposurePreview` 和 `guidedObjectiveCoordinate` 应自然让地图显示路线和最终目标。
6. 补充 XCTest：
   - 多目标场景中点选第二条计划，`focusedCoordinate` 等于该计划 route destination，`guidedObjectiveCoordinate` 等于该目标坐标，`focusedCommandPreview == .move(...)`，单位位置不变。
   - 远距目标计划点选后聚焦推进格并保留最终目标。
   - 点选计划后执行 `executeFocusedCommand()` 才移动；直达目标执行后可占领，远距目标执行后不生成占领摘要。
   - 单位已不可移动、目标已归属己方或目标被占据时，点选不会设置错误预览或消耗行动。
7. 补充 `RulesSmokeTest.swift` 对多目标第二候选预览和执行链路的关键断言。
8. 更新文档：
   - `WW2Tactics/README.md`：目标计划可点选预览。
   - `md/flow/flow.md`：OBJ 目标推进增加候选聚焦入口与“只预览、不执行”边界。
   - `md/flow/flowchart.md`：地图命令流加入目标计划行点击 -> `GameState` 聚焦计划。
   - `md/test/test.md`：若当前基线描述需要扩展，加入“目标计划候选预览”覆盖点。
   - `update_log.md`：追加 v1.6 记录。

## 验收标准

- 侧栏目标计划行可点选，且点击后地图投射该候选路线和最终目标标记。
- 点选计划不会消耗行动、不会移动单位、不会生成战斗/战术/占领结果。
- 第一条计划仍与 OBJ 快捷按钮一致；第二/第三条计划可独立聚焦。
- 执行仍通过现有 `executeFocusedCommand()` / 右键 MOVE 链路完成。
- XCTest 和 smoke test 覆盖候选聚焦和执行边界。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止在 `ContentView` 中直接修改 `focusedCoordinate` 或 `guidedObjectiveCoordinate`。
- 禁止 UI 使用过期 `ObjectiveAdvancePreview.route` 直接执行移动。
- 禁止改变 OBJ 默认排序或快捷按钮执行语义。
- 禁止把点选目标计划做成直接移动或一键占领。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
