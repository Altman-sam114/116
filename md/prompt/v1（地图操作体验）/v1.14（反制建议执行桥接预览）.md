# v1.14（反制建议执行桥接预览）

## 版本号

v1.14

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新版本为 v1.13（反制建议地图聚焦标记）。本轮继续沿 v1 地图操作体验小步迭代，不开启新大版本，因此递增为 v1.14。

## 背景

v1.11 已生成敌方意图反制建议，v1.12 支持点选建议聚焦，v1.13 已把当前建议投射为 ACT/SRC/CTR/TGT 地图标记。当前玩家能看懂“哪支部队、哪个威胁、去哪或打谁”，但还需要自己判断下一步应使用地图 ATK、MOVE 还是侧栏整补入口。

为了更接近 EasyTech 风格的参谋提示到实际操作链路，本轮要给当前已聚焦反制建议增加“执行入口预览”：明确告诉玩家这条建议应该通过哪个既有入口完成，但仍不让反制建议行本身一键执行。

## 目标

1. 新增结构化只读执行桥接预览，能表达当前聚焦反制建议应使用的既有入口：
   - 抢先打击 -> 现有地图 ATK / 执行按钮。
   - 撤出危险区 -> 现有地图 MOVE / 执行按钮。
   - 据点防守 -> 现有地图 MOVE / 执行按钮。
   - 整补支撑 -> 现有单位详情整补按钮。
2. `GameState` 提供当前聚焦反制建议的执行桥接派生属性，例如 `focusedEnemyThreatCountermeasureExecutionPreview`。
3. 预览必须复用当前状态重新校验单位、敌军、路线、目标坐标和整补条件；过期或不匹配时返回不可用说明或空预览。
4. `ContentView` 在反制建议面板中展示当前聚焦建议的执行入口提示，例如“下一步：点地图 ATK / 点执行按钮 / 使用整补按钮”。
5. 桥接预览只提示，不执行命令，不新增一键执行按钮，不消耗行动，不写战报或结果摘要。

## 非目标

- 不让反制建议行直接执行移动、攻击或整补。
- 不新增 confirmation dialog 或自动命令队列。
- 不改变 `executeFocusedCommand()`、`move`、`attack`、`reinforce` 的真实执行规则。
- 不改变敌方意图生成、反制建议生成、排序或 limit 策略。
- 不改变 AI、战斗数值、移动规则、整补成本、胜负条件或回合流程。
- 不引入第三方库、网络依赖、真实美术资产、动画或复杂资产管线。

## 当前架构依据

- `GameModels.swift` 定义反制建议、地图命令预览和 v1.13 反制地图标记模型。若执行桥接预览被 UI 和测试共同使用，应放在 `GameModels.swift`。
- `GameState.swift` 是核心规则状态机，已有 `focusedEnemyThreatCountermeasurePreview`、`focusedEnemyThreatCountermeasureMapMarkers`、`focusEnemyThreatCountermeasure(_:)`、`mapCommandPreview(for:)` 和 `canReinforce(_:)`。
- `ContentView.swift` 中 `InlineMapCommandPreview` 已负责展示地图 MOVE/ATK 执行入口；`EnemyThreatCountermeasurePanel` 已负责反制建议列表；单位详情中已有整补按钮。
- `ContentView` 只能展示 `GameState` 派生字段和转发用户意图，不能在 View 中重新计算路线、攻击或整补有效性。

## 实现步骤

1. 在 `GameModels.swift` 新增轻量模型：
   - 执行入口枚举，例如 `EnemyThreatCountermeasureExecutionKind`，包含 `.attack`、`.move`、`.reinforce`、`.unavailable`。
   - 执行桥接预览结构，例如 `EnemyThreatCountermeasureExecutionPreview`，包含 kind、countermeasureKind、actionTitle、entryTitle、coordinate、unitName、targetName、isExecutable、reason。
2. 在 `GameState` 新增只读派生属性 `focusedEnemyThreatCountermeasureExecutionPreview`：
   - 只基于当前 `focusedEnemyThreatCountermeasurePreview` 生成。
   - 必须先确认 `isEnemyThreatCountermeasureFocused(preview)`。
   - 抢先打击要确认威胁敌军存在、执行单位可攻击、`combatPreview` 可用，并且当前 `mapCommandPreview(for: enemy.position)` 是可执行攻击。
   - 撤出危险区/据点防守要确认目的格存在、路线仍可用，并且当前 `mapCommandPreview(for: destination)` 是可执行移动。
   - 整补支撑要确认执行单位仍可整补，返回整补入口预览。
   - 不匹配或过期时返回不可用预览或 nil；不要调用真实 `move`、`attack`、`reinforce`、`executeFocusedCommand`。
3. `ContentView` 更新 `EnemyThreatCountermeasurePanel`：
   - 读取 `game.focusedEnemyThreatCountermeasureExecutionPreview`。
   - 仅当存在当前聚焦建议时，在列表下方显示一条紧凑的“下一步”提示。
   - 文案必须指向现有入口，不做新的执行按钮。可使用 `Label` 和短码，例如 `ATK`、`MOVE`、`整补`。
   - 提供无障碍 label/hint，说明该区域只是提示，不执行命令。
4. 扩展 XCTest：
   - 抢先打击聚焦后，桥接预览为 `.attack`、可执行，坐标等于敌军坐标。
   - 撤出危险区聚焦后，桥接预览为 `.move`、可执行，坐标等于撤退目的格。
   - 据点防守聚焦后，桥接预览为 `.move`、可执行，坐标等于防守目的格。
   - 整补支撑聚焦后，桥接预览为 `.reinforce`、可执行，单位为受威胁单位。
   - 普通聚焦、过期建议或不满足条件时不保留可执行桥接预览。
   - 查询桥接预览无副作用。
5. 扩展 `RulesSmokeTest.swift` 中反制建议聚焦段，加入同类轻量断言。
6. 同步更新：
   - `WW2Tactics/README.md`：说明反制建议点选后会提示下一步入口。
   - `md/flow/flow.md`：记录执行桥接预览仍是只读状态派生。
   - `md/flow/flowchart.md`：在反制建议点选后补充执行入口提示节点。
   - `md/test/test.md`：更新当前基线覆盖项。
   - `update_log.md`：新增 v1.14 记录。

## 验收标准

1. 点选反制建议后，侧栏能清楚显示下一步应使用 ATK、MOVE 或整补入口。
2. 桥接提示来自 `GameState` 结构化派生状态，`ContentView` 不自行判断规则有效性。
3. 桥接提示不执行任何命令，不消耗行动，不改变 HP、位置、行动状态、据点归属、指令点、战报、胜负或 `latest*Result`。
4. 过期建议、普通聚焦、执行行动、重开或切战役后不会残留可执行桥接预览。
5. XCTest、规则 smoke、SwiftUI typecheck 和测试源码 typecheck 通过。
6. `main` 已推送，GitHub Actions 最新 run 成功。
7. Agent C 已下载最新 artifact，核对 manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与 `origin/main` 一致，并检查 failure summary、JUnit、rules smoke、xcodebuild log 和 `.xcresult`。

## 风险和禁止项

- 禁止把反制建议行改成直接执行按钮。
- 禁止让 `ContentView` 根据建议类型自行计算是否能 MOVE、ATK 或整补。
- 禁止新增绕过 `GameState` 的执行路径。
- 禁止调用真实 `move`、`attack`、`reinforce` 或 `executeFocusedCommand()` 来生成预览。
- 禁止破坏 `InlineMapCommandPreview`、地图右键执行和单位详情整补按钮的既有行为。
- 禁止扩大范围到 AI、数值、战役数据、真实资产或动画。
- 禁止跳过 `main` 直推和 Agent C artifact 验收。
