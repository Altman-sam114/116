# v1.27 AI复盘自动播放控制

## 版本号

v1.27

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.26（AI复盘连续查看控制）。`origin/main` 最新验收 commit `30556da9bbec7240cb3d736175ba7b06dbff17fe` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.26-main-30556da-run28766809710-attempt1` 核对。v1.26 已支持 AI 时间线上一条/下一条连续查看；下一步应让玩家能以慢/中/快三档节奏自动浏览敌方行动，进一步接近 EasyTech 风格敌方回合复盘。本轮递增为 v1.27。

## 背景

当前 AI 复盘需要玩家手动点选或点击上一条/下一条。大地图敌方回合行动多时，手动连续点按仍有负担。本轮加入只读的播放/暂停和节奏控制：播放状态由 `GameState` 管理，SwiftUI 只显示控件并按当前节奏触发“前进一步”；每一步仍复用现有 AI 时间线定位能力，不执行任何真实战斗、移动、部署、整补或 AI 方法。

## 目标

1. 在 `GameModels` / `GameState` 增加 AI 复盘播放状态：
   - 新增三档复盘节奏模型：慢速、标准、快速，包含标题、短标题和秒级间隔。
   - `GameState` 新增只读发布状态 `isAIPhaseTimelinePlaybackActive` 和当前 `aiPhaseTimelinePlaybackPace`。
   - 新增 `canPlayAIPhaseTimeline` 或等价只读属性，供 UI 禁用播放按钮。
   - 新增 `toggleAIPhaseTimelinePlayback()`、`pauseAIPhaseTimelinePlayback()`、`setAIPhaseTimelinePlaybackPace(_:)`、`advanceAIPhaseTimelinePlayback()` 或等价入口。
2. 播放语义：
   - 无当前 order 时，播放的第一次 advance 聚焦第一条事件。
   - 有当前 order 时，每次 advance 聚焦下一条事件。
   - 聚焦到最后一条后自动暂停，保留最后一条的焦点、order 和 marker 强调。
   - 当前已在最后一条时点击播放，不启动播放，只发布明确提示。
   - 无 summary、timeline 为空、事件坐标缺失或坐标不在当前地图时，播放不能改变旧焦点/order；需要暂停并发布明确提示。
   - 下一次 AI 回合开始、重开、切换战役或清空 AI summary 时必须暂停播放，避免计时器继续驱动过期复盘。
3. SwiftUI 控制：
   - 在 AI 行动时间线标题行加入播放/暂停 icon 按钮和节奏选择控件。
   - 播放按钮使用有语义 `Label`、`accessibilityLabel` / `accessibilityHint`，视觉可保持 icon-only。
   - 节奏选择使用菜单或等价选项控件，选项来自 `GameState` / 模型定义，不在 View 硬编码行为。
   - SwiftUI 可以使用计时器按当前节奏调用 `advanceAIPhaseTimelinePlayback()`，但不能自行计算下一个 order 或改变核心状态。
4. 测试：
   - XCTest 覆盖播放从无 order 开始、连续 advance、到末尾自动暂停。
   - 覆盖末尾点击播放不启动且保留状态。
   - 覆盖暂停和节奏切换只改变播放状态/节奏，不改变单位、指令点、summary 和 marker。
   - 覆盖无 summary 时播放/advance 不改变状态。
   - 规则 smoke test 覆盖同一核心链路。
5. 文档同步：
   - README、flow、flowchart、test、prompt README 和 update_log 必须同步 v1.27 行为和验证结果。

## 非目标

- 不改变 AI 决策、行动顺序、时间线事件生成、地图复盘 marker 生成规则或 `latestAIPhaseMapMarkers` 的纯派生语义。
- 不新增逐帧动画、镜头动画、时间轴滑块、历史 AI 回合列表、音效、真实美术资产或第三方库。
- 不让 SwiftUI 自行计算下一个有效 order；播放推进和边界处理必须在 `GameState`。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励、胜负或反制建议语义。

## 当前架构依据

- `GameState.focusAIPhaseTimelineEvent(order:)` 是 AI 时间线定位入口，已负责查找事件、校验坐标、更新 `focusedCoordinate`、`focusedAIPhaseTimelineEventOrder` 和消息。
- `focusNextAIPhaseTimelineEvent()` 已封装相邻导航；播放推进应复用相同目标选择语义或复用定位 helper，避免 View 计算 order。
- `focusedAIPhaseMapMarkers` 已从 `latestAIPhaseMapMarkers` 按当前 order 过滤派生，播放切换 order 后会自动更新 marker 强调。
- `AIPhaseSummaryView` 已持有 `@EnvironmentObject GameState`，适合显示播放按钮和节奏菜单；核心状态仍必须在 `GameState`。

## 验收标准

1. AI 时间线有事件且无当前 order 时，点击播放进入播放状态，第一次 advance 聚焦第一条事件。
2. 播放过程中每次 advance 聚焦下一条事件，并保持 `focusedAIPhaseMapMarkers` 等于最新 marker 对当前 order 的过滤结果。
3. 播放到最后一条后自动暂停，最后一条焦点、order 和消息保留。
4. 当前已在最后一条时点击播放不启动播放，并发布边界提示。
5. 暂停和节奏切换不改变单位 HP/位置、行动状态、据点归属、指令点、AI summary、timeline 或 `latestAIPhaseMapMarkers`。
6. 无 summary、空 timeline、无定位坐标或坐标不在当前地图时，不改变旧焦点/order，并确保播放状态为暂停。
7. SwiftUI 按钮/菜单有可访问名称和 hint，禁用状态来自 `GameState`。
8. View 不计算下一个 order，不绕过 `GameState`。
9. 本地轻量检查通过。
10. Agent B 在 `main` 提交并 `git push origin main`。
11. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
12. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：SwiftUI Timer 若自行计算 order，会与 `GameState` 真实 timeline 失步；必须只调用播放 advance 入口。
- 风险：播放状态跨 AI 回合残留会驱动过期复盘；开始新 AI phase、重开和切战役必须暂停。
- 风险：自动播放不能变成真实动画执行或命令执行；每步只做只读定位和 marker 强调。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘推进状态。
- 禁止新增真实动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
