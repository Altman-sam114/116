# v1.8 整补与增援结果摘要

## 版本号

v1.8

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.7，本轮继续围绕地图操作体验递增为 v1.8。
- v1.2-v1.4 已把普通攻击、战术命令和据点占领做成结构化结果摘要；整补和部署仍只靠 message / battleLog。v1.8 补齐后勤类操作反馈，让指令点换战力的结果更清楚。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类二战战棋体验。当前部署增援和整补受损单位已经消耗指令点、改单位状态并写战报，但侧栏没有像战斗、战术命令、占领一样的结果卡，玩家不容易快速确认本次后勤动作的消耗、恢复量、部署地点和新单位状态。

本轮要为部署和整补新增可测试的结构化结果摘要，并在侧栏结果区展示。规则仍以 `GameState` 为唯一落点，`ContentView` 只读状态渲染。

## 目标

1. 在 `GameModels.swift` 新增部署结果摘要和整补结果摘要模型，记录必要字段。
2. `GameState` 新增 `latestDeploymentResult` 和 `latestReinforcementResult`。
3. `deploy(kind:at:)` 成功时写入部署结果摘要，包含来源据点、坐标、新单位名、兵种、阵营、指令点消耗、部署后剩余指令点。
4. `reinforce(unitID:)` 成功时写入整补结果摘要，包含单位名、兵种、阵营、坐标、HP 前后、恢复量、指令点消耗、整补后剩余指令点。
5. 部署/整补结果与普通攻击、战术命令、据点占领结果互斥展示；成功的部署或整补应清理其他旧结果，攻击/战术/占领也应清理后勤结果。
6. 失败部署/整补不得生成或保留虚假后勤结果。
7. AI 后勤动作允许通过同一路径写入摘要；这能让玩家在敌方回合后看到轴心国最近一次整补或部署。
8. `ContentView` 在现有侧栏结果区新增部署结果卡和整补结果卡，展示指令点、HP 恢复或新单位、地点和阵营。
9. 补充 XCTest 和规则 smoke test，覆盖部署、整补、失败不写摘要、互斥清理、AI 后勤摘要。
10. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不改变部署地点算法、部署成本、整补成本、整补恢复量或 AI 后勤优先级。
- 不新增生产队列、回合后勤面板、动画、音效、真实美术或第三方库。
- 不改变战斗、战术命令、占领、胜负或移动规则。
- 不把后勤规则或结果字段计算写进 `ContentView`。
- 不要求本机完整 simulator 测试；仍按本地轻量检查 + 云端 artifact 验收流程。

## 当前架构依据

- `GameModels.swift` 已有 `CombatResultSummary`、`TacticalCommandResultSummary` 和 `ObjectiveCaptureResultSummary`，本轮应沿用同类 struct 模式。
- `GameState.swift` 已有 `latestCombatResult`、`latestTacticalCommandResult`、`latestObjectiveCaptureResult` 和对应写入/互斥清理点。
- `deploy(kind:at:)` 和私有 `reinforce(unitID:)` 是部署与整补的唯一成功落点；AI 也复用这些路径。
- `ContentView.swift` 的 `InspectorPanel` 已在战报前互斥展示结果卡，本轮应在同一区域加入后勤结果卡。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已有部署与整补行为测试，可扩展摘要断言。

## 实现步骤

1. 在 `GameModels.swift` 新增：
   - `DeploymentResultSummary`
   - `ReinforcementResultSummary`
2. 建议字段：
   - 部署：`id`、`sourceObjectiveName`、`coordinate`、`unitName`、`unitKind`、`faction`、`commandCost`、`commandPointsAfterDeployment`。
   - 整补：`id`、`unitName`、`unitKind`、`faction`、`coordinate`、`startingHP`、`endingHP`、`recoveredHP`、`commandCost`、`commandPointsAfterReinforcement`。
3. 在 `GameState.swift` 新增 `@Published private(set)` 状态：
   - `latestDeploymentResult`
   - `latestReinforcementResult`
4. 新增一个私有清理 helper，例如 `clearLatestResultSummaries()` 或 `clearLogisticsResults()`，避免多个结果卡同时显示。
5. 成功部署时：
   - 保留现有消耗指令点、新增单位、选择新单位、聚焦坐标、战报和胜负检查。
   - 写入 `latestDeploymentResult`。
   - 清理 `latestCombatResult`、`latestTacticalCommandResult`、`latestObjectiveCaptureResult`、`latestReinforcementResult`。
6. 成功整补时：
   - 在修改 HP 前记录 starting HP，修改后记录 ending HP。
   - 写入 `latestReinforcementResult`。
   - 清理 `latestCombatResult`、`latestTacticalCommandResult`、`latestObjectiveCaptureResult`、`latestDeploymentResult`。
7. 失败部署/整补不应清理已有真实结果，除非项目已有明显相反模式；重点是不得生成虚假结果。
8. 普通攻击、战术命令、据点占领、重开/切战役应清理后勤结果，避免旧结果残留。
9. 在 `ContentView.swift`：
   - `InspectorPanel` 的结果区加入 `DeploymentResultSummaryView` 和 `ReinforcementResultSummaryView`。
   - 结果卡使用现有 `CombatResultMetric` 等小组件，保持紧凑。
   - 部署卡显示来源据点、单位、兵种、坐标、指令点消耗、剩余指令点。
   - 整补卡显示单位、坐标、HP 前后、恢复量、指令点消耗、剩余指令点。
10. 补充 XCTest：
   - 成功部署写入部署摘要、字段正确、旧战斗/战术/占领/整补摘要被清理。
   - 成功整补写入整补摘要、HP 前后和恢复量正确、旧部署摘要被清理。
   - 失败部署和失败整补不写入虚假摘要。
   - AI 整补或部署后能通过同一路径写入对应摘要。
11. 补充 `RulesSmokeTest.swift` 对部署和整补摘要字段的关键断言。
12. 更新文档：
   - `WW2Tactics/README.md`：部署/整补结果摘要。
   - `md/flow/flow.md`：后勤结果状态与互斥展示。
   - `md/flow/flowchart.md`：规则状态图或执行流加入后勤结果摘要。
   - `md/test/test.md`：当前基线加入部署/整补结果摘要覆盖点。
   - `update_log.md`：追加 v1.8 记录。

## 云端流程要求

- 当前固定使用 `main` 作为唯一上传、提交、推送和云端验证分支。
- 本轮不使用 `smalldata_test`、`develop`、`codeb/...` 或 PR 流程。
- Agent B 开始前必须执行或核对：
  ```sh
  git fetch origin
  git switch main
  git pull --ff-only origin main
  git status --short --branch
  ```
- Agent B 完成后先跑本地轻量检查，再用 `v1.8: 简要说明` 提交并 `git push origin main`。
- Agent B push 前必须确认当前分支是 `main`，目标远端是 `origin/main`，提交范围只包含本轮相关文件。
- GitHub Actions 必须生成未加密 CI 结果包，至少包含 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、主构建日志和项目专属结果文件。
- Agent C 必须下载本轮最新 `origin/main` commit 对应的 artifact 到 `/private/tmp/ww2tactics-c-review-<run_id>/`。
- Agent C 必须核对 manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与 `origin/main` 最新状态一致。
- 若云端失败，Agent C 写退回清单；Agent B 在 `main` 上追加修复 commit 并重新 push。

## 验收标准

- 成功部署后侧栏显示部署结果摘要，字段与实际新单位、指令点消耗和部署地点一致。
- 成功整补后侧栏显示整补结果摘要，字段与 HP 前后、恢复量和指令点消耗一致。
- 后勤结果与战斗、战术命令、占领结果互斥，不显示旧结果。
- 失败部署/整补不生成虚假结果。
- AI 后勤动作通过同一路径写入摘要。
- XCTest 和 smoke test 覆盖部署、整补、失败、互斥和 AI 后勤摘要。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止在 `ContentView` 中计算后勤结果字段。
- 禁止改变部署/整补成本、恢复量或 AI 选择策略。
- 禁止让失败操作生成或保留误导性后勤摘要。
- 禁止让多个结果卡同时显示。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
