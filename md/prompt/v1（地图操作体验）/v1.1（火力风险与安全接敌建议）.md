# v1.1 / 火力风险与安全接敌建议

## 版本号

v1.1

## 版本分配依据

当前 `v1（地图操作体验）` 已完成 v1.0 路线与战斗预判强化，并已通过 `origin/main` 云端 artifact 验收。人工总目标是持续逼近 EasyTech 风格二战回合制战棋体验，本轮继续沿地图操作体验小版本递增为 v1.1。

## 背景

现有 v1.0 已能在 MOVE/POS 聚焦时显示路线步序、移动消耗、控制区额外消耗、敌火来源和移动后攻击目标。但 THR/路线风险目前主要说明“谁覆盖了该格”，还没有把敌火覆盖转换为玩家可判断的风险等级、预计承伤和 HP 后果。EasyTech 类战棋的关键体验之一是让玩家在推进、接敌、抢据点前看清“这一步是否会把部队送进危险火力网”。

## 目标

- 在 `GameState` 中新增可测试的移动后敌火暴露预览。
- 聚焦合法 MOVE 目的地或 POS 接敌位置时，显示敌火来源、预计可能承受的合计伤害、暴露后剩余 HP、是否可能被击毁和风险等级。
- POS 场景提供安全接敌候选：在可进入攻击位中找出风险更低的候选，但不改变默认 POS 执行目的地。
- 风险预览必须复用现有 `combatPreview(enemy, movedUnit)` 规则估算，不在 View 层计算核心规则。
- UI 在地图命令预览、侧栏命令预览和路线终点 accessibility 文案中展示风险摘要。
- 补充 XCTest 和规则 smoke test，覆盖风险来源、伤害、HP 后果、风险等级和不改变真实战役状态。

## 非目标

- 不引入反应射击、借机攻击、overwatch 或真实移动后自动受击。
- 不改变移动、攻击、AI、胜负、据点控制、补给、士气和战术命令结算。
- 不做纯视觉装饰、动画、真实美术、贴图或资产管线。
- 不新增战役、国家、生产队列或存档系统。
- 不把复杂规则写进 `ContentView`。
- 不引入第三方库或网络依赖。

## 当前架构依据

- `GameModels.swift` 定义 `RouteStepPreview`、`PostMoveAttackPreview`、`MapCommandPreview` 等规则层预览数据。
- `GameState.swift` 是规则状态机，现有 `focusedRouteStepPreviews`、`focusedPostMoveAttackPreviews`、`threateningEnemies(against:at:)` 和 `combatPreview(attacker:defender:)` 已提供本轮所需基础。
- `ContentView.swift` 只展示 `GameState` 输出，并转发地图输入，不应自行计算敌火伤害。
- `GameStateTests.swift` 已有 `postMoveAttackScenario()` 覆盖 MOVE/POS 聚焦，可复用补充风险断言。
- `RulesSmokeTest.swift` 已有 v1.0 路线与移动后攻击 smoke 段，可在同一段追加轻量风险检查。

## 实现步骤

1. 在 `GameModels.swift` 新增风险等级枚举和敌火暴露预览结构，例如：
   - `FireRiskLevel`：`none / low / medium / high / critical`。
   - `FireExposureSourcePreview`：敌火来源 ID、名称、兵种、距离、射程和潜在伤害。
   - `PostMoveFireExposurePreview`：目的地、当前 HP、暴露后预计 HP、合计潜在伤害、最高单次伤害、来源列表、风险等级、单源/合计击毁判断。
   - `SafeEngagementOption`：POS 候选路线、对应暴露预览、目标 ID 和目标名。
2. 在 `GameState.swift` 新增 `focusedThreatExposurePreview` 和 `threatExposurePreview(for:at:)`：
   - 对当前聚焦 MOVE 路线终点或 POS 推荐接敌点计算。
   - 对合法移动目的地创建临时 moved unit，位置改为目的地；若是移动后预览，应按真实 `move` 行为清理防御姿态和战术状态。
   - 用 `threateningEnemies(against:at:)` 找敌火来源，再对每个来源调用 `combatPreview(attacker: enemy, defender: movedUnit)` 得出预计伤害。
   - 汇总来源数量、合计预计伤害、HP 后果、是否可能被击毁和风险等级。
   - 该计算不得写回 `scenario.units`、`selectedUnitID`、`focusedCoordinate` 或战报。
3. 在 `GameState.swift` 新增 `safeEngagementOptions(for:against:)` 与 `focusedSafeEngagementOptions`：
   - 只从现有 `attackPositionRoutes(for:against:)` 派生候选。
   - 不改变 `focusedAttackPositionRoute` 的默认排序和 POS 执行目的地。
   - 排序规则：风险等级低优先，其次合计伤害低、最高单次伤害低、路线消耗低、步数少、坐标稳定排序。
4. 在 `ContentView.swift` 展示风险摘要：
   - MOVE/POS 预览详情中加入风险等级、预计承伤、剩余 HP 和主要来源。
   - POS 聚焦时显示推荐攻击位风险；若存在更安全攻击位，显示安全接敌建议坐标和风险结果。
   - 路线终点可显示简短风险标记，例如 `SAFE / LOW / MED / HIGH / CRIT`，不能只靠颜色。
   - accessibility 文案包含终点风险和预计伤害。
5. 在 `GameStateTests.swift` 增加测试：
   - 风险预览伤害等于从敌军对临时移动后单位调用 `combatPreview` 的结果。
   - 多个来源时能汇总伤害并排序。
   - 无敌火来源时为安全/无暴露风险。
   - 调用风险预览不改变单位位置、HP、行动状态或当前战役状态。
   - POS 聚焦时 `focusedThreatExposurePreview` 指向推荐接敌位置。
   - POS 安全接敌建议来自 `focusedAttackPositionRoutes`，排序稳定，并且不改变默认接敌执行结果。
6. 在 `RulesSmokeTest.swift` 追加 smoke 断言，覆盖 `focusedThreatExposurePreview`、风险等级、预计伤害、来源名称和安全建议不改变真实 MOVE/POS/ATK 链。
7. 同步更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 验收标准

- MOVE 目的地和 POS 推荐接敌点能显示敌火暴露风险摘要。
- 风险预览由 `GameState` 生成，`ContentView` 只展示，不执行规则判定。
- 预计伤害与 `combatPreview(enemy, movedUnit)` 一致。
- 风险预览不改变真实单位位置、HP、行动状态、战报、胜负或据点控制。
- POS 安全建议只展示更安全候选，不改变默认接敌位置和执行语义。
- XCTest 和 `RulesSmokeTest.swift` 覆盖核心风险预览行为。
- 本地轻量检查通过，并推送到 `origin/main` 触发 GitHub Actions。
- Agent C 下载最新 run 的 artifact，核对 `branch=main`、`commitSha`、`runId`、`runAttempt`、manifest、failure summary、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult` 后才能判定通过。

## 风险和禁止项

- 敌火暴露是“可能承受的风险估算”，不是敌军必定会攻击；UI 文案必须避免误导。
- 不要为全地图每个格子做昂贵伤害扫描，优先只计算当前聚焦路线/目的地。
- 不得在 `ContentView` 里复制 `combatPreview`、伤害、补给、士气或地形规则。
- 不得改变现有 MOVE/ATK/POS/NEXT/OBJ/THR 执行语义。
- 不得把旧 artifact、旧 output 或本地结果冒充本轮云端验收。
