# v1.46 据点防守压力列表

## 版本号

v1.46

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.45。
- 本轮继续服务 v1 地图操作体验，不开启新大版本。
- v1.45 已将反制复核关联真实 AI 行动，本轮在同一战线态势链上补充玩家回合的据点防守压力解释。

## 背景

当前战线态势卡已经统计敌方意图、可执行反制、受威胁据点名称和首要定位目标，但“受威胁据点”仍主要是名称摘要。玩家无法在态势卡内快速判断哪个据点压力最大、威胁来自哪支敌军、是直接夺点还是需要提前守点，以及当前是否已有对应反制入口。

## 目标

1. 新增只读模型，按受威胁据点聚合防守压力条目。
2. 每条压力至少说明据点名、坐标、当前归属、威胁来源数量、最高优先威胁、路线/夺点风险、可用反制建议和推荐入口。
3. `GameState.battlefieldSituationSummary` 从既有 `EnemyThreatIntentPreview` 与 `EnemyThreatCountermeasurePreview` 纯派生压力列表，不新增不可测试状态。
4. `ContentView` 在“战线态势”卡内展示最多数条据点压力，不在 View 中重新计算威胁、路线、反制或排序。
5. 增补 XCTest 和 rules smoke，覆盖字段、排序、只读无副作用和无威胁退化。

## 非目标

- 不改变 AI 决策、敌方意图生成、反制建议排序、score、移动、攻击、补给、士气、据点奖励、胜负条件或回合流程。
- 不新增一键执行、自动守点、自动选择反制、动画、音效、真实美术或第三方依赖。
- 不把规则计算写进 `ContentView`。
- 不修改 workflow、Xcode project 或长期分支策略。

## 架构依据

- `GameModels.swift` 定义结构化展示模型。
- `GameState.swift` 是核心规则状态机，负责从现有威胁和反制建议派生据点压力。
- `ContentView.swift` 只展示 `GameState`/`GameModels` 提供的字段，并把点击入口继续转发给既有 `GameState` 方法。
- `GameStateTests.swift` 与 `Tools/RulesSmokeTest.swift` 验证规则、只读派生和边界行为。

## 实现步骤

1. 在 `GameModels.swift` 新增 `BattlefieldSituationObjectivePressure`，包含：
   - `objectiveName`
   - `coordinate`
   - `owner`
   - `threatCount`
   - `primaryThreatTitle`
   - `primaryThreatDetail`
   - `countermeasureTitle`
   - `countermeasureDetail`
   - `actionHint`
   - 稳定 `id`
2. 给 `BattlefieldSituationSummary` 增加 `objectivePressures: [BattlefieldSituationObjectivePressure]`，并让 `id` 纳入压力摘要，避免 UI 旧值复用。
3. 在 `GameState.battlefieldSituationSummary` 中，基于同一批 `threats` 和 `countermeasures` 生成压力列表：
   - 只聚合 `.objectiveCapture` 威胁。
   - 按据点坐标分组。
   - 每组按现有威胁顺序选择首要威胁。
   - 匹配同坐标的 `.objectiveDefense` 反制建议，优先可执行项，再按既有反制排序结果中出现顺序选择。
   - 限制为紧凑列表，建议最多 3 条。
   - 生成 `BattlefieldSituationActionHint`，有可执行守点建议时给 MOVE；无反制建议时给 DEF 查看提示。
4. 在 `ContentView.BattlefieldSituationSummaryView` 中，在态势响应/复盘/首要定位和指标附近展示压力条目：
   - 使用紧凑行，展示据点、威胁来源数、首要威胁、推荐反制和入口短码。
   - 不新增直接执行按钮；若需要定位，继续依赖现有首要定位入口。
   - 保持小屏文字不溢出，使用 `lineLimit` 和 `minimumScaleFactor`。
5. 在 `GameStateTests.swift` 扩展既有战线态势测试：
   - 默认敌方意图场景应生成“后方油库”压力。
   - 压力字段应与 `.objectiveCapture` 威胁和 `.objectiveDefense` 反制一致。
   - 读取 summary 不改变 faction、command points、units、tiles、battle log、message 和 latest result。
   - 无据点威胁场景压力列表为空。
6. 在 `RulesSmokeTest.swift` 加入同等 smoke 断言。
7. 更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`（如测试基线文字变化）和 `update_log.md`。

## 验收标准

- 战线态势 summary 能输出稳定、只读的据点防守压力列表。
- 默认敌方威胁场景中，“后方油库”压力能展示首要威胁和守点建议入口。
- 无据点威胁时压力列表为空，态势卡不伪造压力。
- 读取压力列表不会改变单位、据点、指令点、消息、战报、结果摘要、AI summary 或胜负。
- UI 只展示模型字段，不在 View 计算威胁或反制。
