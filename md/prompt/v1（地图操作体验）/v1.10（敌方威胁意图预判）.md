# v1.10 敌方威胁意图预判

## 版本号

v1.10

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.9，本轮继续围绕地图操作体验和战前态势反馈递增为 v1.10。
- v1.9 已实现 AI 回合行动摘要，但玩家在自己回合开始时仍缺少“下一轮敌方可能攻击哪里、抢哪个点”的结构化预判。本轮用轻量、可测试的敌方威胁意图预判增强 EasyTech 风格大地图态势判断。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类二战回合制战棋体验。当前已有路线风险、移动后火力暴露、安全接敌候选、OBJ 目标计划和 AI 回合行动摘要，但这些信息主要围绕玩家当前选中单位或刚结束的敌方行动。玩家仍需要手动读地图判断敌方下一回合最可能造成的单位损失或据点丢失。

本轮要在不改变 AI 行动逻辑的前提下，为玩家回合提供一组“敌方威胁意图”预判：根据当前轴心单位的位置、射程、可移动范围和据点目标，列出最关键的 3 个威胁，帮助玩家决定撤退、整补、防守或抢先攻击。

## 目标

1. 在 `GameModels.swift` 新增敌方威胁意图模型，建议包含：
   - `EnemyThreatIntentKind`：至少区分直接攻击、机动接敌攻击、据点占领。
   - `EnemyThreatIntentPreview: Identifiable, Equatable`：记录敌方单位 ID/名称/兵种、目标坐标、目标单位或据点名称、当前距离、是否本格可攻击、是否本回合可接敌/占点、预计伤害、目标剩余 HP、是否预计击毁、路线目的地、路线消耗、评分或优先级。
2. 在 `GameState` 新增公开查询：
   - `enemyThreatIntentPreviews(against faction: Faction, limit: Int = 3) -> [EnemyThreatIntentPreview]`
   - 可选便捷属性：`alliedEnemyThreatIntentPreviews` 或 `visibleEnemyThreatIntentPreviews`，供 `ContentView` 使用。
3. 预判必须是只读计算，不改变 `activeFaction`、单位行动状态、地图归属、指令点、战报、消息或任何 `latest*Result`。
4. 预判口径：
   - 直接攻击：敌方单位当前位置已在己方单位射程内，可用 `combatPreview(attacker:defender:)` 计算预计伤害。
   - 机动接敌攻击：敌方单位若在下一回合重置行动后可移动到攻击位并攻击己方单位，显示最佳攻击位、路线消耗和预计伤害。
   - 据点占领：敌方单位若下一回合可移动到非己方、未被占据的据点，显示据点名、坐标、路线消耗和归属变化风险。
   - 本轮只要求轴心威胁盟军的预判，但 API 应尽量按 `against faction` 泛化。
5. 排序建议：预计击毁 > 对指挥官/高价值兵种伤害 > 据点占领 > 预计伤害 > 路线消耗更低 > 坐标/名称稳定排序。
6. `ContentView` 新增侧栏“敌方意图”面板，展示最多 3 条威胁：敌方单位、目标、类型、预计伤害/击毁/占点、路线目的格和消耗。
7. 地图对这些威胁的目标坐标显示简短标记，例如 `INT` 或 `WARN`；只作为提示，不拦截输入，不改变现有 MOVE/ATK/POS/CAP/THR 标记语义。
8. 补充 XCTest 和规则 smoke test，覆盖：
   - 直接攻击威胁能返回敌方、目标、伤害、击毁字段。
   - 机动接敌威胁能返回攻击位、路线消耗和预计伤害。
   - 据点占领威胁能返回目标据点、路线目的地和占点类型。
   - `limit` 生效且排序稳定。
   - 预判查询不会改变 `activeFaction`、单位位置/行动状态、战报或结果摘要。
9. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不改变 AI 优先级、移动路径、攻击目标、战术命令、后勤策略或胜负条件。
- 不引入真实迷雾、概率 AI、动画、音效、逐行动敌方计划时间线或复杂热力图。
- 不把预判计算写进 `ContentView`。
- 不把玩家本回合预览误写入 `latestAIPhaseSummary` 或 battleLog。
- 不新增第三方库、网络依赖或美术资产。

## 当前架构依据

- `GameModels.swift` 已有 `PostMoveAttackPreview`、`PostMoveFireExposurePreview`、`ObjectiveAdvancePreview`、`SafeEngagementOption`、`AIPhaseSummary` 等结构化模型，本轮应沿用同类 struct/enum 模式。
- `GameState.swift` 是移动、攻击、威胁覆盖、AI、目标推进和预览的唯一规则层；`ContentView` 只能读取 `GameState` 提供的结构化预览。
- 当前 `movementRoutes(for:)`、`attackCoverageTiles(for:)`、`attackPositionRoutes(for:against:)` 多数要求 `unit.faction == activeFaction`。敌方意图预判发生在盟军回合但需要评估轴心下一回合，所以应新增私有只读预测 helper，不能临时修改 `activeFaction`。
- `combatPreview(attacker:defender:)` 本身不要求 attacker 是 active faction，适合作为预估伤害的共用入口。
- `ContentView.HexMapView` 已集中计算地图标记集合，适合一次性计算威胁意图目标坐标再传给 `HexTileView`。

## 实现步骤

1. 在 `GameModels.swift` 新增 `EnemyThreatIntentKind` 和 `EnemyThreatIntentPreview`。
2. 在 `GameState.swift` 新增只读路线预测 helper，建议从现有 `movementRoutes(for:)` 抽出内部实现：
   - 保留现有 `movementRoutes(for:)` 行为不变，仍要求当前阵营和可移动。
   - 新增私有预测路径可对非 active faction 单位计算下一回合可达路线，并按下一回合重置行动处理 `hasMoved`/`hasAttacked`。
3. 在 `GameState` 新增直接攻击、机动接敌攻击、据点占领三类候选生成函数，并统一排序、去重和 `limit`。
4. `enemyThreatIntentPreviews(against:limit:)` 只读取当前状态，不调用 `move`、`attack`、`updateObjectiveControl`、`appendLog` 或任何会写状态的方法。
5. `ContentView.swift`：
   - 在 `InspectorPanel` 战报前或单位详情后新增 `EnemyThreatIntentPanel`。
   - `HexMapView` 计算威胁目标坐标并传入 `HexTileView`。

   - 图例增加敌方意图说明。
6. `GameStateTests.swift`：
   - 新增最小场景 helper，覆盖直接攻击、机动接敌和据点占领威胁。
   - 断言字段、排序、limit 和查询无副作用。
7. `RulesSmokeTest.swift` 补充同类高信号断言。
8. 更新文档：
   - `WW2Tactics/README.md`：敌方意图提示。
   - `md/flow/flow.md`：敌方威胁意图预判数据流和只读边界。
   - `md/flow/flowchart.md`：加入 `EnemyThreatIntentPreview`。
   - `md/test/test.md`：当前基线加入敌方威胁意图预判。
   - `update_log.md`：追加 v1.10 记录。

## 验收标准

- `enemyThreatIntentPreviews(against:limit:)` 能返回最多 `limit` 条稳定排序的敌方直接攻击、机动接敌攻击和据点占领威胁。
- 预判字段能说明敌方单位、目标、类型、路线/距离、预计伤害、击毁或占点风险。
- 查询预判不改变任何战役状态、行动状态、结果摘要、战报或消息。
- 侧栏和地图能展示敌方意图提示，且不遮挡现有单项结果卡、AI 回合摘要和地图命令标记。
- XCTest 和 smoke test 覆盖关键字段、排序、limit 和无副作用。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止通过临时修改 `activeFaction` 生成预测。
- 禁止调用 `move`、`attack`、`deploy`、`reinforce`、`useTacticalCommand` 等写状态方法生成预判。
- 禁止改变 AI 决策、AI 行动顺序、战斗数值、移动规则、部署/整补成本、胜负条件。
- 禁止在 `ContentView` 中计算威胁评分或战斗结果。
- 禁止让敌方意图标记覆盖或破坏 MOVE/ATK/POS/CAP/THR 的既有含义。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
