# v1.48 据点压力地图标记

## 版本号

v1.48

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.47。
- v1.47 已让据点压力行可点选定位，本轮继续补齐“侧栏压力 -> 地图反馈”的可视闭环。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

据点压力行现在能定位匹配守点反制或受威胁据点，但地图上主要依赖已有焦点、OBJ 引导和反制 ACT/SRC/CTR/TGT 标记。玩家点击压力行后，缺少一个明确说明“这就是当前压力条目”的地图短码，尤其在同格叠加敌方意图、目标引导和反制标记时，压力来源与当前选择不够直观。

## 目标

1. 新增只读的据点压力地图标记模型，标出当前点选压力的受压据点和可选反制目的格。
2. `GameState` 保存私有当前压力 id，并从最新 `battlefieldSituationSummary.objectivePressures` 重新派生地图标记。
3. `ContentView` 在地图格上显示压力短码，UI 不计算威胁、反制、坐标或排序。
4. 普通地图聚焦、选择单位、直接点选反制建议、重开和切换战役应清理压力标记，避免旧压力污染地图。
5. 增补 XCTest 和 rules smoke，覆盖标记字段、坐标、过期清理和只读边界。

## 非目标

- 不改变敌方意图、反制建议、据点压力排序、AI、score、移动、攻击、胜负或回合规则。
- 不新增自动守点、自动移动、自动攻击、自动整补或自动部署。
- 不在 `ContentView` 里重新计算 marker 数据。
- 不修改 workflow、Xcode project、长期分支策略或引入第三方依赖。

## 架构依据

- `GameModels.swift` 定义 `BattlefieldSituationObjectivePressureMapMarker` 和 role。
- `GameState.swift` 是唯一能重新校验当前压力 id 并派生 marker 的规则状态机。
- `ContentView.swift` 只按坐标分组并渲染 `GameState` 提供的 marker。
- `GameStateTests.swift` 与 `Tools/RulesSmokeTest.swift` 验证只读定位、marker 派生和清理边界。

## 实现步骤

1. 在 `GameModels.swift` 新增：
   - `BattlefieldSituationObjectivePressureMapMarkerRole`
   - `BattlefieldSituationObjectivePressureMapMarker`
   - role 至少包含受压据点和反制目的格。
2. 在 `GameState.swift` 新增私有 `focusedBattlefieldSituationObjectivePressureID`。
3. 新增 computed property `focusedBattlefieldSituationObjectivePressureMapMarkers`：
   - 无 id 或压力已过期时返回空数组。
   - 对受压据点输出压力 marker。
   - 有反制目的格且不同于据点坐标时输出反制目的格 marker。
4. 调整 `focusBattlefieldSituationObjectivePressure(id:)`：
   - 每次先清理旧压力 id。
   - 成功聚焦后设置当前压力 id。
   - 过期 id 只提示并保持 marker 为空。
5. 调整清理路径：
   - `clearObjectiveGuidance()` 清理压力 id。
   - `focusEnemyThreatCountermeasure(_:)` 直接点选普通反制时清理压力 id。
   - `loadScenario()` 清理压力 id。
6. 在 `ContentView`：
   - 从 `game.focusedBattlefieldSituationObjectivePressureMapMarkers` 按坐标分组。
   - 传入 `HexTileView`。
   - 新增压力 marker 视图和无障碍摘要。
   - 调整 marker 顶部 padding，避免和 INT、OBJ、RSP、反制 marker 重叠。
7. 扩展 XCTest 和 smoke：
   - 点选“后方油库”压力后出现受压据点 marker。
   - 有反制目的格时出现反制目的格 marker。
   - marker 读取不改变单位、据点、指令点、战报、latest result、AI summary。
   - 无效压力 id 清空 marker 并提示过期。

## 验收标准

- 点选据点压力行后，地图显示当前压力的受压据点 marker。
- 有匹配守点反制目的格时，地图显示反制目的格 marker。
- 普通聚焦、普通反制点选、重开或切战役不会保留旧压力 marker。
- marker 全部由 `GameState` 派生，`ContentView` 不计算规则。
- XCTest 和 rules smoke 覆盖 marker 坐标、过期清理和只读边界。
- Agent B 在 `main` 以 `v1.48: 增加据点压力地图标记` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。
