# v1.11 敌方意图反制建议

## 版本号

v1.11

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.10，本轮继续围绕地图操作体验、战前态势反馈和 EasyTech 风格决策辅助递增为 v1.11。
- v1.10 已实现敌方威胁意图预判，玩家可以看到下一轮敌方可能攻击的单位、机动接敌点和据点占领目标。本轮在此基础上增加“反制建议”，帮助玩家从威胁读图过渡到可执行的抢先攻击、撤退、据点防守或整补决策。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类二战回合制战棋体验。当前版本已经有路线风险、移动后火力暴露、安全接敌候选、目标计划、AI 回合摘要和敌方意图预判，但玩家仍需要自行判断“看到这个威胁后应该做什么”。

本轮要为 v1.10 的敌方意图预判补上一层只读的反制建议：针对最关键的敌方威胁，给出可解释、可测试、不会改写状态的玩家应对方案。它应像参谋建议一样服务于地图操作，而不是替代玩家执行动作。

## 目标

1. 在 `GameModels.swift` 新增敌方意图反制模型，建议包含：
   - `EnemyThreatCountermeasureKind`：至少区分抢先打击、撤出危险区、据点防守/封堵、整补支撑。
   - `EnemyThreatCountermeasurePreview: Identifiable, Equatable`：记录关联威胁、建议类型、执行单位、目标敌军或坐标、路线目的地和消耗、预计打击/保全效果、是否当前可执行、理由文本和排序评分。
2. 在 `GameState` 新增公开只读查询：
   - `enemyThreatCountermeasurePreviews(for threats: [EnemyThreatIntentPreview], limit: Int = 3) -> [EnemyThreatCountermeasurePreview]`
   - 或便捷属性 `visibleEnemyThreatCountermeasurePreviews`，内部基于 `visibleEnemyThreatIntentPreviews` 生成。
3. 反制建议必须是只读计算，不改变 `activeFaction`、单位位置、行动状态、地图归属、指令点、战报、消息、AI 摘要或任何 `latest*Result`。
4. 建议口径：
   - 抢先打击：若有未行动盟军单位当前即可攻击威胁来源，使用 `combatPreview(attacker:defender:)` 展示预计伤害、敌军剩余 HP 和是否可击毁。
   - 撤出危险区：若被威胁盟军单位仍可移动，给出一个可达且相对更安全的目的格，展示路线消耗和减少暴露的理由。
   - 据点防守/封堵：若敌方意图是占领据点，给出盟军可移动到目标据点或相邻封堵格的建议，展示执行单位和路线消耗。
   - 整补支撑：若被威胁单位受损且当前满足整补条件，展示可整补 HP 或预计整补后承受能力；只能预览，不能调用实际整补方法。
5. 排序建议：当前可执行 > 能击毁威胁来源 > 能保住据点 > 能把被威胁单位移出攻击范围 > 能整补关键单位 > 路线消耗更低 > 坐标/名称稳定排序。
6. `ContentView` 展示反制建议：
   - 可以合并进现有“敌方意图”面板，也可以新增紧邻的“反制建议”面板。
   - 每条建议显示类型、执行单位、目标/目的地、预计效果和一句简短理由。
   - UI 只读取 `GameState` 提供的结构化数据，不在 View 中计算评分、路线或战斗结果。
7. 补充 XCTest 和规则 smoke test，覆盖：
   - 可攻击威胁来源时返回抢先打击建议，字段与 `combatPreview` 一致。
   - 被威胁单位可移动时返回撤出危险区建议，包含目的格和路线消耗。
   - 敌方占点威胁存在时返回据点防守/封堵建议，目标坐标与威胁据点一致或相邻。
   - 受损单位满足条件时返回整补支撑建议，且不实际扣除指令点或改变 HP。
   - `limit` 生效且排序稳定。
   - 查询不会改变状态、战报、消息或结果摘要。
8. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不改变 AI 优先级、AI 实际行动、敌方意图预判口径、战斗数值、移动规则、整补成本、指令点收入、胜负条件。
- 不自动执行任何反制动作，不新增一键执行建议。
- 不引入真实迷雾、概率模拟、复杂热力图、动画、音效或美术资产。
- 不把核心反制计算写进 `ContentView`。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `GameModels.swift` 已有 `EnemyThreatIntentPreview`、`SafeEngagementOption`、`ObjectiveAdvancePreview`、`PostMoveFireExposurePreview`、`AIPhaseSummary` 等结构化预览模型，本轮应沿用 enum + struct 的轻量模型风格。
- `GameState.swift` 是移动、攻击、威胁覆盖、目标推进、整补和所有预览的唯一规则层；`ContentView` 只能展示 `GameState` 提供的结果。
- v1.10 的 `enemyThreatIntentPreviews(from:against:limit:)` 已经能生成敌方直接攻击、机动接敌攻击和据点占领风险，是本轮反制建议的输入。
- `combatPreview(attacker:defender:)` 是只读战斗估算入口，适合抢先打击建议复用。
- 现有 `movementRoutes(for:)`、`fireExposurePreview(for:to:)`、`objectiveAdvancePreviews(for:)`、`safeEngagementOptions(for:)`、整补判定 helper 等应优先复用；若现有 API 有 active faction 限制，必须保持玩家回合语义，不得临时修改 `activeFaction`。
- 所有会改变状态的方法只能作为禁止项参考，不能用于生成预览。

## 实现步骤

1. 在 `GameModels.swift` 新增 `EnemyThreatCountermeasureKind` 和 `EnemyThreatCountermeasurePreview`。
2. 在 `GameState.swift` 新增只读反制建议入口：
   - 基于传入 threats 或 `visibleEnemyThreatIntentPreviews`。
   - 为每条 threat 生成 0 到多条候选建议。
   - 统一排序、去重、裁剪 `limit`。
3. 实现抢先打击候选：
   - 查找当前可行动盟军单位。
   - 只允许当前位置已能攻击威胁来源的简单方案。
   - 使用 `combatPreview` 填充预计伤害、敌方剩余 HP 和击毁字段。
4. 实现撤出危险区候选：
   - 仅针对有 targetUnitID 的攻击类威胁。
   - 使用当前玩家可用移动路线寻找更安全目的地。
   - 可用当前威胁来源射程或既有暴露预览判断“风险降低”，算法保持稳定、可测试。
5. 实现据点防守/封堵候选：
   - 仅针对据点占领威胁。
   - 优先建议盟军进入目标据点；若不可进入，可建议相邻可达封堵格。
   - 路线和消耗来自规则层。
6. 实现整补支撑候选：
   - 仅针对受损且满足当前整补条件的被威胁盟军单位。
   - 只计算预计 HP 提升和理由，不调用 `reinforce`。
7. `ContentView.swift`：
   - 将反制建议面板放在敌方意图面板附近。
   - 保持信息密度，避免占用地图操作空间。
   - 不新增核心规则计算。
8. `GameStateTests.swift`：
   - 复用或扩展 v1.10 小地图场景 helper。
   - 覆盖抢先打击、撤退、防守/封堵、整补、limit、稳定排序和无副作用。
9. `RulesSmokeTest.swift` 补充高信号断言，确保命令行 smoke 覆盖新预览链路。
10. 更新文档：
   - `WW2Tactics/README.md`：说明敌方意图面板会给出反制建议。
   - `md/flow/flow.md`：加入反制建议数据流和只读边界。
   - `md/flow/flowchart.md`：加入 `EnemyThreatCountermeasurePreview` 流程。
   - `md/test/test.md`：当前基线加入反制建议测试。
   - `update_log.md`：追加 v1.11 记录。

## 验收标准

- `visibleEnemyThreatCountermeasurePreviews` 或等价 API 能基于当前敌方意图返回最多 `limit` 条稳定排序的反制建议。
- 建议字段能说明建议类型、执行单位、目标或目的地、路线消耗、预计效果和理由。
- 查询反制建议不改变任何战役状态、行动状态、结果摘要、战报、消息、HP、指令点或地图归属。
- 侧栏能展示反制建议，且不遮挡现有单项结果卡、AI 回合摘要、敌方意图和地图命令标记。
- XCTest 和 smoke test 覆盖关键字段、limit、排序和无副作用。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止通过临时修改 `activeFaction` 或单位行动状态生成建议。
- 禁止调用 `move`、`attack`、`deploy`、`reinforce`、`useTacticalCommand`、`advanceTurn`、`performAIPhase` 等写状态方法生成建议。
- 禁止改变 AI 决策、敌方意图预判、战斗数值、移动规则、部署/整补成本、胜负条件。
- 禁止在 `ContentView` 中计算反制评分、路线、战斗结果或整补结果。
- 禁止让反制建议 UI 抢占地图主操作或覆盖 MOVE/ATK/POS/CAP/THR/INT 的既有含义。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
