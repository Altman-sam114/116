# v1.52 据点压力态势对照

## 版本号

v1.52

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.51。
- v1.51 已让当前据点压力能保守关联 AI 复盘线索，本轮继续强化压力行本身的可读性，把“当前守势”和“推荐应对”并列说明。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

据点压力列表已经能显示受压据点、威胁来源、地图标记、当前态和复盘线索，但压力行仍主要是“敌军威胁夺取 + 入口短码”。玩家需要在一眼内判断这个据点当前是己方守点、中立争夺还是已被敌方控制，以及当前是否有可执行守点入口。EasyTech 风格大地图攻防强调快速读懂“当前态势 -> 应对动作”，本轮补一层只读态势对照。

## 目标

1. 为 `BattlefieldSituationObjectivePressure` 增加结构化 `comparison`，表达当前守势和推荐应对。
2. comparison 必须从已有 owner、威胁来源数量、首要威胁、actionHint、countermeasure/replay 纯派生。
3. `ContentView` 在压力行内展示紧凑的“当前 / 应对”两行，不新增按钮、不嵌套按钮。
4. 只使用“态势对照 / 当前 / 应对 / 线索”等解释措辞，不模拟敌方未来行动。
5. 增补 XCTest 和 rules smoke，覆盖字段、UI 所需数据、只读边界和 stale 清理不回归。

## 非目标

- 不改变敌方意图、反制建议、AI、据点压力排序、score、移动、攻击、胜负或回合规则。
- 不新增自动防守、自动移动、自动攻击、自动整补、自动部署或未来分支模拟。
- 不在 SwiftUI 中计算 owner、威胁来源、可执行性或推荐文本。
- 不新增第三方库、资产管线、workflow、Xcode project 或长期分支策略。

## 架构依据

- `BattlefieldSituationObjectivePressure` 已是压力行的数据模型，适合承载 UI 需要的只读解释字段。
- `GameState.battlefieldSituationObjectivePressures(threats:countermeasures:)` 已集中构造 pressure，可在这里从已有字段派生 comparison。
- `ContentView.BattlefieldSituationObjectivePressureRow` 只展示 pressure 字段并转发 id，不应反查单位、威胁或反制。

## 实现步骤

1. 在 `GameModels.swift` 增加：
   - `BattlefieldSituationObjectivePressureComparisonLevel`：例如 `.held`、`.contested`、`.enemyHeld`、`.neutral`，提供 `title` / `shortTitle`。
   - `BattlefieldSituationObjectivePressureComparison`：包含 `level`、`currentTitle`、`currentDetail`、`responseTitle`、`responseDetail`。
   - `BattlefieldSituationObjectivePressure.comparison` 字段。
2. 在 `GameState` 构造 pressure 时填充 comparison：
   - 当前态：己方控制写“守点承压”，敌方控制写“敌控据点”，中立写“中立争夺”。
   - 当前详情：归属 + 威胁来源数量 + 路线状态。
   - 应对标题：使用 actionHint 的入口标题或 countermeasure 类型。
   - 应对详情：说明是否可执行、是否有复盘线索；只能解释入口，不执行命令。
3. 在 `BattlefieldSituationObjectivePressureRow` 中显示 comparison：
   - 左侧在威胁详情和 action hint 之间增加两行紧凑文本或 label。
   - 右侧短码可继续显示 threat title/source/action code。
   - 保持行本身仍是外层按钮，不新增内层按钮。
4. 扩展 `testBattlefieldSituationObjectivePressureFocusesWithoutExecuting()`：
   - 断言 comparison 当前态、当前详情、应对标题、应对详情包含预期信息。
   - 注入 AI replay 后 comparison 应显示有复盘线索，但不改变 pressure id。
   - 现有只读边界保持不变。
5. 扩展 `RulesSmokeTest.swift` 同等断言。
6. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 据点压力行显示结构化“当前 / 应对”态势对照。
- comparison 完全由 `GameState` 构造，SwiftUI 不计算威胁、反制或可执行性。
- 点选压力、点选复盘线索、普通聚焦和 stale id 的只读边界不回归。
- XCTest 和 rules smoke 覆盖 comparison 字段和只读边界。
- Agent B 在 `main` 以 `v1.52: 增加据点压力态势对照` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。

## 风险和禁止项

- 禁止把 comparison 写成未来模拟或因果预测。
- 禁止让态势对照执行 MOVE、ATK、整补、部署、战术命令、endTurn 或 AI。
- 禁止在 SwiftUI 中反查单位、扫描 threats/countermeasures 或新增独立压力状态。
- 禁止改变据点压力排序、反制 score、敌方意图或 AI 行动来适配文案。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端验收。
