# v1.39（普通行动态势响应）

## 版本号

v1.39

## 版本分配依据

当前 `v1（地图操作体验）` 最新功能版本为 v1.38，已把战线态势卡与 AI 复盘关键事件定位连接起来。本轮继续沿“行动结果 -> 态势解释 -> 复盘定位”的链路递增为 v1.39，把普通攻击、战术命令、部署和整补结果纳入战线态势响应，让玩家不依赖反制建议也能在态势卡看到最近行动的实际影响。

## 背景

现有 `battlefieldSituationResponseSummary` 只覆盖反制敌方回合复核、反制执行回放和据点占领结果。普通攻击、战术命令、主动部署和主动整补虽然已有专门结果卡，但战线态势卡不会显示这些行动反馈，导致玩家在执行非反制动作后，态势卡仍像“没有响应”。EasyTech 风格战棋需要在每次关键操作后快速告诉玩家战线发生了什么变化。

## 目标

- 将普通攻击、战术命令、部署和整补结果纳入 `battlefieldSituationResponseSummary`。
- 保持既有优先级：敌方回合复核 > 反制执行回放 > 据点占领 > 普通战斗/战术/后勤结果。
- 新响应只从已有 `latestCombatResult`、`latestTacticalCommandResult`、`latestDeploymentResult`、`latestReinforcementResult` 纯派生，不新增 `@Published` 状态。
- 战线态势卡继续只展示 `BattlefieldSituationResponseSummary`；SwiftUI 不计算伤害、恢复、消耗、坐标或响应优先级。
- 普通移动仍不生成态势响应；失败命令、预览、聚焦和过期状态不伪造响应。
- 补充 XCTest 和规则 smoke test，覆盖四类普通响应、优先级、只读无副作用，以及旧的“普通攻击不生成响应”断言更新为新行为。
- 同步 README、flow、flowchart、test、prompt README 和 update_log；同时补齐 v1.38 记录中的最终云端验收细节。

## 非目标

- 不改变普通攻击、战术命令、部署、整补、据点占领、反制执行或 AI 的生成/清理语义。
- 不改变战斗数值、士气、经验、指令点、补给、AI 行为或胜负判定。
- 不新增自动执行按钮或新的行动入口。
- 不让普通移动生成响应。
- 不把核心判断写进 `ContentView`。

## 当前架构依据

- `CombatResultSummary` 已记录攻击方/防御方 HP 前后、伤害、反击、击毁、追击和防御姿态消耗。
- `TacticalCommandResultSummary` 已记录命令类型、施放者/目标、伤害、指令点消耗、士气损失、状态、击毁和无反击。
- `DeploymentResultSummary` 和 `ReinforcementResultSummary` 已记录单位、坐标、消耗、恢复量和剩余指令点。
- `BattlefieldSituationResponseSummary` 是战线态势响应统一展示模型，适合扩展 response kind 并复用现有 UI。
- `battlefieldSituationResponseSummary` 是 computed property，当前优先级已固定为 follow-up、countermeasure、objective capture，本轮应只在后面追加普通结果兜底。

## 实现步骤

1. 在 `BattlefieldSituationResponseKind` 增加普通响应类型，例如：
   - `combat`
   - `tacticalCommand`
   - `deployment`
   - `reinforcement`
2. 为新类型补充 `shortTitle` 和 `iconName`，短码和图标应能在态势卡中区分 ATK、CMD、DEP、REP。
3. 更新 `GameState.battlefieldSituationResponseSummary` 优先级：
   - `latestEnemyThreatCountermeasureFollowUpResult`
   - `latestEnemyThreatCountermeasureExecutionResult`
   - `latestObjectiveCaptureResult`
   - `latestCombatResult`
   - `latestTacticalCommandResult`
   - `latestDeploymentResult`
   - `latestReinforcementResult`
4. 新增私有映射 helper：
   - 普通攻击：标题表达攻击/击毁结果，detail 表达 attacker -> defender，resultTitle 显示伤害/反击，resultDetail 显示目标 HP 前后和追击/防御消耗等关键影响，coordinate 使用 defender 仍存在时的位置，否则攻击者位置或 nil 的稳定可用坐标。
   - 战术命令：标题表达命令名和是否击毁，detail 表达 caster -> target，resultTitle 显示伤害和指令消耗，resultDetail 显示士气、状态、无反击或防御姿态消耗，coordinate 使用 target 仍存在时的位置，否则 caster 位置或 nil。
   - 部署：标题表达部署单位，detail 表达来源据点和兵种，resultTitle 显示指令消耗/剩余，resultDetail 显示部署坐标，coordinate 使用部署坐标。
   - 整补：标题表达整补单位，detail 表达 HP 前后，resultTitle 显示恢复量和指令消耗，resultDetail 显示剩余指令点，coordinate 使用整补坐标。
5. 若需要坐标回查，只能在 `GameState` 中按 result 的 unitID 查当前单位位置；View 不做回查。
6. 更新 `ContentView` 的 response color 映射，让新 kind 有清晰但不抢占当前状态色的颜色。
7. 更新 XCTest：
   - 普通攻击现在应生成 `.combat` 态势响应，并保持读取只读不变性。
   - 战术命令生成 `.tacticalCommand` 态势响应。
   - 部署生成 `.deployment` 态势响应。
   - 整补生成 `.reinforcement` 态势响应。
   - 反制执行响应仍优先于同一次攻击/整补的普通响应。
   - 据点占领仍优先于普通移动/战斗结果；敌方回合 follow-up 仍最高优先级。
   - 普通移动、失败命令和预览仍不生成响应。
8. 更新 `RulesSmokeTest.swift` 中旧断言：
   - `ordinary attack without focused countermeasure should not publish battlefield situation response` 改为断言 `.combat`；
   - 普通整补增加 `.reinforcement`；
   - 补充普通战术命令和部署响应 smoke；
   - 保留反制 replay 优先级断言。
9. 同步文档和日志。

## 验收标准

- 普通攻击、战术命令、部署、整补成功后，战线态势卡能显示对应 `BattlefieldSituationResponseSummary`。
- 响应优先级保持 follow-up > countermeasure > objective capture > ordinary combat/tactical/logistics。
- 普通移动、失败命令、预览和聚焦不生成态势响应。
- 读取响应摘要不改变单位、据点、指令点、战报、消息、AI summary、latest result 或胜负。
- SwiftUI View 不计算响应规则，只展示 `GameState` 给出的字段。
- 本地轻量检查通过，提交并 push 到 `origin/main`。
- Agent C 下载最新 run artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。

## 风险和禁止项

- 禁止改变现有 `latest*Result` 的互斥清理语义。
- 禁止让普通移动生成响应，避免地图探索和实际执行混淆。
- 禁止降低 v1.37 follow-up 和 v1.36 反制/占点响应优先级。
- 禁止在 View 中推导伤害、恢复、消耗、坐标或响应优先级。
- 禁止跳过 Agent C artifact 验收。
- 若 GitHub 权限、账号、远端或 artifact 下载失败，必须停止并说明阻塞，不能伪装云端验证完成。
