# v1.25 AI复盘事件选中态和地图标记强调

## 版本号

v1.25

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.24（AI时间线点选定位复盘）。`origin/main` 最新验收 commit `762f87034e85ac03a9e3911852ac1e6723d1d51a` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.24-main-762f870-run28744791567-attempt1` 核对。v1.24 已让侧栏 AI 时间线行可点选并定位地图焦点；下一步应补齐复盘操作反馈，让玩家清楚知道当前正在查看哪一条 AI 事件，以及地图上哪些 AI 标记与该事件相关。本轮递增为 v1.25。

## 背景

当前点选 AI 时间线后，地图能滚动到事件坐标并显示 `AI复盘 #<order>` 消息，但侧栏时间线行不会保留选中态，地图上同坐标聚合的 AI 标记也不会强调“当前事件”。在大地图、多事件复盘时，玩家仍可能难以判断自己刚点的是哪一步、哪个标记与该事件关联。

本轮目标是新增“当前 AI 复盘事件”只读选中态：`GameState` 记录当前选中的 timeline order，侧栏行根据该 order 显示选中样式，地图 AI 复盘标记根据同一 order 做视觉强调和无障碍说明。点选仍然只定位复盘，不执行命令、不改变 AI 结果。

## 目标

1. 在 `GameState` 增加当前 AI 复盘事件选中态：
   - 新增 `@Published private(set) var focusedAIPhaseTimelineEventOrder: Int?`。
   - `focusAIPhaseTimelineEvent(order:)` 成功定位时设置该 order。
   - 无 summary、无事件、无坐标或坐标不在当前地图时不改变旧 order。
   - 切战役、重开、重新加载场景、下一次 AI summary 被清理或覆盖时清理旧 order，避免过期选中态。
2. 增加只读派生 helper：
   - 新增 `focusedAIPhaseMapMarkers` 或等价派生属性，从 `latestAIPhaseMapMarkers` 过滤当前 order。
   - 该派生属性不新增独立 marker 存储，不改变 `latestAIPhaseMapMarkers`。
3. 强化 SwiftUI 复盘反馈：
   - `AIPhaseSummaryView`/`AIPhaseTimelineEventRow` 根据 `focusedAIPhaseTimelineEventOrder` 显示当前行选中态。
   - 选中态必须不只靠颜色，建议增加细描边、左侧小符号或当前行背景。
   - `HexMapView` 将 `focusedAIPhaseMapMarkers` 按坐标传给 `HexTileView`。
   - `HexTileView` / `AIPhaseMapReplayMarker` 对当前事件相关标记显示更高优先级边框或强调徽标，并在 tile 无障碍文案中说明“当前AI复盘”。
4. 保持架构边界：
   - View 只读取 `GameState` 状态和转发点选意图。
   - 坐标有效性、order 设置和清理都在 `GameState`。
   - 不把 UI 选中态作为规则执行条件。
5. 补充 XCTest 和规则 smoke test：
   - 成功点选时间线后，`focusedAIPhaseTimelineEventOrder == event.order`。
   - `focusedAIPhaseMapMarkers` 只包含该 order 的 markers，且等于 `latestAIPhaseMapMarkers` 中对应过滤结果。
   - 无效 order / 无 summary 不改变旧 order。
   - 重开、切战役、下一次 AI 回合或 summary 清理会清空旧 order。
   - 点选和选中态不改变单位、指令点、AI summary、timeline 或 `latestAIPhaseMapMarkers`。
6. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变 AI 决策、行动顺序、时间线事件生成、地图复盘 marker 生成规则或 `latestAIPhaseMapMarkers` 的纯派生语义。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励或胜负条件。
- 不新增逐帧播放、自动镜头动画、暂停/继续回放、历史 AI 回合列表、音效、真实美术资产或复杂回放控制器。
- 不让 SwiftUI 自行决定当前 order 是否有效；有效性判断必须在 `GameState`。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `GameState.focusAIPhaseTimelineEvent(order:)` 是 v1.24 引入的唯一 AI 时间线点选入口，已负责查找事件、校验坐标、更新 `focusedCoordinate` 和消息。
- `latestAIPhaseMapMarkers` 已从 `latestAIPhaseSummary.timeline` 纯派生，适合作为当前事件 marker 过滤来源。
- `ContentView.HexMapView` 已按坐标把 `latestAIPhaseMapMarkers` 分组传入 `HexTileView`。
- `HexTileView` 已通过 `borderColor`、`borderWidth`、`AIPhaseMapReplayMarker` 和 `aiPhaseMapAccessibilityText` 展示 AI 复盘标记。
- `AIPhaseSummaryView` 已用 `Button` 转发时间线行点选到 `GameState`。

## 实现步骤

1. 修改 `GameState.swift`：
   - 在现有 AI summary 状态附近新增 `@Published private(set) var focusedAIPhaseTimelineEventOrder: Int?`。
   - 新增 `var focusedAIPhaseMapMarkers: [AIPhaseMapMarker]`，guard 当前 order 后从 `latestAIPhaseMapMarkers` 过滤。
   - `focusAIPhaseTimelineEvent(order:)` 成功分支设置 `focusedAIPhaseTimelineEventOrder = event.order`；失败分支保留旧值。
   - 在 `loadScenario()`、`restart()` 或实际清理 `latestAIPhaseSummary` 的路径中同步清理 order。
   - 在 `beginAIPhaseRecording` 或 `finishAIPhaseRecording` 发布新 summary 前后清理/覆盖旧 order，确保新 AI 回合不会保留上一轮选中态。
2. 修改 `ContentView.swift`：
   - 在 `HexMapView` 中按坐标分组 `game.focusedAIPhaseMapMarkers`，并传给 `HexTileView`。
   - `HexTileView` 新增 `focusedAIPhaseMapMarkers: [AIPhaseMapMarker]` 参数。
   - `AIPhaseMapReplayMarker` 新增 `isFocused: Bool` 参数；当前事件 marker 使用更亮背景、描边或小符号，不只靠颜色。
   - `borderColor` / `borderWidth` 在有 focused AI marker 时提高到 AI marker 的优先级，但不要压过 selected/MOVE/ATK/POS/反制建议等更强交互状态。
   - `aiPhaseMapAccessibilityText` 对当前 marker 加入“当前AI复盘第 N 步”。
   - `AIPhaseTimelineEventRow` 接收 `isFocused`，选中行添加背景/描边/小符号，并在 accessibility label 或 hint 中说明“当前复盘事件”。
3. 修改 XCTest：
   - 扩展 v1.24 的 `testAxisAIMovesThenUsesArtilleryBarrageWhenOnlyBarrageIsInRange`。
   - 成功 focus 后断言 `focusedAIPhaseTimelineEventOrder` 和 `focusedAIPhaseMapMarkers`。
   - 断言 `latestAIPhaseMapMarkers` 没被改写，单位/指令点/summary 保持不变。
   - 无效 order 不改变旧 order。
   - 在 `testAIPhaseSummaryClearsOnScenarioReset` 或新增测试中断言重开/切战役清理 focused order 和 focused markers。
4. 修改 `RulesSmokeTest.swift`：
   - 在移动后弹幕复盘 focus smoke 中断言 focused order 和 focused markers。
   - 在 reset 或无效 order smoke 中断言旧 order 保留或清理规则。
5. 文档同步：
   - `WW2Tactics/README.md` AI / 地图交互 / 战斗辅助说明加入“当前 AI 复盘事件选中态和地图标记强调”。
   - `md/flow/flow.md` AI 回合摘要段落加入 order 状态、派生 markers 和清理规则。
   - `md/flow/flowchart.md` 在 AI timeline -> map marker 链路加入 selected order / highlighted marker 节点。
   - `md/test/test.md` smoke / XCTest 基线加入 AI 复盘事件选中态和标记强调。
   - `update_log.md` 追加 v1.25 记录，并修正顶部当前状态。

## 验收标准

1. 点选 AI 时间线成功后，`focusedAIPhaseTimelineEventOrder` 等于被点事件 order。
2. `focusedAIPhaseMapMarkers` 从 `latestAIPhaseMapMarkers` 过滤得出，只包含当前 order，不新增独立 marker 状态。
3. 侧栏当前时间线行有可见选中态，且选中态不只依赖颜色。
4. 当前事件相关地图 AI 标记有可见强调，tile 无障碍文案包含当前复盘说明。
5. 无效 order、无 summary、无坐标或坐标不在地图时不改变旧 order 或旧焦点。
6. 重开、切战役和新 AI 回合会清理旧 focused order / focused markers。
7. 点选和选中态不会移动单位、攻击、消耗指令点、改变 HP、改变 AI summary、改变 timeline 或改变 `latestAIPhaseMapMarkers`。
8. 本地轻量检查通过。
9. Agent B 在 `main` 提交并 `git push origin main`。
10. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
11. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：若不清理旧 focused order，切战役或新 AI 回合后会显示过期选中态；必须在 summary 生命周期内清理。
- 风险：若把 focused marker 存成独立数组，可能与 `latestAIPhaseMapMarkers` 失步；必须派生过滤。
- 风险：如果地图强调优先级过高，可能遮挡 MOVE/ATK/POS/反制建议等当前交互；应低于真实命令和反制聚焦。
- 风险：只用颜色区分选中态不利于无障碍；必须加形状、描边、符号或文案。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
