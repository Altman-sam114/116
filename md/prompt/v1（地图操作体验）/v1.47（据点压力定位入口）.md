# v1.47 据点压力定位入口

## 版本号

v1.47

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.46。
- v1.46 已在战线态势卡中展示据点防守压力列表，本轮继续在同一链路上增强可操作性。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

战线态势卡已经能把 `.objectiveCapture` 威胁按据点聚合为“据点压力”，并显示据点归属、威胁来源、占点风险和推荐入口。但压力行本身仍只是信息展示，玩家看到某个据点承压后，还需要再通过首要定位或其他列表间接寻找防守位置。

## 目标

1. 让据点压力行成为只读定位入口。
2. 点击压力行时，`GameState` 重新读取最新 `battlefieldSituationSummary.objectivePressures` 并校验压力 id。
3. 若压力仍有匹配反制建议，复用 `focusEnemyThreatCountermeasure(_:)` 定位执行单位、目的格和守点引导。
4. 若压力没有匹配反制建议，定位受威胁据点坐标并保留目标引导。
5. 补充 XCTest 和 rules smoke，证明压力定位不会执行移动、攻击、整补、部署、战术命令或 AI，也不会改变单位、据点、指令点和战报。

## 非目标

- 不新增一键守点、自动移动、自动攻击、自动整补或自动部署。
- 不改变敌方意图、反制建议、据点压力排序、AI、score、移动、攻击、胜负或回合规则。
- 不在 `ContentView` 里重新计算威胁、反制、路线或定位有效性。
- 不修改 workflow、Xcode project、长期分支策略或引入第三方依赖。

## 架构依据

- `GameModels.swift` 已提供 `BattlefieldSituationObjectivePressure` 的稳定 id 和反制预览引用。
- `GameState.swift` 是唯一能重新校验当前压力、反制和地图坐标的规则状态机。
- `ContentView.swift` 只把按钮点击转发到 `GameState`，展示模型字段和无障碍文案。
- `GameStateTests.swift` 与 `Tools/RulesSmokeTest.swift` 验证定位行为和只读边界。

## 实现步骤

1. 在 `GameState.swift` 新增 `focusBattlefieldSituationObjectivePressure(id:)`：
   - 胜负已结算时只提示复盘/重新开局。
   - 从最新 `battlefieldSituationSummary.objectivePressures` 查找 id。
   - 找不到时提示“据点压力目标已过期”。
   - 有 `countermeasurePreview` 时调用 `focusEnemyThreatCountermeasure(_:)`，再把 message 包装为“据点压力定位：...”，保持只读。
   - 无 countermeasure 时校验据点坐标，清理临时引导，设置 `focusedCoordinate` 和 `guidedObjectiveCoordinate`，写入防守查看提示。
2. 在 `ContentView.BattlefieldSituationSummaryView` 中，将 `BattlefieldSituationObjectivePressureRow` 包成 `Button`，按钮 action 传 pressure id 给 `GameState`。
3. 更新压力行无障碍文案，明确“只定位，不执行命令”。
4. 在 `GameStateTests.swift` 增加或扩展压力定位测试。
5. 在 `Tools/RulesSmokeTest.swift` 加入同等 smoke 断言。
6. 更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- 据点压力行可点选定位，但不会自动执行任何命令。
- 压力行点击由 `GameState` 重新校验最新压力 id，过期 id 不会使用旧对象继续聚焦。
- 有匹配反制时复用现有反制聚焦和执行入口桥接；无匹配反制时定位受威胁据点。
- XCTest 和 rules smoke 覆盖压力定位只读边界。
- README、flow、flowchart、test、prompt README 和 update_log 与真实行为一致。
- Agent B 在 `main` 以 `v1.47: 增加据点压力定位入口` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。

## 风险和禁止项

- 禁止让压力行直接调用 MOVE、ATK、整补、部署、战术命令、endTurn 或 AI。
- 禁止把压力 id 以外的旧规则对象作为唯一真相。
- 禁止在 SwiftUI 中计算威胁来源、反制选择、路线合法性或地图坐标有效性。
- 禁止改变据点压力排序、反制 score 或 AI 行动来适配 UI。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端验收。
