# v1.55 据点压力来源标识

## 版本号

v1.55

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.54。v1.54 已把真实守点 follow-up 的敌方回合前后影响映射回据点压力行；本轮继续围绕同一条战线态势压力链路递增为 v1.55。

## 背景

v1.54 让压力列表在当前夺点威胁消失后仍能保留“敌方回合后复核”的压力结果，但当前威胁和回合复核在模型和 UI 上仍共用同一组标题字段。玩家需要更快分辨“现在必须处理的压力”和“上一敌方回合的结果复核”。

## 目标

- 为 `BattlefieldSituationObjectivePressure` 增加来源标识，区分当前威胁和回合复核。
- 当前威胁压力应稳定排在回合复核压力之前，避免已复核结果压过当回合需要操作的据点压力。
- `ContentView` 压力行显示来源短码/标题，并把来源写入无障碍文案。
- 补充 XCTest 和 RulesSmokeTest，覆盖来源字段、排序和只读边界。

## 非目标

- 不改变敌方意图、反制建议、follow-up、AI、移动、攻击、整补、部署或据点归属规则。
- 不新增未来模拟、AI 因果归因或压力评分。
- 不改变 v1.53 的 PRS/RSP/KEY 复盘来源优先级。
- 不引入第三方库、网络依赖、动画、音效或新美术资源。

## 当前架构依据

- `GameModels.swift` 定义结构化数据和枚举。
- `GameState.swift` 负责只读派生来源和排序。
- `ContentView.swift` 只展示 `GameState` 派生字段，不反推规则。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 锁定压力行行为、排序和只读边界。

## 实现步骤

1. 在 `GameModels.swift` 新增 `BattlefieldSituationObjectivePressureSource`，包含当前威胁和回合复核的 title、shortTitle、sortRank。
2. 在 `BattlefieldSituationObjectivePressure` 新增 `source` 字段；当前 `.objectiveCapture` 压力使用 `.currentThreat`，follow-up 派生压力使用 `.enemyPhaseFollowUp`。
3. 调整 `battlefieldSituationObjectivePressureSort`，优先按 source.sortRank 排序，确保当前威胁排在回合复核前。
4. 在 `BattlefieldSituationObjectivePressureRow` 显示来源短码，并更新 accessibility summary。
5. 扩展 `testBattlefieldSituationObjectivePressureFocusesWithoutExecuting` 和守点 follow-up 测试，覆盖 source 和 id 稳定性。
6. 扩展 RulesSmokeTest 对应断言。
7. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 当前威胁压力 source 为 `.currentThreat`，回合复核压力 source 为 `.enemyPhaseFollowUp`。
- 当前威胁压力稳定排在回合复核压力前。
- UI 压力行显示来源短码，VoiceOver 文案包含来源标题。
- 点选压力行仍不执行移动、攻击、整补、部署、战术命令或 AI。
- 本地轻量检查通过。
- `main` 推送后 GitHub Actions run 成功，Agent C 下载 artifact 并核对 manifest 的 branch、commitSha、runId、runAttempt。
