# v1.5 据点推进计划摘要

## 版本号

v1.5

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.4，本轮继续围绕地图操作体验和 OBJ 据点引导递增为 v1.5。
- v1.4 已完成据点占领结果摘要和 CAP 地图反馈，本轮在不改变 OBJ 执行规则的前提下，把已有 OBJ 推进计划公开为可读、可测、可在 UI 呈现的计划摘要。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类大地图、多据点、单位移动、攻击判定和目标推进体验。当前 `GameState` 已有私有 `ObjectiveAdvancePlan`、`nearestObjectiveTarget(for:)`、`objectiveAdvanceRoute(for:to:)` 和 `focusNearestObjectiveTarget()`，OBJ 快捷按钮能选择最近空置敌方/中立据点并移动或推进。

当前不足是玩家只能看到一个 OBJ 快捷动作结果，无法提前比较多个据点推进方向、距离、移动消耗和火力风险。v1.5 要把同一套 OBJ 计划排序结果转换为结构化预览，并在选中单位时显示最多 3 个据点推进计划。

## 目标

1. 新增公开、只读、可测试的据点推进计划摘要模型，表达最多 3 个候选目标。
2. `GameState` 暴露基于现有 `objectiveAdvancePlans(for:)` 的预览 API，保证首项与 `nearestObjectiveTarget(for:)` / OBJ 快捷按钮完全一致。
3. UI 在选中单位时显示紧凑的“目标计划”摘要，包含目标名、归属、目的格、是否本回合可占领、距离变化、移动消耗、控制区惩罚和终点火力风险。
4. 补充 XCTest 和规则 smoke test，覆盖直接占领、远距推进、占据目标过滤、最多 3 条且排序稳定、首项与 OBJ 快捷动作一致。
5. 同步 README、flow、flowchart、update_log，记录 v1.5 行为、架构和验证结果。

## 非目标

- 不改变移动、路径搜索、控制区、攻击、占领、奖励、胜负、AI 或战术命令规则。
- 不改变 `focusNearestObjectiveTarget()` 的执行语义和提示语，除非为复用新预览模型做等价重构。
- 不新增第三方库、网络依赖、大体积资产或复杂资产管线。
- 不把 OBJ 推进核心规则写进 `ContentView`。
- 不做新战役、新单位生产、将领系统或完整 AI 攻防扩展。

## 当前架构依据

- `WW2Tactics/WW2Tactics/GameModels.swift` 放数据模型、枚举、战役定义、地图格、单位、命令预览类型。本轮新增的公开预览模型应放这里。
- `WW2Tactics/WW2Tactics/GameState.swift` 放核心规则状态机和目标推进逻辑。本轮应复用私有 `ObjectiveAdvancePlan` 与 `objectiveAdvancePlans(for:)`，避免 UI 或测试复制排序逻辑。
- `WW2Tactics/WW2Tactics/ContentView.swift` 放 SwiftUI 表现层。只能读取 `GameState` 暴露的结构化预览并渲染，不得计算核心目标推进规则。
- `WW2Tactics/WW2TacticsTests/GameStateTests.swift` 放 XCTest 规则和交互行为测试。
- `WW2Tactics/Tools/RulesSmokeTest.swift` 放命令行 smoke test，必须覆盖主规则链。

## 实现步骤

1. 在 `GameModels.swift` 新增 `ObjectiveAdvancePreview` 或同等命名结构体，建议字段：
   - `objectiveName: String`
   - `coordinate: HexCoordinate`
   - `owner: Faction?`
   - `route: MovementRoute`
   - `reachesObjective: Bool`
   - `currentDistance: Int`
   - `remainingDistance: Int`
   - `fireExposure: PostMoveFireExposurePreview?`
   - 可选便捷展示属性，例如 `ownerTitle`、`destinationText`、`stanceText`。
2. 在 `GameState.swift` 新增：
   - `func objectiveAdvancePreviews(for unit: BattleUnit, limit: Int = 3) -> [ObjectiveAdvancePreview]`
   - `var focusedObjectiveAdvancePreviews: [ObjectiveAdvancePreview]`
   - 内部转换函数从 `ObjectiveAdvancePlan` 生成预览，并用 `fireExposurePreview(for:at:)` 计算路线终点风险。
3. 保证 `objectiveAdvancePreviews(for:)` 返回顺序与 `objectiveAdvancePlans(for:)` 一致，默认最多 3 条；`limit <= 0` 返回空数组。
4. 保证 `nearestObjectiveTarget(for:)` 和 OBJ 快捷动作仍从同一套计划排序取得首项；首项 `coordinate` 必须等于 `nearestObjectiveTarget(for:)?.coordinate`。
5. 在 `ContentView.swift` 为选中单位侧栏或 HUD 增加紧凑“目标计划”块：
   - 仅在选中单位且有预览时显示。
   - 第一条以视觉强调标注 OBJ 当前快捷目标。
   - 每条显示目标名、归属、`夺取/占领/推进`、`q,r` 目的格、当前距离到剩余距离、移动消耗、控制区惩罚、火力风险等级。
   - 使用现有字体层级和 8px 以内圆角风格；按钮/图标要保留可访问文本。
6. 补充测试：
   - 直接可达目标预览 `reachesObjective == true`，route 终点是目标格，首项匹配 `nearestObjectiveTarget`。
   - 远距目标预览 `reachesObjective == false`，route 终点是最佳推进格，remainingDistance 正确。
   - 被单位占据的目标不会出现在预览里。
   - 多目标场景最多返回 3 条，排序稳定，首项和 OBJ 快捷动作一致。
   - 若测试场景有敌火，断言 `fireExposure` 风险字段来自路线终点；没有敌火时允许为 nil 或 low，按现有 `fireExposurePreview` 行为判断。
7. 更新 smoke test 中 OBJ 相关段落，加入预览 API 的关键断言。
8. 更新文档：
   - `WW2Tactics/README.md`：用户可见行为增加“目标计划”摘要。
   - `md/flow/flow.md`：当前核心逻辑加入 OBJ 计划预览生成和 UI 消费。
   - `md/flow/flowchart.md`：数据流/执行流图加入 OBJ 计划预览。
   - `md/test/test.md`：若测试命令或触发条件不变，可记录 v1.5 覆盖点；若不变也应保持与真实命令一致。
   - `update_log.md`：追加 v1.5 记录，包含核心变更、关键文件、验证结果和遗留事项。

## 验收标准

- 选中可移动单位时，UI 能看到最多 3 条 OBJ 据点推进计划。
- 第一条目标与 OBJ 快捷按钮实际聚焦/执行目标一致。
- 直接可达目标显示为本回合可夺取/占领，远距目标显示推进目的格和剩余距离。
- 占据了敌军单位的目标不会进入计划列表。
- 控制区惩罚、移动消耗和终点火力风险来自 `GameState` 现有路线和火力预览。
- XCTest 和 smoke test 覆盖新增预览 API 的核心行为。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止在 `ContentView` 中重新计算目标排序、路径或火力风险。
- 禁止让新增预览 API 与 `focusNearestObjectiveTarget()` 使用不同排序来源。
- 禁止改变 v1.4 的 CAP 反馈、占领奖励和战斗/战术结果摘要清理规则。
- 禁止为了展示计划而扩大地图布局、引入营销页或静态展示页。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
