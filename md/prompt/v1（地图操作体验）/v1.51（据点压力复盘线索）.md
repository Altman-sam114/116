# v1.51 据点压力复盘线索

## 版本号

v1.51

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.50。
- v1.50 已把据点压力地图标记补齐为 PRS/SRC/DEF 空间链路，本轮继续把当前压力和最近 AI 行动时间线做保守关联。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

玩家点选据点压力行后，地图已经能看到受压据点、威胁来源和守点目的格，但还难以判断“这条压力与上一轮 AI 行动复盘中哪些动作有关”。EasyTech 风格的大地图攻防需要把当前战线状态和敌方回合复盘串起来，但这类提示必须保守表达为“关联复盘线索”，不能宣称某条 AI 行动精确导致当前压力。

## 目标

1. `BattlefieldSituationObjectivePressure` 增加可选 `replayTarget`，从最新 `AIPhaseSummary.timeline` 纯派生关联复盘线索。
2. `GameState` 为当前点选压力提供只读 `focusedBattlefieldSituationObjectivePressureReplayTarget` 和定位入口。
3. 侧栏压力列表下方显示独立“复盘线索”按钮，点击后复用既有 AI 时间线定位路径。
4. 复盘线索必须使用“关联复盘 / 复盘线索 / 关联 AI 行动”等保守措辞，不写“导致压力”“造成压力”等因果措辞。
5. 增补 XCTest 和 rules smoke，覆盖线索派生、点选定位、无 summary/无匹配/过期压力以及只读边界。

## 非目标

- 不改变 AI 行动逻辑、AI 时间线记录逻辑、据点压力排序、敌方意图、反制建议、score、移动、攻击、胜负或回合规则。
- 不新增自动守点、自动移动、自动攻击、自动整补、自动部署或自动播放。
- 不在 SwiftUI 中计算 timeline 匹配、坐标有效性或压力线索。
- 不新增第三方库、资产管线、workflow、Xcode project 或长期分支策略。

## 架构依据

- `GameState.battlefieldSituationSummary` 已从威胁、反制建议、据点和 AI summary 纯派生战线态势。
- `BattlefieldSituationReplayTarget` 已作为战线态势关联 AI 复盘入口存在，可复用其 order/title/detail/coordinate 结构。
- `focusAIPhaseTimelineEvent(order:)` 已负责 AI 时间线重新校验、坐标定位、当前复盘 order 和地图复盘 marker 强调。
- `ContentView` 的压力行已经是按钮；本轮 UI 必须在压力列表下方增加独立按钮，避免按钮嵌套。

## 实现步骤

1. 在 `BattlefieldSituationObjectivePressure` 新增 `replayTarget: BattlefieldSituationReplayTarget?`：
   - 不建议把 `replayTarget` 纳入 `id`，避免 AI summary 变化导致当前压力行选中态无谓失效。
   - 构造 pressure 时传入从当前压力威胁组和最新 AI 时间线派生的目标。
2. 在 `GameState` 增加 helper，为单个受压据点和该压力的威胁列表选择关联 timeline event：
   - 优先匹配 `event.actorUnitID` 或 `event.targetUnitID` 命中当前压力威胁来源敌军 ID。
   - 其次匹配 `event.to == pressure.coordinate` 或 `event.from == pressure.coordinate`。
   - 事件类型优先 `objectiveCapture`、`move`、`attack`、`tacticalCommand`，同等条件保持 timeline 原始 order 稳定。
   - 定位坐标使用 `event.to ?? event.from`，必须存在且仍是当前地图格。
   - 标题和详情复用时间线信息，文案表达为“关联 AI 行动”或“复盘线索”。
3. 在 `GameState` 增加：
   - `var focusedBattlefieldSituationObjectivePressureReplayTarget: BattlefieldSituationReplayTarget?`：从当前压力 id 和最新 pressure 列表派生。
   - `func focusBattlefieldSituationObjectivePressureReplayTarget()`：重新校验当前压力和 replay target；失效时只写提示；有效时调用 `focusAIPhaseTimelineEvent(order:)` 并写保守提示。
4. 在 `ContentView` 的战线态势压力列表下方新增独立按钮：
   - 只在 `game.focusedBattlefieldSituationObjectivePressureReplayTarget != nil` 时显示。
   - 按钮 action 调用 `game.focusBattlefieldSituationObjectivePressureReplayTarget()`。
   - 标签可显示 `复盘线索`、`AI #order`、target title/detail，避免嵌套到压力行按钮内部。
   - 增加清晰 accessibility label/value，保持 SwiftUI 只转发状态。
5. 扩展 `GameStateTests.swift`：
   - 有匹配 AI timeline 时，当前 pressure 含 replay target，order/coordinate/title/detail 可校验。
   - 点选 pressure 后，调用压力复盘入口会更新 `focusedAIPhaseTimelineEventOrder` 和 `focusedCoordinate`。
   - 定位入口不得改变单位、据点、指令点、战报、latest result、AI summary、响应历史。
   - 无 AI summary、无匹配事件、过期 pressure id 或无当前压力时返回 nil 或只提示，不生成虚假定位。
6. 扩展 `RulesSmokeTest.swift` 同等主链路断言。
7. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 当前压力存在可关联 timeline event 时，压力列表下方显示独立复盘线索按钮。
- 点击按钮后复用既有 AI timeline 定位，地图强调对应 AI 复盘 marker，并保守提示“据点压力复盘线索”。
- 无当前压力、压力过期、无 AI summary、无匹配事件或坐标无效时不显示按钮或只写失效提示。
- UI 不嵌套按钮，不计算 timeline 匹配，不保存独立 replay target 状态。
- XCTest 和 rules smoke 覆盖线索派生、定位和只读边界。
- Agent B 在 `main` 以 `v1.51: 增加据点压力复盘线索` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。

## 风险和禁止项

- 禁止写因果措辞，避免把关联线索描述成“导致压力”。
- 禁止让复盘线索执行 MOVE、ATK、整补、部署、战术命令、endTurn 或 AI。
- 禁止在 SwiftUI 中反查单位、扫描 timeline 或新增独立压力线索 `@State`。
- 禁止改变据点压力排序、反制 score、敌方意图或 AI 行动来适配 UI。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端验收。
