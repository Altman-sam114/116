# v1.35（战线态势下一步提示）

## 版本号

v1.35

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新已完成版本为 v1.34（战线态势定位入口）。本轮继续在同一地图操作体验阶段递增小版本，围绕 v1.34 的首要定位目标补充“定位后下一步通过哪个既有入口执行”的解释。

## 背景

v1.34 已经让玩家从“战线态势”卡一键定位到首要反制、守点、OBJ 推进或待命单位。但定位后，玩家仍需要从地图标记、单位详情或反制建议面板中判断下一步应点 ATK、MOVE、整补还是先查看单位。EasyTech 风格战棋的战场提示应形成“态势 -> 定位 -> 入口”的闭环，同时必须避免把提示按钮变成自动执行。

## 目标

- 为 `BattlefieldSituationFocusTarget` 增加纯派生下一步行动提示，描述定位后应通过哪个既有入口继续操作。
- 反制目标应按反制类型提示 ATK、MOVE 或整补入口；OBJ 推进目标提示 MOVE；待命单位提示先选择并查看可用入口；守点目标提示派待命单位封堵或进驻。
- 在“战线态势”卡中展示下一步提示，文本紧凑、可读、支持 VoiceOver。
- 增加 XCTest 和规则 smoke test，证明下一步提示派生正确且不改变战场状态。

## 非目标

- 不新增自动执行按钮，不调用 `move`、`attack`、`useTacticalCommand`、`deploy`、`reinforce`、`executeFocusedCommand`、`endTurn` 或 AI 方法。
- 不改变 v1.34 的首要目标排序、反制建议排序、OBJ 计划排序、AI、战斗数值、据点归属、补给、士气或胜负规则。
- 不把下一步提示写进 SwiftUI 计算逻辑；UI 只能展示 `GameState`/`GameModels` 给出的字段。
- 不引入第三方库、网络依赖或资产管线。

## 当前架构依据

- `GameModels.swift` 定义纯数据模型，适合新增 `BattlefieldSituationActionHint` 和类型枚举。
- `GameState.swift` 已在 `battlefieldSituationPrimaryFocusTarget(threats:countermeasures:)` 中统一派生首要定位目标，适合同时填充下一步提示。
- `ContentView.swift` 的 `BattlefieldSituationSummaryView` 已展示定位按钮，适合在按钮内或紧邻按钮处展示下一步提示；按钮 action 仍只调用 `focusBattlefieldSituationPrimaryTarget()`。
- 现有 `EnemyThreatCountermeasureExecutionPreview` 是反制建议聚焦后的执行桥接，不能被态势 summary 直接依赖当前选中态；本轮应做轻量只读提示，不强行复用需要焦点状态的执行 preview。

## 实现步骤

1. 在 `GameModels.swift` 新增 `BattlefieldSituationActionHintKind` 和 `BattlefieldSituationActionHint`，至少包含类型、短标签、标题、入口文本、说明文本和是否当前可执行。
2. 扩展 `BattlefieldSituationFocusTarget`，增加 `actionHint` 字段，并更新稳定 `id`。
3. 在 `GameState` 的态势目标派生中填充 action hint：
   - `.firstStrike` -> ATK；
   - `.withdraw` / `.objectiveDefense` -> MOVE；
   - `.reinforce` -> 整补；
   - 受威胁据点 -> 防守/查看提示；
   - OBJ 推进 -> MOVE；
   - 待命单位 -> 选择/查看提示。
4. 在 `BattlefieldSituationSummaryView` 显示 action hint，使用 `Label` 或文本+短标签，保持按钮语义和无障碍文案完整。
5. 扩展 `GameStateTests.swift`，覆盖反制目标 action hint 与反制类型匹配、OBJ 目标 action hint 为 MOVE，并继续断言 summary 派生和定位只读不变性。
6. 扩展 `RulesSmokeTest.swift`，覆盖战线态势 action hint 的主链路字段和不变性。
7. 更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- `BattlefieldSituationFocusTarget.actionHint` 能随目标类型稳定给出 ATK、MOVE、整补、选择或防守查看提示。
- 战线态势卡显示下一步提示，按钮仍只执行定位。
- 查询 summary 和点击定位不改变单位 HP/位置、行动状态、据点归属、指令点、战报、结果摘要、AI 摘要、反制复核或胜负。
- XCTest 和 smoke test 覆盖 action hint 派生和只读不变性。
- 本地轻量检查通过，功能提交已 push 到 `origin/main`。
- Agent C 下载 artifact 后核对 manifest 的 `branch`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 一致，且静态检查、规则 smoke 和 Xcode build 成功。

## 风险和禁止项

- 禁止让态势卡按钮执行 ATK、MOVE、整补、部署、战术命令或结束回合。
- 禁止在 SwiftUI 中重新计算 action hint。
- 禁止让 action hint 依赖当前 UI 选中态而导致 summary 随焦点漂移到不一致入口。
- 禁止把旧 workflow run 或旧 artifact 冒充本轮云端验收。
- 禁止在非 `main` 分支提交、推送或创建 PR。
