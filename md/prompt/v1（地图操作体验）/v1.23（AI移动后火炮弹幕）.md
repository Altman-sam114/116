# v1.23 AI移动后火炮弹幕

## 版本号

v1.23

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.22（AI直取据点优先）。`origin/main` 最新验收 commit `4995d0fbb306833d283c21ae9bb1f8840edeb873` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.22-main-4995d0f-run28741253079-attempt1` 核对。v1.22 已让敌方 AI 在无击杀机会时优先夺取本回合可达据点；下一步应继续增强 AI 的兵种行为，让火炮在推进后能使用远程战术命令压制目标。本轮递增为 v1.23。

## 背景

当前 `runAxisAI()` 会在单位移动前检查 `bestTacticalCommandPlan(for:)`，但移动后收尾 `resolveAxisPostMoveAction(unitID:)` 只尝试普通攻击；若移动后的炮兵无法进行普通攻击却已经进入 `artilleryBarrage` 射程，AI 会直接待命。这让火炮推进缺少 EasyTech 风格的远程压制感，也浪费了既有火炮弹幕、AI timeline 和地图复盘能力。

本轮目标是把“移动后普通攻击不可用时尝试战术命令”接入现有 AI 收尾链，重点覆盖炮兵移动后火炮弹幕。实现应复用已有 `bestTacticalCommandPlan(for:)`、`useTacticalCommand`、`AIPhaseSummary.timeline` 和 `latestAIPhaseMapMarkers`，不新增一套战术命令规则。

## 目标

1. 增加 AI 移动后战术命令收尾：
   - 保持移动后普通攻击优先。
   - 当移动后没有普通攻击目标时，尝试 `bestTacticalCommandPlan(for:)`。
   - 若可用战术命令，执行 `useTacticalCommand` 并结束该单位行动。
   - 若仍无战术命令，保持现有待命/构筑防御姿态语义。
2. 重点覆盖炮兵移动后火炮弹幕：
   - 轴心火炮移动后若目标处于普通射程外、`artilleryBarrage` 射程内，且指令点足够，应使用火炮弹幕。
   - 时间线应形成 `move -> tacticalCommand`。
   - 地图 AI 复盘标记应包含移动起终点、战术命令行动单位和目标坐标。
3. 保持现有优先级：
   - 后勤仍最先执行。
   - 可击毁普通攻击仍先于移动和移动后战术命令。
   - 移动后普通攻击仍先于移动后战术命令。
   - v1.22 直取据点优先不退化。
4. 补充 XCTest 和规则 smoke test，覆盖：
   - 火炮移动后使用火炮弹幕。
   - 普通攻击可用时，移动后仍先普通攻击，不被战术命令抢走。
   - timeline、AI summary、战术命令结果和地图复盘标记同步正确。
5. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变火炮弹幕、突破突击、普通攻击、反击、伤害、士气、经验、地形、补给、控制区、移动力或胜负数值。
- 不改变移动前战术命令选择逻辑、战术命令评分公式或 `shouldUseTacticalCommand` 判定。
- 不让移动后战术命令抢在移动后普通攻击之前。
- 不新增概率 AI、搜索树、全局目标分配、撤退 AI、生产队列或复杂战略规划。
- 不修改 SwiftUI 交互、地图标记样式、侧栏布局、动画、音效或资源。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `GameState.runAxisAI()` 是轴心国 AI 行动顺序的唯一落点。
- `resolveAxisPostMoveAction(unitID:)` 是 AI 移动后攻击或待命的收尾 helper；本轮应优先在这里接入移动后战术命令，避免在多处分叉重复逻辑。
- `bestTacticalCommandPlan(for:)` 已根据当前单位位置、指令点、目标价值和 `shouldUseTacticalCommand` 选择战术命令。
- `useTacticalCommand` 已负责扣指令点、伤害、士气、状态、结果摘要、AI timeline 和 AI action count。
- `AIPhaseTimelineEvent` 已支持 `.move` 和 `.tacticalCommand`，`latestAIPhaseMapMarkers` 已从 timeline 纯派生地图复盘标记。
- `TacticalCommand.artilleryBarrage.range == 4`，火炮普通射程为 `UnitKind.artillery.range == 3`，可构造“移动后普通攻击不可用但弹幕可用”的小场景。

## 实现步骤

1. 修改 `GameState.resolveAxisPostMoveAction(unitID:)`：
   - 先按现有逻辑刷新单位并尝试 `bestAttackTarget(for:)`。
   - 若普通攻击目标存在，执行 `attack` 和既有 `advanceAfterManeuverPursuitIfPossible`，保持现有优先级。
   - 若没有普通攻击目标，再刷新或复用当前单位调用 `bestTacticalCommandPlan(for:)`。
   - 若战术命令计划存在，调用 `useTacticalCommand(plan.command, casterID: refreshed.id, targetID: plan.target.id)`。
   - 只有普通攻击和战术命令都不存在时，才进入现有待命/构筑防御姿态分支。
2. 确认 `bestTacticalCommandPlan(for:)` 在移动后可用：
   - 它当前只排除 `hasAttacked`，而 `move` 只会标记 `hasMoved`，因此移动后的火炮仍可执行火炮弹幕。
   - 不要放宽 `tacticalCommandTargets` 或 `useTacticalCommand` 的合法性；若当前规则拒绝，应让测试暴露。
3. 新增 XCTest 场景 helper，例如 `axisPostMoveBarrageScenario()`：
   - 小地图使用连续 road/plain，保证轴心火炮本回合能移动到目标 4 格内。
   - 目标保持在火炮普通射程 3 外、火炮弹幕射程 4 内。
   - 轴心指令点足够执行 `artilleryBarrage`，但不触发部署或整补干扰。
   - 避免目标被移动前战术命令直接命中；初始距离应大于 4。
4. XCTest 断言：
   - `endTurn()` 后轴心火炮位置变为预期移动目的地。
   - `latestTacticalCommandResult?.command == .artilleryBarrage`。
   - 目标 HP 下降、士气下降或状态进入压制，且没有普通反击伤害。
   - `latestAIPhaseSummary.moves == 1`，`tacticalCommands == 1`，`attacks == 0`。
   - timeline 顺序包含 `move -> tacticalCommand`。
   - `latestAIPhaseMapMarkers` 包含移动 destination 和 tactical target 标记。
5. 增加或保留一个“普通攻击仍优先”的断言：
   - 可复用现有 full advance / moving attack 场景。
   - 断言移动后普通攻击可用时，AI summary 的 `attacks` 增加，`tacticalCommands` 不因为移动后分支误增。
6. `RulesSmokeTest.swift` 增加同类 smoke 场景和断言，覆盖移动后火炮弹幕主链路。
7. 文档同步：
   - `WW2Tactics/README.md` AI 描述加入“移动后若普通攻击不可用，火炮可在弹幕射程内压制目标”。
   - `md/flow/flow.md` AI 执行流和 AI 回合摘要段落加入移动后战术命令收尾。
   - `md/flow/flowchart.md` 在 AI 状态图中标出普通移动后可接普通攻击或战术命令。
   - `md/test/test.md` 当前 smoke / XCTest 基线加入 AI 移动后火炮弹幕。
   - `update_log.md` 追加 v1.23 记录，本地验证后写具体命令结果，Agent C 验收后补云端结果。

## 验收标准

1. 轴心火炮在移动后普通攻击不可用但火炮弹幕可用时，会执行 `artilleryBarrage`。
2. 移动后普通攻击仍优先于移动后战术命令。
3. 移动后战术命令复用 `useTacticalCommand`，不绕过指令点、伤害、士气、状态、行动消耗和结果摘要。
4. AI summary 记录 `moves == 1`、`tacticalCommands == 1`、`attacks == 0` 的移动后弹幕场景。
5. 时间线形成 `move -> tacticalCommand`，地图复盘标记能回连移动和战术命令目标。
6. v1.22 直取据点优先、可击毁攻击优先、后勤优先和既有移动后普通攻击测试不退化。
7. 本地轻量检查通过。

## 风险和禁止项

- 风险：移动后战术命令若放在普通攻击前，会破坏既有移动后攻击行为；必须先查普通攻击。
- 风险：`bestTacticalCommandPlan` 可能选择突破突击等非炮兵命令；本轮测试重点锁住炮兵弹幕，且不要改变现有战术命令评分。
- 风险：新增测试场景若初始距离小于等于 4，会被移动前战术命令抢先执行；必须让初始距离超出弹幕射程。
- 风险：后勤部署或整补可能干扰新测试场景；测试应控制指令点、据点归属和单位 HP。
- 禁止在 View 中实现 AI 决策。
- 禁止绕过 `useTacticalCommand` 直接修改 HP、士气、指令点或 timeline。
- 禁止改变伤害、移动、补给、士气、部署、整补、胜负和反制建议语义。
- 禁止新增不可测试的 AI 状态或全局策略缓存。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
