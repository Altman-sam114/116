# v1.32 反制复核结论定位

## 版本号

v1.32

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.31（据点推进优先级解释）。`origin/main` 最新验收 commit `c40ccbf6dd423c6a6eb28a0aedef4dcb64e2689a` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.31-main-c40ccbf-run28776225924-attempt1` 核对。v1.31 已增强 OBJ 决策解释；下一步应把反制建议的“执行 -> 敌方回合复核”闭环做得更像战棋战报：玩家能一眼判断反制是否奏效，并能快速定位执行单位、威胁源和受威胁目标，因此本轮递增为 v1.32。

## 背景

项目已有敌方意图、反制建议、执行回放和敌方回合后复核。当前复核卡展示结论和对照，但缺少结构化等级，也缺少从复核结果直接定位相关地图对象的入口。EasyTech 风格的大地图战棋需要让玩家把战报结论快速映射回地图，本轮补上只读结论等级和定位入口。

## 目标

1. 给 `EnemyThreatCountermeasureFollowUpSummary` 增加纯派生复核结论等级：
   - 区分奏效、部分奏效、失败。
   - 从既有 `conclusion` 和 `comparisons` 派生，不改变 AI、反制建议、执行回放或复核生成逻辑。
2. 给复核结果增加只读定位目标：
   - 至少包含执行单位、威胁来源和受威胁目标/据点三类入口。
   - 定位目标只保存目标类型、标题、坐标或单位 ID，不新增不可测试 UI 状态。
3. 在 `GameState` 增加复核定位入口：
   - 入口从当前 `latestEnemyThreatCountermeasureFollowUpResult` 重新解析目标。
   - 成功时只更新 `focusedCoordinate` 和 `message`，不移动、不攻击、不整补、不改变 AI summary 或 follow-up。
   - 目标过期或坐标不在地图时只写提示。
4. `ContentView` 的敌方回合复核卡展示结论等级徽标，并提供紧凑定位按钮。
5. 扩展 XCTest 和 smoke test，覆盖结论等级、定位目标、定位只读边界和重开清理。
6. 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不改变敌方 AI 决策、AI 时间线、AI 复盘播放、战斗数值、移动、补给、士气、据点、部署、整补、战术命令或胜负。
- 不改变敌方意图、反制建议排序、score、收益解释、排序对比、执行入口桥接或执行回放语义。
- 不新增历史复核列表、自动镜头、动画、音效、真实美术资产、第三方库或网络依赖。
- 不让 SwiftUI 自行推导复核等级、地图坐标或目标有效性。

## 当前架构依据

- `EnemyThreatCountermeasureFollowUpSummary` 已保存 countermeasure id/kind、执行类型、执行单位、目标单位、威胁来源、据点坐标、AI 回合和 comparisons。
- `GameState` 已在 AI 回合后发布 `latestEnemyThreatCountermeasureFollowUpResult`，并在重开、切战役或新执行路径清理旧复核。
- `ContentView.EnemyThreatCountermeasureFollowUpSummaryView` 已展示复核卡，适合接入派生等级和按钮。
- 架构边界要求核心定位和有效性校验放在 `GameState`，UI 只展示字段并转发意图。

## 实现步骤

1. 修改 `GameModels.swift`：
   - 新增 `EnemyThreatCountermeasureFollowUpOutcomeLevel`。
   - 新增 `EnemyThreatCountermeasureFollowUpFocusTargetKind` 和 `EnemyThreatCountermeasureFollowUpFocusTarget`。
   - 在 `EnemyThreatCountermeasureFollowUpSummary` 上增加 `outcomeLevel`、`outcomeTitle`、`focusTargets` 等纯派生属性。
2. 修改 `GameState.swift`：
   - 新增 `focusEnemyThreatCountermeasureFollowUpTarget(_:)`。
   - 入口基于当前 summary 的目标重新定位单位或坐标，只更新地图焦点和提示。
3. 修改 `ContentView.swift`：
   - 复核卡显示 outcome badge。
   - 渲染 `summary.focusTargets` 为按钮，点击调用 `game.focusEnemyThreatCountermeasureFollowUpTarget(target)`。
4. 修改测试：
   - XCTest 在现有 `testEnemyThreatCountermeasureFollowUpPublishesAfterEnemyTurn` 中断言 outcome、focusTargets 和定位只读不变性。
   - smoke test 在 `verifyCountermeasureFollowUpVerification()` 中做同样覆盖。
5. 文档同步并追加版本记录。

## 验收标准

1. 敌方回合复核卡能显示奏效/部分奏效/失败等级。
2. 复核卡能定位执行单位、威胁来源和受威胁目标/据点。
3. 定位入口只更新地图焦点和消息，不执行命令，不改变单位、据点、指令点、AI summary、follow-up 或战报。
4. SwiftUI 不自行推导复核等级、坐标或目标有效性。
5. 本地轻量检查通过。
6. Agent B 在 `main` 提交并 `git push origin main`。
7. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 manifest、summary、JUnit、rules-smoke、xcodebuild 和 xcresult。

## 风险和禁止项

- 风险：复核等级若重新模拟 AI 会与真实 AI 结果失步；必须只从现有复核字段派生。
- 风险：UI 若自行定位单位或坐标，可能绕过过期校验；必须调用 `GameState` 入口。
- 禁止改变 AI、移动、攻击、整补、部署、战术命令、据点、胜负、反制建议排序和执行回放语义。
- 禁止新增不可测试 UI-only 状态。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
