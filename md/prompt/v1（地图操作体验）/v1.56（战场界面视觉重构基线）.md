# v1.56 战场界面视觉重构基线

## 版本号

v1.56

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.55。v1.55 已完善据点压力来源标识；人工新目标要求在新模型阶段开始重构此前偏粗糙的游戏界面，因此本轮在同一地图操作体验阶段递增为 v1.56。

## 背景

当前界面已经具备完整地图、顶栏、HUD、侧栏和大量战术信息，但视觉语言仍偏原型：深色渐变、面板层级和状态芯片缺少统一战场指挥台风格。下一阶段需要先建立可持续扩展的视觉基线，让后续战斗、单位、据点和复盘 UI 都能沿同一体系迭代。

## 目标

- 重构首屏外壳：顶栏、地图容器、HUD 背板、状态芯片和侧栏容器形成统一战区指挥台风格。
- 保持当前所有地图操作入口和 `GameState` 调用链不变。
- 用共享的 SwiftUI 主题常量/表面样式减少重复色值和零散描边。
- 保持操作型战棋界面密度，不做营销页、静态展示页或装饰性大图。
- 本轮按人工要求不跑本地测试；提交后只用 GitHub Actions 云端结果包验收。

## 非目标

- 不改变移动、攻击、AI、据点、反制建议、战线态势或任何核心规则。
- 不引入新素材、第三方库、复杂资产管线、动画系统或真实美术资源。
- 不重写全部 `ContentView` 子面板；本轮只建立 UI 视觉基线和最关键外壳。
- 不创建 PR，不使用非 `main` 分支。

## 当前架构依据

- `ContentView.swift` 负责 SwiftUI 表现层，允许调整布局、颜色、面板样式和视觉层级。
- `GameState.swift` 保持规则状态机边界，本轮不应修改核心规则。
- README、flow、flowchart、test、update_log 必须同步用户可见行为和验证策略变化。

## 实现步骤

1. 在 `ContentView.swift` 增加轻量主题常量和通用表面样式，覆盖背景、面板、边框、状态芯片和 HUD。
2. 重构 `ContentView` 外层背景和宽/窄布局间距，让地图和侧栏成为首屏核心。
3. 重构 `TopCommandBar`、`CommandTitle`、`StatusChip` 和 `EndTurnButton`，增强战役身份、回合状态和操作按钮层级。
4. 重构 `BattlefieldView`、`MapCommandCenter`、`MapToolbar`、`MapHudBackground` 和 `InspectorPanel` 的容器样式，统一圆角、描边、半透明面板和信息密度。
5. 保持所有按钮、Picker、ScrollView、地图滚动、HUD 数据和 GameState 方法调用不变。
6. 更新 README、flow、flowchart、test、prompt README 和 update_log。
7. 提交并 `git push origin main`，等待 GitHub Actions 生成结果包。
8. Agent C 下载 artifact，核对 manifest、failure summary、JUnit、rules smoke log、xcodebuild log 和 xcresult。
