# v1.57 地图格与单位棋子视觉升级

## 版本号

v1.57

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.56。v1.56 已建立首屏战区指挥台视觉基线；本轮继续沿 UI 重构目标推进，把最核心的地图格和单位棋子从原型色块升级为更清晰的战棋视觉。

## 背景

用户明确要求新模型阶段重构此前偏丑的游戏界面。v1.56 已统一外壳、顶栏、地图容器、HUD 和侧栏容器，但地图格本身仍偏平面色块，单位棋子仍像调试标牌。EasyTech 风格战棋的核心观感首先来自地图材质、据点归属、单位棋子和战术标记的可读层级。

## 目标

- 升级 `HexTileView` 的地图格表现：地形渐变、内阴影/高光、据点归属层、焦点与行动边框层级更清晰。
- 升级 `TerrainSymbol` / `TerrainCodeBadge` / `ObjectiveNamePlate`，让地形、据点名和归属更像地图符号而不是调试标签。
- 升级 `UnitShapeBadge`，给单位棋子增加高光、底影、阵营边框和状态层级，但保留原有兵种形状、代码和无障碍描述。
- 保持所有地图点击、滚动、标记、HUD 和 `GameState` 调用链不变。
- 本轮按人工要求不跑本地测试；提交后只用 GitHub Actions 云端结果包验收。

## 非目标

- 不改变移动、攻击、AI、据点、反制建议、战线态势或任何核心规则。
- 不引入图片资产、第三方库、动画系统或复杂美术管线。
- 不重写所有地图标记；本轮只升级地图格基底、地形符号和单位棋子。
- 不处理未纳入本轮的本地 `project.pbxproj` 改动。

## 当前架构依据

- `ContentView.swift` 负责 SwiftUI 表现层，允许调整地图格、单位棋子、地形符号和面板样式。
- `GameState.swift` 和 `GameModels.swift` 保持规则和数据边界，本轮不应修改核心规则。
- README、flow、flowchart、test、update_log 必须同步用户可见 UI 变化和云端验证策略。

## 实现步骤

1. 在 `ContentView.swift` 为地形格增加共享的地图材质 helper，例如地形渐变、高光、低光和据点归属覆盖。
2. 调整 `HexTileView` 的 base layer：从单色填充升级为地形渐变 + 轻微高光/阴影 + 当前状态边框。
3. 调整 `TerrainSymbol`、`TerrainCodeBadge`、`ObjectiveNamePlate` 的字体、背板、描边和透明度，让它们和 v1.56 战区主题一致。
4. 调整 `UnitShapeBadge`：保留 `UnitMarkerShape`，增加底影、阵营渐变、高光和状态徽标层级。
5. 保持所有 `Button`、`HexInputReader`、地图标记、accessibility label 和 `GameState` 方法调用不变。
6. 更新 README、flow、flowchart、test、prompt README 和 update_log。
7. 提交并 `git push origin main`，等待 GitHub Actions 生成结果包。
8. Agent C 下载 artifact，核对 manifest、failure summary、JUnit、rules smoke log、xcodebuild log 和 xcresult。
