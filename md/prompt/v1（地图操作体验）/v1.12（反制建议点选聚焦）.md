# v1.12 反制建议点选聚焦

## 版本号

v1.12

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.11，本轮继续围绕地图操作体验和 EasyTech 风格战前决策辅助递增为 v1.12。
- v1.11 已实现敌方意图反制建议，但建议仍是静态文本。本轮把建议行变成可点选的地图预览入口，让玩家能快速定位执行单位、敌军、撤退格或防守格。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类二战回合制战棋体验。当前侧栏已经能展示敌方意图和反制建议，但玩家看到“抢先打击”“撤出危险区”“据点防守”“整补支撑”后，还需要手动在地图上寻找对应单位和格子。

本轮要延续已有 OBJ 目标计划和安全接敌候选的交互模式：点选建议只切换选中单位、地图焦点和相关预览，不消耗行动、不移动、不攻击、不整补。

## 目标

1. 在 `GameState` 新增反制建议聚焦入口，建议命名为：
   - `focusEnemyThreatCountermeasure(_ preview: EnemyThreatCountermeasurePreview)`
2. 聚焦行为必须按建议类型区分：
   - 抢先打击：选中执行单位，聚焦威胁来源敌军；若仍可攻击，地图应自然显示 ATK 预览。
   - 撤出危险区：选中被威胁单位，聚焦建议目的格；若路线仍合法，地图应自然显示 MOVE 预览和路线风险。
   - 据点防守：选中执行单位，聚焦建议目的格；若目标据点和目的格不同，保留 `guidedObjectiveCoordinate` 指向被威胁据点，方便地图显示最终目标。
   - 整补支撑：选中受威胁单位，聚焦其当前位置，提示可以整补；不调用真实整补。
3. 聚焦必须重新校验当前状态：
   - 执行单位仍存在、属于当前阵营且未被摧毁。
   - 敌军目标、撤退路线、守点路线或整补条件若已不可用，给出明确消息并只做安全聚焦或清理引导。
4. 聚焦只允许改变 UI 预览状态：
   - `selectedUnitID`
   - `focusedCoordinate`
   - `guidedObjectiveCoordinate`
   - `focusedSafeEngagementTargetID` / `focusedSafeEngagementDestination`
   - `message`
5. 聚焦禁止改变单位 HP、位置、行动状态、据点归属、指令点、战报、胜负、AI 摘要或任何 `latest*Result`。
6. `ContentView` 将“反制建议”行改为 `Button`，使用 `.buttonStyle(.plain)`，并提供清晰的 accessibility label/hint：
   - label 应包含建议类型、执行单位、目标/目的地、当前是否已聚焦。
   - hint 应说明“只聚焦预览，不执行命令”。
7. 行 UI 应能标记当前聚焦的建议，例如边框、背景或短文本状态，不能只靠颜色区分。
8. 补充 XCTest 和规则 smoke test，覆盖：
   - 抢先打击建议点选后选中执行单位、聚焦敌军，并保持 HP/位置/战报/结果摘要不变。
   - 撤退建议点选后选中单位、聚焦目的格，路线仍可查。
   - 据点防守建议点选后选中单位、聚焦目的格，并设置 `guidedObjectiveCoordinate`。
   - 整补建议点选后选中单位、聚焦当前位置，不扣指令点、不改变 HP。
   - 过期建议或不合法建议不会执行动作，状态不会被破坏。
9. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不新增一键执行建议。
- 不自动移动、攻击、整补、部署或使用战术命令。
- 不改变 v1.11 反制建议生成算法、敌方意图预判、AI 行为、战斗数值、移动规则、整补成本、胜负条件。
- 不新增地图 marker 类型；本轮继续复用现有 MOVE/ATK/OBJ/INT 等预览语义。
- 不引入第三方库、网络依赖、美术资产、动画或音效。

## 当前架构依据

- `GameState` 已有 `focusObjectiveAdvancePreview(_:)`、`focusObjectiveAdvanceTarget(coordinate:)`、`focusSafeEngagementOption(_:)` 和 `focusSafeEngagement(targetID:destination:)`，它们提供“点选侧栏候选，只切换地图预览，不执行命令”的本地模式。
- `ContentView` 中目标计划和安全接敌候选已经用 `Button + .buttonStyle(.plain)` 实现可访问的点选预览，本轮应沿用该模式，避免 `onTapGesture`。
- `EnemyThreatCountermeasurePreview` 已包含建议类型、执行单位、敌军/目标、目的格、路线消耗、预计效果和关联威胁字段，足以支撑聚焦。
- `ContentView` 只能调用 `GameState` 聚焦方法，不得在 View 中重新校验路线、战斗或整补规则。

## 实现步骤

1. 在 `GameState.swift` 新增 `focusEnemyThreatCountermeasure(_:)`。
2. 为不同建议类型拆出私有聚焦 helper，或在一个方法中清晰分支：
   - 聚焦前清理安全接敌焦点。
   - 对据点防守设置 `guidedObjectiveCoordinate = preview.threatTargetCoordinate`。
   - 其他建议默认清理 `guidedObjectiveCoordinate`。
3. 聚焦时重新查当前单位和路线：
   - 抢先打击重新确认执行单位可攻击威胁来源；失败时聚焦敌军并提示建议已过期。
   - 撤退/据点防守重新确认 `movementRoute(for:to:)` 仍存在；失败时聚焦执行单位并提示路线不可用。
   - 整补重新确认 `canReinforce(_:)`；失败时提示整补条件不满足。
4. `ContentView.swift`：
   - `EnemyThreatCountermeasurePanel` 增加 `@EnvironmentObject private var game: GameState`。
   - 每条建议用 `Button` 包裹 row，点击调用 `game.focusEnemyThreatCountermeasure(preview)`。
   - 给 row 增加 `isFocused` 参数，显示聚焦状态。
   - 增加 accessibility label/hint。
5. `GameStateTests.swift`：
   - 复用 `enemyThreatIntentScenario(axisSpent:)`。
   - 新增测试点选四类反制建议后的选中、焦点、引导和无副作用。
6. `RulesSmokeTest.swift` 补充高信号断言，覆盖主规则链。
7. 更新文档：
   - `WW2Tactics/README.md`：反制建议可点选聚焦。
   - `md/flow/flow.md`：反制建议聚焦流和只读边界。
   - `md/flow/flowchart.md`：地图命令执行流加入反制建议点选。
   - `md/test/test.md`：当前基线加入反制建议点选聚焦。
   - `update_log.md`：追加 v1.12 记录。

## 验收标准

- 反制建议行可点选，点选后能聚焦对应执行单位和目标/目的格。
- 点选抢先打击、撤退、据点防守、整补四类建议后，地图能自然显示相应 ATK/MOVE/OBJ 或单位聚焦语义。
- 点选建议不会移动、攻击、整补、扣指令点、改变 HP、写战报或生成结果摘要。
- 过期建议不会破坏状态，并给出明确提示。
- UI 使用 Button 而不是 `onTapGesture`，有 accessibility label/hint，当前聚焦状态不只靠颜色。
- XCTest 和 smoke test 覆盖四类建议点选、无副作用和过期建议。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止把反制建议点选做成一键执行。
- 禁止在 `ContentView` 中重新计算路线、攻击、整补或建议可行性。
- 禁止调用 `move`、`attack`、`deploy`、`reinforce`、`useTacticalCommand`、`advanceTurn`、`performAIPhase` 等写状态方法。
- 禁止改变 AI、战斗数值、移动规则、部署/整补成本、胜负条件。
- 禁止新增不可测试的 UI 状态或让建议聚焦状态只能通过颜色识别。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
