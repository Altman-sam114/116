# v1.60（AI战况回放界面重构）

## 版本号

v1.60

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.59。v1.56-v1.59 已完成战场外壳、地图/棋子、攻击反馈和战线态势简报重构。本轮继续递增，聚焦敌方回合摘要、复盘结论、关键事件、时间线和播放控制。

## 背景

当前 AI 回合复盘已经具备结构化摘要、结论、关键事件、地图定位、上一条/下一条、自动播放和速度控制，但 UI 仍以通用指标格和密集文本列表为主。复盘控制、战果结论和逐条行动层级接近；关键事件与时间线行点击区域偏小；自动播放进入第 6 条以后，地图会继续定位，但侧栏只显示前 5 条，缺少明确的当前回放反馈。

## 目标

- 将 `AIPhaseSummaryView` 重构为战况回放面板：回合头部、战果指标、结论、回放控制、当前事件、时间线和尾注层级清晰。
- 新增 AI 专属指标 View，不修改高复用 `CombatResultMetric`。
- 保留 `summary.timeline.prefix(5)` 的时间线摘要，同时在存在焦点 order 时显示独立当前回放事件，确保播放到隐藏事件后侧栏仍有反馈。
- 重构 `AIPhaseReplayConclusionView`，强化结论、四项派生指标和关键事件。
- 将关键事件和时间线按钮点击区提升到至少 44pt，并保留原无障碍 label/hint。
- 将播放/暂停、速度、上一条、下一条放入独立控制带，保持窄侧栏可用。
- 复用 `BattlefieldTheme` 和 AI 局部语义色，不修改全局主题 token。

## 非目标

- 不修改 `GameModels.swift`、`GameState.swift`、AI 行为、时间线记录、结论分类、关键事件优先级、marker 生成或播放状态机。
- 不修改 `InspectorPanel` 的显示顺序、宽度或滚动结构。
- 不改变 Timer publisher、播放 interval、active 判断或 `advanceAIPhaseTimelinePlayback()` 调用。
- 不改变前 5 条摘要规则、事件顺序、关键事件 order 或地图定位坐标。
- 不修改 `CombatResultMetric`、地图 `AIPhaseMapReplayMarker` 或外部态势/复核入口。
- 不新增动画、音效、第三方依赖、图片资产或独立播放状态。

## 当前架构依据

- `latestAIPhaseSummary`、`focusedAIPhaseTimelineEventOrder`、`isAIPhaseTimelinePlaybackActive` 和 `aiPhaseTimelinePlaybackPace` 由 `GameState` 唯一管理。
- 所有复盘入口最终调用 `focusAIPhaseTimelineEvent(order:)`，只改变地图焦点、复盘 order 和消息，不执行游戏命令。
- 播放按钮 disabled 语义为 `!canPlayAIPhaseTimeline && !isAIPhaseTimelinePlaybackActive`，保证播放中始终可暂停。
- Timer 只在 active 时调用 `advanceAIPhaseTimelinePlayback()`；到最后一条自动暂停。
- `AIPhaseReplayConclusion`、metrics 和 keyEvents 均由模型纯派生，View 不得重算分类或排序。

## 实现步骤

1. 重构 `AIPhaseSummaryView`：
   - 保持 `visibleTimeline = prefix(5)` 和 hidden count。
   - 新增只读隐藏焦点事件派生，仅当当前 order 不在前五条时从 summary.timeline 查找并显示当前回放条，避免重复展示和朗读。
   - 保留 `.onReceive` publisher、active 条件和 advance 调用原样。
2. 新增 AI 专属头部和指标组件：
   - 显示阵营、回合、行动、伤害、承伤、移动、攻击、战术、后勤、占点和指令点变化。
   - 不修改或复用改造 `CombatResultMetric`。
3. 重构 `AIPhaseReplayConclusionView`：
   - 保持 conclusion、metrics、keyEvents 输入和 `focusedOrder`。
   - 关键事件仍只把 order 转发给 `focusAIPhaseTimelineEvent(order:)`。
   - 关键事件点击区至少 44pt，取消容易截断的固定 58pt 标题列。
4. 新增回放控制带：
   - 保留现有播放/暂停、速度、上一条、下一条 action 和 disabled 条件。
   - 保留速度菜单读取/写入 `GameState`，不存本地 pace。
5. 重构 `AIPhaseTimelineEventRow`：
   - 保留 order、shortCode、summary 和当前态。
   - 最小点击高度 44pt，改善窄侧栏和 Dynamic Type。
   - wrapper 继续使用 `.accessibilityElement(children: .ignore)` 及完整 label/hint。
6. 同步 README、flow、flowchart、test、prompt README 和 update_log。

- AI 摘要、结论、控制和时间线形成清晰战况回放层级。
