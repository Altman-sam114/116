# v1.34（战线态势定位入口）

## 版本号

v1.34

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新已完成版本为 v1.33（战线态势汇总）。本轮继续在同一地图操作体验阶段递增小版本，围绕 v1.33 的战线态势卡补一个可验证、可提交、可云端验收的定位入口。

## 背景

v1.33 已经把当前阵营的指令点、待命部队、据点进度、敌方意图、可执行反制、受威胁据点和首要建议汇总到侧栏。但玩家看到首要建议后仍需要手动在地图、反制建议或 OBJ 计划中寻找目标。EasyTech 风格的大地图战棋需要更直接的“从态势提示跳到战场热点”的操作闭环。

## 目标

- 为战线态势汇总增加一个纯派生的首要定位目标模型，能描述本轮态势建议应该跳转到哪里。
- 在 `GameState` 中增加定位入口，按当前最新态势重新选择目标并聚焦地图。
- 目标优先级应与 v1.33 的态势建议一致：可执行反制优先，其次受威胁据点，其次待命单位的 OBJ 推进计划，其次第一支待命单位，最后为无可定位目标。
- 在侧栏“战线态势”卡增加紧凑按钮，让玩家可以直接定位首要建议。
- 增加 XCTest 和规则 smoke test，证明定位入口只改变选择、焦点、引导和消息，不消耗行动、不移动单位、不攻击、不改变据点、指令点、战报、结果摘要或胜负。

## 非目标

- 不新增自动执行按钮，不调用 `move`、`attack`、`useTacticalCommand`、`deploy`、`reinforce`、`endTurn` 或 AI 方法。
- 不改变敌方意图、反制建议、OBJ 计划、安全接敌、AI、战斗数值、据点归属、补给或胜负规则。
- 不改变 v1.33 的态势建议排序语义，除非为了复用同一优先级暴露定位目标。
- 不引入第三方库、网络依赖、资产管线或新的长期分支。

## 当前架构依据

- `GameModels.swift` 定义纯数据模型，应放置 `BattlefieldSituationFocusTarget` 一类只读结构和定位类型枚举。
- `GameState.swift` 是唯一规则状态机，应负责从当前状态派生态势定位目标，并提供 `focusBattlefieldSituationPrimaryTarget()` 之类的入口。
- `ContentView.swift` 只能展示模型字段并把按钮点击转发给 `GameState`，不能在 View 中重算威胁、OBJ 计划或定位有效性。
- 现有 `focusEnemyThreatCountermeasure(_:)`、`focusObjectiveAdvancePreview(_:)`、`focusObjectiveAdvanceTarget(coordinate:)`、`select(unitID:)`、`focus(coordinate:)` 和目标引导清理逻辑应尽量复用。

## 实现步骤

1. 在 `GameModels.swift` 增加战线态势定位目标模型，至少包含类型、标题、说明、坐标和可选单位 ID/反制建议/OBJ 信息；需要 `Equatable`，并提供稳定 `id` 和用于 UI 的短标签。
2. 扩展 `BattlefieldSituationSummary`，增加可选 `primaryFocusTarget` 字段，使 UI 能知道是否显示定位按钮。
3. 在 `GameState.battlefieldSituationSummary` 中按当前态势和既有派生数据填充 `primaryFocusTarget`。该 computed property 仍必须只读，不调用真实执行路径。
4. 在 `GameState` 中增加 `focusBattlefieldSituationPrimaryTarget()`，每次调用都基于当前最新态势重新派生定位目标。若目标过期或不可定位，只写提示。
5. 定位行为建议：可执行反制复用 `focusEnemyThreatCountermeasure(_:)`；受威胁据点聚焦据点坐标并设置或清理必要引导；OBJ 推进复用 `focusObjectiveAdvancePreview(_:)`；待命单位选择并聚焦该单位。所有路径都不得消耗行动或改变真实规则结果。
6. 在 `ContentView.BattlefieldSituationSummaryView` 添加 `@EnvironmentObject GameState` 或从父层传入 action，按钮使用 `Button` 并带文本 label 与 system image，遵守 VoiceOver 可读性。按钮文案应紧凑，例如“定位”。
7. 扩展 `GameStateTests.swift`：覆盖有可执行反制时的目标派生和定位、有 OBJ 推进时的兜底定位、无待命/已结算时的不可定位提示或 resolved 目标，并断言只读不变性。
8. 扩展 `RulesSmokeTest.swift`：覆盖主规则链中的态势定位入口，至少验证反制定位和 OBJ 兜底定位。
9. 更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- `BattlefieldSituationSummary` 能给出与首要建议一致的 `primaryFocusTarget`。
- 侧栏战线态势按钮点击后能定位反制、受威胁据点、OBJ 推进或待命单位中的首要目标。
- 定位入口不改变单位 HP/位置、行动状态、据点归属、指令点、战报、结果摘要、AI 摘要、反制复核或胜负。
- XCTest 和 smoke test 覆盖目标派生、定位行为和只读不变性。
- 本地轻量检查通过，功能提交已 push 到 `origin/main`。
- Agent C 下载 artifact 后核对 manifest 的 `branch`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 一致，且静态检查、规则 smoke 和 Xcode build 成功。

## 风险和禁止项

- 禁止在 View 中计算定位目标或调用真实执行方法。
- 禁止为了定位而修改战斗、移动、OBJ、反制建议排序或 AI 规则。
- 禁止把旧 workflow run 或旧 artifact 冒充本轮云端验收。
