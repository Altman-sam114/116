# v1.61 地图标记拥挤度治理

## 版本号

v1.61

## 版本分配依据

上一正式版本为 v1.60（AI 战况回放界面重构）。本轮继续 `v1（地图操作体验）` 小版本递增，处理 prompt README 与 update_log 明确遗留的“地图标记拥挤度”问题。

## 背景

`HexTileView` 在单格上叠加了多类短码/徽标：命令 MOVE/ATK/POS、路线、威胁 THR/火力、目标 OBJ/CAP/INT、态势反制/压力/响应、AI 复盘。当前仅有局部 `*MarkerTopPadding` / `aiPhaseMarkerBottomPadding` 硬编码错位，同格多标记时会互相遮挡单位棋子/据点铭牌，并与行动短码抢位。

## 目标

在不改规则层前提下，重构 `ContentView` 地图格标记布局：

1. 为 `HexTileView` 建立统一 marker 槽位（top stack / bottom stack / corner icons），替换分散 ad-hoc padding。
2. 显示优先级：可执行命令标记（MOVE/ATK/POS）与不可用提示始终角标；焦点辅助（反制/压力/响应）优先于火力/OBJ/CAP/INT；一般 AI 与状态点入底栈或角落。
3. 顶/底栈各最多显示 2 个徽标，溢出显示 `+N`。
4. 单位棋子 / 据点名牌 / 地形代码保留中下部主视区。
5. 无障碍文案仍完整列出全部标记语义。
6. 同步 README / flow / flowchart / test / prompt README / update_log。

## 非目标

- 不修改 `GameModels.swift`、`GameState.swift` 或 marker 生成/聚焦规则。
- 不改变点击命中与命令执行链。
- 不新增 `@Published` 状态。
- 不做命令预览去重（v1.62）。
- 不新增动画、音效、第三方库、图片资产。
- 不跑本地 Xcode / Simulator；验证全部走 GitHub Actions。

## 当前架构依据

- 所有 marker 数据已由 `GameState` 派生并传入 `HexTileView`。
- 同类 marker 视图已支持同坐标 role 合并。
- `borderColor` / `borderWidth` 已有优先级；布局优先级应与之语义一致。
- `accessibilityLabel` 已拼接各标记文案，折叠不得删减。

## 实现步骤

1. 在 `ContentView.swift` 内新增只读布局 helper，从现有 `HexTileView` 参数纯派生有序芯片列表。
2. 重构 `HexTileView.body`：保留地形/归属/单位/路线底纹；顶区态势类短码入统一顶栈；底区 AI/攻击位入统一底栈；命令 ActionMarker 与 THR/控制区/覆盖固定角落。
3. 删除旧的 `countermeasureMarkerTopPadding` 等分散 padding。
4. 顶/底可见芯片上限 2 + `+N`。
5. 图例可补一句折叠说明（可选、极简）。
6. 同步文档。
7. 提交 `v1.61: 治理地图标记拥挤度` 并 `git push origin main`。
8. Agent C 下载最新 Actions artifact 核对 `commitSha`。
