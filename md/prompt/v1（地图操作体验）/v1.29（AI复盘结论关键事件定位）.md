# v1.29 AI复盘结论关键事件定位

## 版本号

v1.29

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.28（AI复盘战果结论面板）。`origin/main` 最新验收 commit `d50267b2763816f3592633fc62c4a1e7b25b3e53` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.28-main-d50267b-run28770020966-attempt1` 核对。v1.28 已能从 AI summary 纯派生战果结论和关键事件，但关键事件只静态展示；下一步应让玩家从结论直接定位地图复盘焦点，因此本轮递增为 v1.29。

## 背景

当前 AI 复盘已有两条信息链：行动时间线可点选定位地图，战果结论能快速总结敌方回合。两者尚未连通，玩家看到结论中的“关键事件”后仍需要在下方时间线中寻找对应条目。EasyTech 风格战棋强调战报和地图的直接联动，本轮让结论关键事件复用既有 `focusAIPhaseTimelineEvent(order:)` 入口，点选后定位地图复盘标记并同步当前事件选中态。

## 目标

1. 让 `AIPhaseReplayConclusionView` 的关键事件行变成可点选控件：
   - 点选关键事件时调用既有 `GameState.focusAIPhaseTimelineEvent(order:)`。
   - 成功后应与点选同 order 的时间线行效果一致：更新 `focusedCoordinate`、`focusedAIPhaseTimelineEventOrder`、`message` 和 `focusedAIPhaseMapMarkers`。
   - 按钮文案和无障碍 hint 必须明确“只定位地图复盘，不执行命令”。
2. 保持核心规则只读边界：
   - 不新增 `@Published` 状态。
   - 不改变 `AIPhaseSummary.replayConclusion` 分类、指标、关键事件选择或排序。
   - 不调用移动、攻击、战术命令、部署、整补、AI、回合推进或任何真实命令执行入口。
   - 不改变单位 HP/位置、行动状态、据点归属、指令点、AI summary、timeline、latestAIPhaseMapMarkers、战报或反制建议。
3. 增强反馈一致性：
   - 关键事件行应能显示当前复盘事件选中态，至少与时间线行共享 `focusedAIPhaseTimelineEventOrder` 的视觉/无障碍信号。
   - 若关键事件不在当前 summary timeline 中，既有 `focusAIPhaseTimelineEvent(order:)` 失败路径负责提示；UI 不自行修复或重算事件。
4. 补充测试：
   - XCTest 覆盖用结论关键事件 order 调用定位后，与直接时间线定位一致，且不改变单位、指令点、summary、timeline、latestAIPhaseMapMarkers 和播放状态。
   - Smoke test 至少在已有 post-move barrage 或机动追击场景中覆盖结论关键事件 order 定位、地图 marker 强调和只读不变性。
5. 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不改变 AI 决策、行动顺序、时间线事件生成、地图复盘 marker 派生规则或播放速度/播放 tick 语义。
- 不新增历史 AI 回合列表、逐帧动画、镜头动画、时间轴滑块、音效、真实美术资产或第三方库。
- 不让 SwiftUI 计算结论分类、指标、关键事件排序或 marker 派生。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励、胜负或反制建议语义。

## 当前架构依据

- `AIPhaseSummary.replayConclusion.keyEvents` 已保存关键事件 `order`，该 order 与 `AIPhaseTimelineEvent.order` 对齐。
- `GameState.focusAIPhaseTimelineEvent(order:)` 已是 AI 时间线点选定位的唯一入口，成功路径更新焦点、当前 order、消息和临时引导清理；失败路径只写提示并保留旧复盘焦点。
- `focusedAIPhaseMapMarkers` 已从 `latestAIPhaseMapMarkers` 和 `focusedAIPhaseTimelineEventOrder` 纯派生，不需要新增状态。
- `AIPhaseSummaryView` 已把 `@EnvironmentObject GameState` 注入时间线区域；结论子视图可以用相同状态只读展示选中态，并把按钮事件转发给 `GameState`。

## 实现步骤

1. 修改 `ContentView.swift`：
   - 让 `AIPhaseReplayConclusionView` 重新持有 `@EnvironmentObject private var game: GameState`。
   - 将关键事件静态行改为 `Button`，点击后调用 `game.focusAIPhaseTimelineEvent(order: event.order)`。
   - 根据 `game.focusedAIPhaseTimelineEventOrder == event.order` 显示当前关键事件选中态，保持紧凑布局，不把按钮做成大卡片。
   - 增加可读 accessibility label/hint，说明只定位地图复盘、不执行命令。
2. 修改 `GameStateTests.swift`：
   - 在现有 AI 时间线点选、post-move barrage 或机动追击测试附近，使用 `summary.replayConclusion.keyEvents.first` 的 order 调用 `focusAIPhaseTimelineEvent(order:)`。
   - 断言聚焦坐标、当前 order、focused markers 与同 order 时间线事件一致。
   - 断言单位数组、指令点、summary、timeline、latestAIPhaseMapMarkers 和播放状态不变。
3. 修改 `RulesSmokeTest.swift`：
   - 在已有 AI replay smoke 场景中复用结论关键事件 order 触发定位。
   - 断言对应 focused order、focused marker、消息和只读不变性。
4. 文档同步：
   - README AI、地图交互、战斗辅助和 smoke 描述加入“结论关键事件可点选定位”。
   - `md/flow/flow.md` AI 回合摘要段落加入结论关键事件按钮复用 `focusAIPhaseTimelineEvent(order:)` 的边界。
   - `md/flow/flowchart.md` AI summary/结论到 UI 再到 focus 入口的链路补充。
   - `md/test/test.md` smoke / XCTest 基线加入 AI 复盘结论关键事件定位。
   - `md/prompt/README.md` 当前建议递进到 v1.29 后续方向。
   - `update_log.md` 追加 v1.29 记录，并修正顶部当前状态。

## 验收标准

1. AI 复盘结论关键事件可点选定位地图复盘焦点。
2. 点选关键事件与点选同 order 时间线事件使用同一 `GameState.focusAIPhaseTimelineEvent(order:)` 入口。
3. 点选后当前复盘 order、focused markers、消息和地图焦点一致更新。
4. 点选不会改变单位、据点、指令点、summary、timeline、latestAIPhaseMapMarkers、播放状态、战报或反制建议。
5. SwiftUI 不计算结论分类、指标、关键事件排序或 marker 派生。
6. 本地轻量检查通过。
7. Agent B 在 `main` 提交并 `git push origin main`。
8. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
9. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：若关键事件按钮绕过 `focusAIPhaseTimelineEvent(order:)` 自行更新焦点，会与时间线定位语义失步。
- 风险：若 View 自行匹配 timeline 或 marker，会与模型层派生失步；必须只转发 order。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负、播放控制和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘定位状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
