# v1.43（态势响应连续查看）

## 版本号

v1.43

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最新版本为 v1.42，本轮继续在 v1 地图操作体验阶段递增为 v1.43。
- v1.39-v1.41 已完成普通行动响应、响应地图标记和单条响应定位；v1.42 已补足据点防守取舍解释。本轮承接 `md/prompt/README.md` 中“态势响应连续查看”的下一轮建议。

## 背景

当前 `GameState.battlefieldSituationResponseSummary` 只从各类 `latest*Result` 即时派生当前最高优先级响应，`battlefieldSituationResponseMapMarker` 也只投射当前响应。玩家连续执行攻击、命令、部署、整补、占点、反制或敌方回合复核后，旧响应会被新的 `latest*Result` 清掉，无法在战线态势卡内连续回看最近几次执行结果。

## 目标

1. 保存最近 5 条真实态势响应，供玩家在战线态势卡连续查看。
2. 新增上一条/下一条响应查看入口，切换后地图标记和定位按钮指向当前查看的历史响应。
3. 历史只由真实成功结果追加：攻击、战术命令、据点占领、部署、整补、反制执行、反制敌方回合复核。
4. 普通移动、预览、聚焦、失败命令、无结果路径不得追加历史。
5. 重开、切换战役或重新载入场景时清空响应历史。
6. 保持所有连续查看操作只读：不改变单位、格子、资源、胜负、AI 复盘、战报结果对象或命令执行状态。

## 非目标

- 不新增自动播放、动画时间轴、筛选器、搜索或大型历史日志。
- 不改变战斗、AI、反制、补给、据点或资源规则。
- 不引入第三方库、网络依赖或复杂资产管线。
- 不把核心规则写入 SwiftUI View。
- 不下载旧 artifact 或历史大型产物。

## 当前架构依据

- `WW2Tactics/WW2Tactics/GameModels.swift` 保存响应类型、响应摘要和地图标记模型。
- `WW2Tactics/WW2Tactics/GameState.swift` 是核心规则状态机，应承载响应历史状态、追加、裁剪、清空、导航和定位逻辑。
- `WW2Tactics/WW2Tactics/ContentView.swift` 只渲染战线态势卡和按钮，不能在 View 中推导或追加历史。
- `WW2Tactics/WW2TacticsTests/GameStateTests.swift` 覆盖规则状态机行为。
- `WW2Tactics/Tools/RulesSmokeTest.swift` 覆盖命令行主规则链和关键 smoke 断言。

## 实现步骤

1. 在模型层增加轻量历史条目，或为响应摘要补充稳定序号；条目必须可比较、可识别，并保存响应摘要所需的字段，不保存完整战斗、AI 或 timeline 大对象。
2. 在 `GameState` 增加最近响应历史、当前查看序号/索引、历史容量常量和只读派生属性：
   - 最近历史最多 5 条。
   - 最新响应默认成为当前查看项。
   - `battlefieldSituationResponseSummary` 与 `battlefieldSituationResponseMapMarker` 应从当前查看项派生；无历史时保持 `nil`。
   - 提供 `canFocusPreviousBattlefieldSituationResponse`、`canFocusNextBattlefieldSituationResponse` 或等价只读能力属性。
3. 在所有真实成功结果发布点追加响应历史，且追加必须发生在明确的状态改变成功路径，不得在 computed property 中产生副作用。
4. 在所有场景载入、重开和切换路径清空历史，避免旧地图坐标污染新场景。
5. 增加 `focusPreviousBattlefieldSituationResponse()`、`focusNextBattlefieldSituationResponse()` 和复用/调整 `focusBattlefieldSituationResponseTarget()`：
   - 上一条/下一条只改变当前查看项和焦点消息，不执行命令。
   - 定位只聚焦当前查看响应的 marker 坐标。
   - 坐标不存在时给出明确消息。
6. 在 `BattlefieldSituationSummaryView` 的响应卡中增加上一条/下一条和计数提示：
   - 使用可访问的 `Button`/`Label`，不要使用裸 `onTapGesture`。
   - 按钮禁用状态来自 `GameState` 能力属性。
   - 不在 View 中写业务规则。
7. 更新测试，覆盖连续追加、容量裁剪、只读导航、普通移动和失败命令不追加、重开/切战役清空、地图 marker 跟随当前查看项。
8. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 连续执行多类真实响应后，历史按时间顺序保留最近 5 条，最新条目默认被选中。
- 上一条/下一条能切换当前查看响应，地图 marker、定位按钮和可访问文本跟随当前查看项。
- 连续查看和定位不改变单位、地图、资源、命令点、胜负、AI 复盘或结果对象。
- 普通移动、聚焦、预览和失败命令不会追加历史。
- 重开或切换场景后历史清空，旧响应不会显示在新地图。
- 本地轻量检查通过，推送到 `origin/main` 后 Agent C 下载最新 run 的 artifact，并核对 `branch=main`、`commitSha`、`runId`、`runAttempt`、JUnit、smoke log、主构建日志和项目结果文件。

## 风险和禁止项

- 禁止在 `battlefieldSituationResponseSummary` 这类 computed property 中追加历史，避免读取 UI 状态时产生副作用。
- 禁止把旧 `latest*Result` 被清空后的信息重新拼装成历史；必须在真实成功发布点保存摘要快照。
- 禁止把 View 当作历史状态源。
- 禁止保存完整大型 AI 时间线或战斗对象导致状态膨胀。
- 禁止绕过 `main` 直推和云端 artifact 验收流程。
- 禁止把旧 run、旧 artifact 或本地输出冒充本轮云端结果。
