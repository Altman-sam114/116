# v1.50 据点压力威胁来源标记

## 版本号

v1.50

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.49。
- v1.49 已把据点压力行和 PRS/DEF 地图标记对齐，本轮继续补齐“敌军从哪里压过来”的 SRC 来源标记。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

玩家点击据点压力行后，地图已经能看到受压据点 PRS 和守点目的格 DEF，但威胁来源仍只停留在压力行文字里。多单位、多据点同时受压时，玩家需要快速判断敌军来源、守点格和据点之间的空间关系。EasyTech 风格战棋的攻防判断依赖“敌军来源 -> 守点/封堵 -> 据点”这条地图链路，本轮补齐这段只读反馈。

## 目标

1. `BattlefieldSituationObjectivePressure` 暴露威胁来源坐标列表，来源必须由最新敌方意图和当前单位位置派生。
2. 当前点选压力的地图标记除 PRS/DEF 外，还要显示 SRC 威胁来源标记。
3. SRC 标记必须只展示当前压力的威胁来源，不显示过期来源、不跨压力泄漏。
4. 普通地图聚焦、普通反制点选、过期压力 id、重开和切战役仍清理当前压力标记。
5. 增补 XCTest 和 rules smoke，覆盖来源坐标、SRC marker、过期清理和只读边界。

## 非目标

- 不改变据点压力排序、敌方意图、反制建议、AI、score、移动、攻击、胜负或回合规则。
- 不新增自动守点、自动移动、自动攻击、自动整补或自动部署。
- 不新增持久 UI 状态、动画、音效、资产或第三方依赖。
- 不修改 workflow、Xcode project 或长期分支策略。

## 架构依据

- `GameState.battlefieldSituationSummary` 已从最新威胁和反制建议纯派生据点压力列表。
- `GameState` 已保存私有当前压力 id，并从最新 pressure 纯派生 PRS/DEF 地图 marker。
- 本轮只扩展 pressure 数据和 marker role，继续保持 `ContentView` 只展示 `GameState` marker，不计算威胁来源。

## 实现步骤

1. 在 `BattlefieldSituationObjectivePressure` 新增 `threatSourceCoordinates: [HexCoordinate]`：
   - 来源为同一压力分组内敌方威胁的 `enemyUnitID` 对应当前单位位置。
   - 过滤已不存在或坐标无效的单位。
   - 去重并稳定排序，避免 marker 顺序抖动。
2. 在 `BattlefieldSituationObjectivePressureMapMarkerRole` 新增 `.threatSource`：
   - `title` 为“威胁来源”。
   - `shortTitle` 为 `SRC`。
   - `compactTitle` 为 `S`。
   - 排序建议 PRS -> SRC -> DEF。
3. 在 `battlefieldSituationObjectivePressureMapMarkers(for:)` 中为当前 pressure 的每个合法来源坐标追加 SRC marker。
4. 保持 `BattlefieldSituationObjectivePressureMapMarkerView` 继续通过 role 列表展示短码组合；同坐标多角色显示紧凑组合，不新增复杂 UI 状态。
5. 扩展 `testBattlefieldSituationObjectivePressureFocusesWithoutExecuting()`：
   - pressure 的 `threatSourceCoordinates` 包含对应威胁敌军当前坐标。
   - 点选 pressure 后 marker 包含 `.threatSource` 和对应坐标。
   - stale id 和普通地图 focus 仍清空所有压力 marker。
   - 断言单位、据点、指令点、战报、latest result、AI summary 不变。
6. 扩展 rules smoke 同等断言。
7. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 点选据点压力行后，地图显示 PRS 受压据点、SRC 威胁来源，若存在不同守点目的格则继续显示 DEF。
- SRC marker 坐标来自当前最新单位位置和最新 pressure，不由 SwiftUI 计算。
- 过期 id、普通地图聚焦、重开或切战役会清除当前压力来源 marker。
- UI 不保存独立 marker 状态，不计算威胁、反制或来源坐标。
- XCTest 和 rules smoke 覆盖 SRC marker 和只读边界。
- Agent B 在 `main` 以 `v1.50: 增加据点压力威胁来源标记` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。
