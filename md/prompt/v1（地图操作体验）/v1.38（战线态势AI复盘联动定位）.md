# v1.38（战线态势AI复盘联动定位）

## 版本号

v1.38

## 版本分配依据

当前 `v1（地图操作体验）` 最新功能版本为 v1.37，已让战线态势卡优先显示反制执行后的敌方回合影响。本轮继续沿“态势 -> 执行 -> 敌方回合影响 -> 复盘定位”的链路递增为 v1.38，把战线态势卡和既有 AI 复盘关键事件连接起来。

## 背景

现有系统已经在敌方 AI 回合后生成 `latestAIPhaseSummary`、`AIPhaseReplayConclusion`、最多 3 条 `AIPhaseReplayKeyEvent`、AI 时间线和地图复盘标记。侧栏的 AI 复盘面板可以点选关键事件或时间线条目定位地图，但战线态势卡只显示敌方回合影响摘要，玩家还需要再去 AI 复盘面板寻找“是哪条敌方关键行动造成了态势变化”。

## 目标

- 为战线态势卡提供一个只读的“敌方关键复盘事件”入口。
- 当存在 `latestAIPhaseSummary.replayConclusion.keyEvents` 时，战线态势卡应显示首条关键事件的标题、摘要和定位入口。
- 点击该入口必须复用 `GameState.focusAIPhaseTimelineEvent(order:)`，让地图、当前 AI 复盘 order 和复盘标记强调沿既有路径更新。
- 保持纯派生：不新增 `@Published` 状态，不改变 AI 行为、时间线生成、关键事件排序、地图 marker、播放控制、战线态势主建议、反制建议或胜负规则。
- 无 AI summary、无关键事件、事件无坐标或坐标不在当前地图时应安全退化，不伪造可定位入口。
- 补充 XCTest 和规则 smoke test，覆盖只读派生、定位复用、缺失数据退化和无副作用。
- 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不新增新的 AI 行动、AI 评分或关键事件排序算法。
- 不改变 `AIPhaseReplayKeyEvent.keyEvents(for:)` 的优先级和最多 3 条限制。
- 不新增自动播放、跳转上一条/下一条的新状态。
- 不让战线态势按钮执行移动、攻击、整补、部署、战术命令或结束回合。
- 不把 timeline 查找、坐标选择或 marker 过滤逻辑写进 SwiftUI View。

## 当前架构依据

- `AIPhaseSummary.replayConclusion.keyEvents` 已从时间线纯派生关键事件，包含 `order`、`title` 和 `detail`。
- `GameState.focusAIPhaseTimelineEvent(order:)` 是既有 AI 复盘定位入口，会校验事件、坐标和地图，并更新 `focusedCoordinate`、`focusedAIPhaseTimelineEventOrder`、message 与目标引导清理。
- `BattlefieldSituationSummary` 是战线态势卡的只读模型，可扩展一个可选的 AI 复盘定位摘要字段。
- `ContentView.BattlefieldSituationSummaryView` 只应展示 `GameState` 已派生的信息，并把按钮动作转发给 `GameState`。

## 实现步骤

1. 在 `GameModels.swift` 新增轻量模型，例如 `BattlefieldSituationReplayTarget`：
   - 字段包含 `order`、`title`、`detail`、`coordinate`；
   - `id` 应稳定使用 order、标题和坐标；
   - 模型只表达战线态势卡可展示/可定位的信息，不持有完整 `AIPhaseTimelineEvent`。
2. 在 `BattlefieldSituationSummary` 增加可选字段 `replayTarget: BattlefieldSituationReplayTarget?`，并纳入 `id`。
3. 在 `GameState.battlefieldSituationSummary` 中通过私有 helper 派生 replay target：
   - 读取 `latestAIPhaseSummary?.replayConclusion.keyEvents.first`；
   - 用 key event 的 `order` 回查同一 summary 的 `timeline`；
   - 只接受 `event.to ?? event.from` 存在且 `tile(at:)` 可定位的事件；
   - 输出 key event 的标题、详情和定位坐标；
   - 若任一条件不满足，返回 nil。
4. 新增 `GameState.focusBattlefieldSituationReplayTarget()`：
   - 每次点击重新读取最新 `battlefieldSituationSummary.replayTarget`；
   - 无 target 时只更新提示；
   - 有 target 时调用 `focusAIPhaseTimelineEvent(order:)`；
   - 不直接设置坐标、不改 marker、不执行命令。
5. 在 `ContentView.BattlefieldSituationSummaryView` 中于 response 区或定位按钮附近新增紧凑按钮：
   - 仅当 `summary.replayTarget != nil` 显示；
   - 文案表达“复盘影响/定位敌方关键行动”；
   - 按钮 action 调用 `game.focusBattlefieldSituationReplayTarget()`；
   - 无障碍文案包含 order、标题和 detail；
   - UI 不自行读取 `game.latestAIPhaseSummary`、timeline 或 marker。
6. 增加 XCTest：
   - 有 AI summary/key event 时，`battlefieldSituationSummary.replayTarget` 使用首条关键事件 order、标题、详情和事件坐标；
   - 点击 `focusBattlefieldSituationReplayTarget()` 后，`focusedAIPhaseTimelineEventOrder`、`focusedCoordinate` 和 `focusedAIPhaseMapMarkers` 与直接调用 `focusAIPhaseTimelineEvent(order:)` 一致；
   - 读取 summary 和点击 replay target 不改变 AI summary、timeline、地图 marker、单位 HP/位置、据点归属、指令点、战报或胜负；
   - 无 summary、空 key events、事件无坐标、事件坐标不在当前地图时不生成 replay target，点击只提示且保留旧复盘焦点。
7. 更新 `RulesSmokeTest.swift` 的 AI 复盘/战线态势 smoke 链，断言战线态势可定位首条关键事件并保持只读。
8. 同步文档和日志。

## 验收标准

- 战线态势 summary 在存在可定位 AI 关键事件时暴露 replay target；无可定位关键事件时为 nil。
- 战线态势复盘按钮复用 `focusAIPhaseTimelineEvent(order:)`，点击后 AI 复盘选中态和地图标记强调与既有 AI 面板一致。
- 读取 summary 和点击 replay target 不改变 AI summary、timeline、marker、单位、据点、指令点、战报、结果摘要、胜负或 AI 行为。
- SwiftUI View 不计算 timeline、坐标或 marker，只展示模型并转发 action。
- 本地轻量检查通过，提交并 push 到 `origin/main`。
- Agent C 下载最新 run artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。

## 风险和禁止项

- 禁止改变 `AIPhaseReplayKeyEvent` 的排序、筛选和文案生成语义。
- 禁止新增独立 UI 状态导致 AI 复盘面板、地图 marker 和战线态势卡不同步。
- 禁止在 View 中查找 timeline、推导坐标或过滤 marker。
- 禁止把旧 run、旧 artifact 或本地 output 冒充 Agent C 云端验收。
- 若 GitHub 权限、账号、远端或 artifact 下载失败，必须停止并说明阻塞，不能伪装云端验证完成。
