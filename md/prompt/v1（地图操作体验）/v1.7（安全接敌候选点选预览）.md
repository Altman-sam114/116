# v1.7 安全接敌候选点选预览

## 版本号

v1.7

## 版本分配依据

- 人工未指定具体版本号。
- 当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.6，本轮继续围绕地图操作体验递增为 v1.7。
- v1.1 已生成 `SafeEngagementOption` 并在 POS 接敌预览里显示安全接敌建议，但建议仍是只读文字；本轮把安全候选变成可点选预览入口，继续提升 EasyTech 风格的地图操作决策感。

## 背景

项目目标是持续逼近 EasyTech《World Conqueror 4》《Glory of Generals》一类大地图、多据点、移动接敌、火力风险判断和回合制推进体验。当前 `GameState` 已能为射程外敌军生成 `focusedAttackPositionRoutes` 和按火力风险排序的 `focusedSafeEngagementOptions`，`ContentView` 会在命令预览文案中提示一个更安全的接敌格。

当前不足是玩家只能看到安全建议，不能把建议路线投射到地图上比较步序、消耗、敌火风险和执行结果。v1.7 要让安全接敌候选可点选，但实际移动仍必须通过既有右键或执行按钮链路进入 `GameState.executeFocusedCommand()`。

## 目标

1. `GameState` 新增规则入口，允许根据某条 `SafeEngagementOption` 或目标敌军和目的坐标聚焦安全接敌路线。
2. 点选安全候选后，地图显示该候选路线、路线步序、目的格和终点火力风险，并保持目标敌军作为接敌对象。
3. 点选候选只预览，不消耗移动/攻击，不改变单位位置，不生成普通攻击、战术命令或占领结果。
4. 默认 POS 路线排序和右键敌军的既有行为保持不变；只有点选安全候选后，焦点路线才切换到该候选目的格。
5. 侧栏或命令预览区域展示可点选的安全接敌候选，至少包含目的坐标、风险短码、潜在承伤、移动消耗和主要敌火来源。
6. 候选控件必须使用 SwiftUI `Button` 或等价可访问控件，提供清晰 label / accessibility 文案；禁止使用裸 `onTapGesture`。
7. 补充 XCTest 和规则 smoke test，覆盖安全候选聚焦、预览不执行、执行仍走既有 POS/MOVE 链路、目标失效或过期候选不产生错误预览。
8. 同步 README、flow、flowchart、test、update_log。

## 非目标

- 不改变 `safeEngagementOptionSort` 排序规则。
- 不改变默认 POS 路线、右键敌军接敌语义或 `handleSecondaryAction(on:)` 行为。
- 不新增一键直接执行安全候选；点选候选只切换焦点预览，执行仍走 `executeFocusedCommand()` 或右键已聚焦目的格。
- 不改变移动、攻击、反击、火力暴露、占领、胜负、AI 或战术命令数值。
- 不新增真实美术、动画、音效、第三方库或网络依赖。
- 不把路线、火力风险或目标有效性判断写进 `ContentView`。

## 当前架构依据

- `GameModels.swift` 已有 `SafeEngagementOption`、`MovementRoute` 和 `PostMoveFireExposurePreview`。
- `GameState.swift` 是核心规则状态机，已有 `focusedSafeEngagementOptions`、`safeEngagementOptions(for:against:)`、`focusedAttackPositionRoutes`、`focusedAttackPositionRoute`、`focusedMovementRoute`、`focusedRouteStepPreviews` 和 `focusedFireExposurePreview`。
- `ContentView.swift` 只能展示候选并转发用户意图；不得自行计算安全路线或直接写入 `focusedCoordinate`。
- 现有地图渲染已经能消费 `focusedAttackPositionRoute` / `focusedMovementRoute`、路线步序和火力风险；本轮应复用这些链路。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已有 `safeEngagementScenario()`，可扩展为候选点选和执行边界测试。

## 实现步骤

1. 在 `GameState.swift` 新增公开方法，建议命名：
   - `func focusSafeEngagementOption(_ option: SafeEngagementOption)`
   - 或 `func focusSafeEngagement(targetID: BattleUnit.ID, destination: HexCoordinate)`
2. 该方法必须：
   - 要求 `winner == nil`。
   - 要求存在当前 `selectedUnit`、目标敌军存在且未被击毁、目标仍非己方。
   - 基于当前 `selectedUnit` 和目标敌军重新调用 `safeEngagementOptions(for:against:)` 查找同一目的坐标，避免 UI 使用过期 route 直接写状态。
   - 查找成功时设置 `focusedCoordinate = matched.route.destination`，并保持 `guidedObjectiveCoordinate` 清空。
   - 写入 message，说明已切换到安全接敌候选、目的坐标、风险等级、潜在承伤和仍需执行按钮/右键才移动。
   - 查找失败时清理不适用的焦点或保留当前安全状态，并给出不可用提示；不得消耗行动。
3. 若现有 `focusedAttackPositionRoute` 只按敌军焦点取第一条路线，必要时调整 `focusedAttackPositionRoute` 或 `mapCommandPreview(for:)`，确保当 `focusedCoordinate` 是某条合法攻击位路线目的格时，预览使用该目的格对应的 route，而不是始终回到默认第一条。
4. 保持 `handleSecondaryAction(on: enemy.position)` 和直接聚焦敌军时默认 POS 路线不变。
5. 在 `ContentView.swift`：
   - 在 `FocusedCommandPreviewPanel` 的接敌预览下方，或新增小型 `SafeEngagementOptionsPanel`，展示 `game.focusedSafeEngagementOptions` 中非当前默认路线的前 2-3 条候选。
   - 每条候选用 `Button` 调用 `game.focusSafeEngagementOption(option)`。
   - 当前聚焦候选应有视觉状态，例如“当前”短码、描边或更高透明度。
   - 文案需紧凑，适配侧栏：目的坐标、风险短码、潜在承伤、路线消耗、主要敌火来源。
6. 补充 XCTest：
   - 在 `safeEngagementScenario()` 中点选敌军后，默认 `focusedAttackPositionRoute.destination == q3,r0`，安全候选首项为 `q3,r1`。
   - 调用安全候选聚焦方法后，`focusedCoordinate == q3,r1`，`focusedFireExposurePreview.coordinate == q3,r1`，`focusedCommandPreview` 为 `.approachAttack` 或等价可执行接敌预览，单位位置不变，`hasMoved == false`，结果摘要保持 nil。
   - 聚焦安全候选后执行 `executeFocusedCommand()`，单位移动到 `q3,r1`，未自动攻击，`focusedCoordinate == target.position`，目标仍可继续攻击。
   - 目标被击毁、单位不可移动、候选目的格被占据或目标不再敌对时，聚焦方法不会移动单位或保留过期可执行预览。
7. 补充 `RulesSmokeTest.swift` 对安全候选点选和执行链路的关键断言。
8. 更新文档：
   - `WW2Tactics/README.md`：安全接敌建议可点选预览。
   - `md/flow/flow.md`：POS 接敌流加入安全候选聚焦入口和“只预览、不执行”边界。
   - `md/flow/flowchart.md`：地图命令流加入安全候选点击 -> `GameState` 重新查候选 -> 聚焦路线。
   - `md/test/test.md`：当前基线加入安全接敌候选点选预览覆盖点。
   - `update_log.md`：追加 v1.7 记录。

## 验收标准

- 侧栏或命令预览区出现可点选安全接敌候选。
- 点选安全候选后，地图投射该候选路线、目的格、路线步序和终点火力风险。
- 点选候选不会消耗行动、不会移动单位、不会自动攻击、不会生成结果摘要。
- 默认 POS 路线和右键敌军接敌行为保持不变。
- 执行仍通过现有 `executeFocusedCommand()` / 右键已聚焦目的格链路完成。
- XCTest 和 smoke test 覆盖候选聚焦、预览/执行分离和失效候选边界。
- 本地轻量检查通过，`main` push 后 GitHub Actions 最新 run 通过。
- Agent C 已下载并核对最新 artifact，manifest 与 `origin/main` 最新 commit 完全一致。

## 风险和禁止项

- 禁止在 `ContentView` 中直接修改 `focusedCoordinate` 或自行选择 route。
- 禁止 UI 使用过期 `SafeEngagementOption.route` 直接执行移动。
- 禁止改变默认 POS 路线排序或右键敌军行为。
- 禁止把候选按钮做成直接移动、直接攻击或一键消耗行动。
- 禁止伪造本地测试、云端 run、artifact 或 manifest 结果。
