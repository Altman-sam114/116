# v1.49 据点压力行选中态

## 版本号

v1.49

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.48。
- v1.48 已把据点压力定位投射为地图 PRS/DEF 标记，本轮继续补齐侧栏压力行与地图标记之间的当前选中态。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

玩家点击据点压力行后，地图已经能显示 PRS 受压据点和 DEF 守点目的格，但侧栏压力列表本身没有明确标出哪一行正在驱动当前地图标记。多据点受压时，玩家需要把地图标记和列表内容反复对照，操作闭环还不够接近 EasyTech 风格的“列表选中 -> 地图高亮”反馈。

## 目标

1. `GameState` 提供只读查询，判断某条据点压力是否为当前聚焦压力。
2. `ContentView` 在据点压力行上显示当前选中态，并在无障碍文案中说明当前压力。
3. 选中态必须由 `GameState` 最新压力列表和当前压力 id 派生，不能由 View 自行保存状态或反查规则。
4. 普通地图聚焦、普通反制点选、过期压力 id、重开和切战役仍清理当前压力选中态。
5. 增补 XCTest 和 rules smoke，覆盖选中态、过期清理和普通聚焦清理。

## 非目标

- 不改变据点压力排序、敌方意图、反制建议、AI、score、移动、攻击、胜负或回合规则。
- 不新增自动守点、自动移动、自动攻击、自动整补或自动部署。
- 不新增持久 UI 状态、动画、音效或第三方依赖。
- 不修改 workflow、Xcode project 或长期分支策略。

## 架构依据

- `GameState.swift` 已保存私有当前压力 id，并从最新 summary 派生地图 marker。
- 本轮只新增只读查询方法，继续保持 `ContentView` 只展示和转发。
- `GameStateTests.swift` 与 `Tools/RulesSmokeTest.swift` 验证选中态和清理边界。

## 实现步骤

1. 在 `GameState.swift` 新增 `isBattlefieldSituationObjectivePressureFocused(id:)`：
   - id 必须等于当前压力 id。
   - 最新 `battlefieldSituationSummary.objectivePressures` 必须仍包含该 id。
2. 在 `ContentView.BattlefieldSituationSummaryView` 调用该查询，并把 `isFocused` 传给 `BattlefieldSituationObjectivePressureRow`。
3. 在 pressure row 中显示紧凑“当前”标识、边框或背景强调，并更新 accessibility label/hint。
4. 扩展 `testBattlefieldSituationObjectivePressureFocusesWithoutExecuting()`：
   - 点选压力后该 id 为 focused。
   - stale id 后 focused 变 false。
   - 普通 `focus(coordinate:)` 后 focused 变 false。
5. 扩展 rules smoke 同等断言。
6. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 点选据点压力行后，该行显示当前选中态，并与地图 PRS/DEF marker 对应。
- 过期 id 或普通聚焦会清除当前压力选中态和地图压力 marker。
- UI 不保存独立选中状态，不计算威胁、反制或坐标。
- XCTest 和 rules smoke 覆盖选中态和清理边界。
- Agent B 在 `main` 以 `v1.49: 增加据点压力行选中态` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。
