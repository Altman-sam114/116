# v1.54 据点压力敌方回合影响

## 版本号

v1.54

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.53，且 v1.51-v1.53 已连续完善据点压力复盘线索、压力态势对照和战线态势复盘来源筛选。本轮继续在同一阶段递增为 v1.54。

## 背景

项目目标是持续逼近 EasyTech 风格二战回合制战棋。现有战线态势卡已经能列出受威胁据点、当前/应对态势对照、压力行定位、PRS/SRC/DEF 地图标记和关联 AI 复盘线索，但压力行对“上一敌方回合后，这个守点应对到底守住还是失守”的解释仍分散在反制复核卡中。

## 目标

- 为据点压力行增加只读的敌方回合后影响摘要。
- 影响摘要必须从真实 `latestEnemyThreatCountermeasureFollowUpResult` 和 `latestAIPhaseSummary` 派生，不模拟未来 AI。
- 当最新 follow-up 匹配当前压力据点时，在压力行展示前后对比、结果和 AI 回合信息。
- 保持 v1.53 复盘来源优先级：当前压力 PRS > 响应位置 RSP > 全局关键事件 KEY。
- 补充 XCTest 和 RulesSmokeTest，覆盖字段派生、只读边界和无 follow-up 时不伪造影响。

## 非目标

- 不改 AI 决策、移动、攻击、战术命令、整补、部署或据点归属规则。
- 不新增未来模拟、概率预测或新的威胁排序。
- 不改变据点压力排序。
- 不改变反制建议排序、执行入口或复盘播放行为。
- 不引入第三方库、网络依赖或资产管线。

## 当前架构依据

- `GameModels.swift` 放结构化模型。
- `GameState.swift` 放核心规则状态机和只读派生逻辑。
- `ContentView.swift` 只展示 `GameState` 已派生字段，不反推规则。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 锁定压力行、follow-up、AI 复盘和只读定位边界。

## 实现步骤

1. 在 `GameModels.swift` 增加 `BattlefieldSituationObjectivePressureEnemyPhaseImpact`，包含 outcome、标题、详情、before/after/result、AI 回合和来源说明。
2. 在 `BattlefieldSituationObjectivePressure` 增加 `enemyPhaseImpact: ...?`，但不要把该字段加入 `id`，避免压力行身份随 AI 后摘要抖动。
3. 在 `GameState.battlefieldSituationObjectivePressures` 中，为每个据点压力调用 helper，从最新 objective-defense follow-up 中匹配 `threatTargetCoordinate`。
4. helper 只接受真实 follow-up 和 AI summary；无 follow-up、坐标不匹配、非 objective defense 时返回 nil。
5. `ContentView.BattlefieldSituationObjectivePressureRow` 展示第三段“敌方回合后”对比，并更新无障碍文案。
6. 补 XCTest：初始压力无影响；守点反制执行并结束回合后，匹配压力暴露影响，且读取/聚焦不改单位、据点、指令点、战报、AI summary 或 follow-up。
7. 补 smoke test：覆盖同一主链路和无 follow-up 不伪造影响。
8. 更新 README、flow、flowchart、test 和 update_log。

- 压力行能展示匹配守点 follow-up 的敌方回合后对比。
- 无真实 follow-up 时不显示或伪造敌方回合影响。
- 压力行 `id` 不因影响字段变化而改变。
- 压力定位和复盘定位仍不执行命令。
- 本地轻量检查通过。
- `main` 推送后 GitHub Actions run 成功，Agent C 下载 artifact 并核对 manifest 的 branch、commitSha、runId、runAttempt。
