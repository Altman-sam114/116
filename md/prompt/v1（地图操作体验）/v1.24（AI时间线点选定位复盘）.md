# v1.24 AI时间线点选定位复盘

## 版本号

v1.24

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.23（AI移动后火炮弹幕）。`origin/main` 最新验收 commit `ae6efb507fa95818408956523c38d02608e9b99b` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.23-main-ae6efb5-run28742308938-attempt1` 核对。v1.23 已让 AI 行动时间线和地图复盘覆盖移动后火炮弹幕；下一步应让玩家能从侧栏时间线直接定位到对应地图复盘坐标。本轮递增为 v1.24。

## 背景

当前敌方 AI 回合结束后，侧栏会展示 `AIPhaseSummary.timeline` 的最多 5 条行动时间线，地图也会显示由同一 timeline 派生的 `latestAIPhaseMapMarkers`。但是时间线行目前只是静态文本，玩家需要自己在大地图上寻找对应 AI 标记。EasyTech 风格大地图战棋需要更强的战报复盘定位能力：点一条战报，地图应该滚到相关地点，但不应误触发移动、攻击或战术命令。

本轮目标是为 AI 时间线行增加点选定位入口。实现必须把定位规则放在 `GameState`，SwiftUI 只转发用户意图；点选时间线只更新地图焦点和提示消息，不改变单位、指令点、AI 摘要、地图复盘标记或任何真实结果。

## 目标

1. 在 `GameState` 增加可测试入口，例如 `focusAIPhaseTimelineEvent(order:)`：
   - 根据最近一次 `latestAIPhaseSummary?.timeline` 查找指定顺序号事件。
   - 优先定位到 `event.to`，若没有终点则定位 `event.from`。
   - 若事件不存在、事件没有坐标或坐标已不在地图内，只更新明确提示，不改变当前焦点。
   - 成功定位时更新 `focusedCoordinate`，并设置类似 `AI复盘 #<order>：<summary>` 的提示。
2. 侧栏 AI 时间线行改为可点选：
   - `AIPhaseSummaryView` 或其子视图通过 `@EnvironmentObject GameState` 或回调调用规则层入口。
   - 使用 `Button` 承载行点击，不使用 `onTapGesture()`。
   - 保持现有紧凑视觉样式，不新增可见说明文字。
   - 增加无障碍 hint，说明该操作只定位地图复盘，不执行命令。
3. 保持地图自动滚动链路：
   - 复用现有 `focusedCoordinate` 与 `MapCommandCenter` 的 `ScrollViewReader` 监听。
   - 不新增独立滚动状态或 UI 缓存。
4. 补充 XCTest 和规则 smoke test：
   - 成功点选移动、攻击或战术命令事件后，焦点应落到 `event.to ?? event.from`。
   - 提示消息应包含 `AI复盘 #<order>` 和事件摘要。
   - 点选定位不能改变单位 HP/位置、指令点、AI summary 或 AI map markers。
   - 无效顺序号、无坐标事件或不在地图内坐标不能错误改焦点。
5. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变 AI 决策、行动顺序、战术命令选择、直取据点、移动后攻击或移动后火炮弹幕。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励或胜负条件。
- 不新增逐帧播放、自动镜头动画、暂停/继续回放、音效、真实美术资产或历史 AI 回合列表。
- 不新增独立 `@Published` 地图复盘焦点状态；应复用 `focusedCoordinate`。
- 不让 SwiftUI 根据 timeline 自行计算坐标有效性或修改规则状态。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `AIPhaseSummary.timeline` 是敌方回合成功动作的结构化记录，事件中已包含 `from`、`to`、行动单位、目标、伤害、战术命令和据点等字段。
- `latestAIPhaseMapMarkers` 已从 `latestAIPhaseSummary.timeline` 纯派生地图复盘标记，不应新增持久状态保存标记。
- `GameState.focus(coordinate:)` 已负责普通地图聚焦、消息和清理目标引导；本轮入口可参考其清理策略，但消息必须明确来自 AI 复盘。
- `MapCommandCenter` 已监听 `game.focusedCoordinate?.id` 并滚动到对应格；本轮只需正确更新 `focusedCoordinate`。
- `ContentView` 是表现层，只能展示状态和转发点选意图，不能绕过 `GameState` 自行执行规则。

## 实现步骤

1. 在 `GameState.swift` 增加 `focusAIPhaseTimelineEvent(order:)`：
   - 从 `latestAIPhaseSummary?.timeline.first { $0.order == order }` 查找事件。
   - 坐标选择规则为 `event.to ?? event.from`。
   - 失败分支：
     - 无 summary 或无事件：`message = "未找到AI复盘事件 #\(order)"`，不改变 `focusedCoordinate`。
     - 无坐标：`message = "AI复盘 #\(order) 没有可定位坐标"`，不改变 `focusedCoordinate`。
     - 坐标不在 `scenario.tiles`：`message = "AI复盘 #\(order) 坐标不在当前地图"`，不改变 `focusedCoordinate`。
   - 成功分支：
     - 调用 `clearObjectiveGuidance()` 清理 OBJ/SAFE/反制聚焦等临时引导。
     - 设置 `focusedCoordinate = coordinate`。
     - 设置消息 `AI复盘 #\(event.order)：\(event.summary)`，必要时保持简短。
   - 不调用 `move`、`attack`、`deploy`、`reinforce`、`useTacticalCommand`、`endTurn` 或 AI 方法。
2. 在 `ContentView.swift` 连接侧栏：
   - 让 `AIPhaseSummaryView` 能访问 `GameState`，优先沿用项目当前 `@EnvironmentObject` 风格。
   - 将 `AIPhaseTimelineEventRow` 包在 `Button` 中，action 调用 `game.focusAIPhaseTimelineEvent(order: event.order)`。
   - 使用 `.buttonStyle(.plain)` 保持现有行视觉。
   - 给按钮添加 `accessibilityHint("只定位地图复盘，不执行命令")` 或等价 VoiceOver hint。
3. 增加 XCTest：
   - 复用现有能生成 AI timeline 的小场景，例如移动后火炮弹幕、直取据点或机动追击场景。
   - `endTurn()` 后选取至少一个含 `to` 或 `from` 的事件。
   - 调用 `focusAIPhaseTimelineEvent(order:)`，断言 `focusedCoordinate == event.to ?? event.from`。
   - 保存调用前的单位快照、指令点、summary 和 map markers，断言调用后不变。
   - 覆盖无效 order 不改变旧焦点和不会清空 summary。
4. 增加 `RulesSmokeTest.swift` 断言：
   - 在既有 AI summary/timeline smoke 链路后调用 focus 入口。
   - 验证焦点坐标和消息，且 timeline/markers 数量不变。
5. 文档同步：
   - `WW2Tactics/README.md` AI 或地图交互说明加入“AI 时间线行可点选定位地图复盘标记，但不执行命令”。
   - `md/flow/flow.md` 的 AI 回合摘要和行动时间线段落加入点选定位语义。
   - `md/flow/flowchart.md` 在输入转发、地图命令执行流或 AI 状态图中加入 `focusAIPhaseTimelineEvent`。
   - `md/test/test.md` 当前 smoke / XCTest 基线加入 AI 时间线点选定位复盘。
   - `update_log.md` 追加 v1.24 记录，本地验证后写具体命令结果，Agent C 验收后补云端结果。

## 验收标准

1. 侧栏 AI 时间线行可点选，并通过 `GameState.focusAIPhaseTimelineEvent(order:)` 定位地图焦点。
2. 定位坐标规则稳定为 `event.to ?? event.from`。
3. 无 summary、无事件、无坐标或坐标不在当前地图时，不改变旧焦点，并给出明确消息。
4. 点选 AI 时间线不会移动单位、攻击、消耗指令点、改变 HP、改变 AI summary、改变 timeline 或改变 `latestAIPhaseMapMarkers`。
5. 地图自动滚动复用 `focusedCoordinate` 监听链，不新增独立复盘焦点状态。
6. SwiftUI 行点击使用 `Button`，有无障碍 hint，并保持现有紧凑视觉。
7. 本地轻量检查通过。
8. Agent B 在 `main` 提交并 `git push origin main`。
9. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
10. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：若 View 直接从 timeline 反推地图或写状态，容易绕过 `GameState`；必须只调用规则层入口。
- 风险：点选时间线若复用普通命令执行链，可能误触发 MOVE/ATK/POS；必须只更新焦点和消息。
- 风险：AI timeline 事件可能缺少坐标；失败分支必须保留旧焦点，不能强行选择当前单位坐标。
- 风险：旧 summary 可能来自上一回合而当前地图已切换；坐标必须重新校验在当前 `scenario.tiles` 中。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
