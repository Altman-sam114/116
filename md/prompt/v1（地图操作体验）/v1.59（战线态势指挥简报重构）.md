# v1.59（战线态势指挥简报重构）

## 版本号

v1.59

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最大版本为 v1.58。v1.56-v1.58 已依次完成战场外壳、地图格/单位棋子和攻击反馈卡视觉升级。本轮继续在 `v1（地图操作体验）` 下递增，重构侧栏靠前的战线态势面板。

## 背景

`BattlefieldSituationSummaryView` 汇集当前优先级、战线摘要、最近响应、敌方回合复盘入口、首要定位、据点压力和战场指标，是玩家每回合最重要的指挥简报。但当前实现把多个交互分区塞在一个超长 `body` 中，视觉层级偏平，文字过密，历史导航和定位按钮点击区域小于 44pt。需要在不改变任何 `GameState` 行为的前提下，把它重构为更接近 EasyTech 风格的指挥简报卡。

## 目标

- 将 `BattlefieldSituationSummaryView` 拆成职责清晰的私有 SwiftUI 子 View，降低单个 body 复杂度。
- 保持现有显示顺序：态势标题 -> 最近响应 -> 全局复盘影响 -> 首要定位 -> 据点压力/压力复盘 -> 指标 -> 焦点摘要。
- 将主卡升级为战区指挥简报视觉：优先级结论、摘要、响应、行动入口、压力行和指标有明确层级。
- 强化据点压力行的当前态、来源、威胁、当前/应对对照和敌方回合影响可读性。
- 将态势响应历史上一条/下一条和定位按钮的可点击区域提升到至少 44x44，同时保留图标式视觉和 VoiceOver 文案。
- 复用 `BattlefieldTheme`，不修改全局主题 token、颜色映射或高复用结果组件。

## 非目标

- 不修改 `GameModels.swift`、`GameState.swift`、态势派生、压力排序、响应历史、AI 时间线或地图 marker。
- 不改变侧栏中态势面板的位置。
- 不改变条件显示、`objectivePressures.prefix(3)`、任何按钮 action、disabled 条件或聚焦行为。
- 不修改 `CombatResultMetric`、`CombatantResultLine`、`CombatPreview` helper 或地图响应颜色映射。
- 不新增规则状态、动画、音效、第三方依赖或图片资产。

## 当前架构依据

- `GameState.battlefieldSituationSummary` 和相关 response/replay/pressure 状态是唯一数据来源，View 只消费派生值。
- 所有态势按钮只做定位、历史浏览或 AI 复盘聚焦，不移动、不攻击、不整补、不部署、不消耗行动。
- `BattlefieldSituationSummaryView` 只由 `InspectorPanel` 使用；其压力行、压力复盘按钮和指标块也没有其他侧栏调用点，可安全局部重构。
- `BattlefieldTheme` 已提供 `commandDeckDeep`、`brass`、`ink`、`mutedInk` 和 `hairline`，本轮只复用，不改变常量。

## 实现步骤

1. 在 `ContentView.swift` 中重构 `BattlefieldSituationSummaryView`：
   - 保持现有分区顺序和条件分支。
   - 把标题、响应、全局复盘、首要目标、压力区和页脚拆成私有子 View。
   - 子 View 只接收模型、颜色、状态值和 action closure，不拥有规则状态。
2. 重构最近响应卡：
   - 突出响应类型、标题、结果和历史位置。
   - 上一条/下一条仍分别调用 `focusPreviousBattlefieldSituationResponse` / `focusNextBattlefieldSituationResponse`。
   - 定位仍调用 `focusBattlefieldSituationResponseTarget`。
   - 交互点击区至少 44x44。
3. 重构全局复盘和首要目标入口：
   - 保留 `focusBattlefieldSituationReplayTarget` 和 `focusBattlefieldSituationPrimaryTarget`。
   - 保留来源短码、事件 order、action hint、可执行语义和无障碍文案。
4. 重构据点压力区：
   - 保留最多前三项及 `GameState` 原排序。
   - 压力行仍调用 `focusBattlefieldSituationObjectivePressure(id:)`。
   - 压力复盘按钮保持独立，禁止嵌套进压力行 Button。
   - 强化当前态、NOW/CHK、威胁源数量、当前/应对对照和敌方回合影响层级。
5. 重构 `BattlefieldSituationMetricPill` 和页脚，让指标和当前焦点更容易扫描，不改变内容、数量或排序。
6. 同步 README、flow、flowchart、test、prompt README 和 update_log。
