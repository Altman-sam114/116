# v1.15（反制建议收益解释）

## 版本号

v1.15

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新版本为 v1.14（反制建议执行桥接预览）。本轮继续沿 v1 地图操作体验小步迭代，不开启新大版本，因此递增为 v1.15。

## 背景

v1.11 生成敌方意图反制建议，v1.12 支持点选聚焦，v1.13 支持地图 ACT/SRC/CTR/TGT 标记，v1.14 支持下一步执行入口桥接。当前玩家已经能知道“谁去做、打谁或去哪、从哪个入口执行”，但还不能快速看出“为什么这条建议值得优先处理”。

为了更接近 EasyTech 风格的参谋提示，本轮要把反制建议的收益拆成结构化说明，让玩家在不读长句的情况下看懂战果、生存、守点、恢复和行动成本等关键因素。

## 目标

1. 新增结构化只读收益解释模型，表达反制建议的主要价值：
   - 抢先打击：预计伤害、是否击毁、敌军剩余耐久、己方反击后耐久。
   - 撤出危险区：预计保留耐久、目的格、路线消耗。
   - 据点防守：是否进驻/封堵目标据点、目的格、路线消耗。
   - 整补支撑：恢复耐久、整补后耐久、当前位置/己方据点。
2. `EnemyThreatCountermeasurePreview` 或 `GameState` 提供稳定、可测试的收益解释字段，例如 `benefitMetrics`、`benefitSummary` 或等价结构。
3. `ContentView` 在反制建议行内用紧凑 chips/指标展示收益解释，并在无障碍 label 中包含收益摘要。
4. 解释必须来自规则层已有字段，不在 View 中重新计算战斗、路线、整补或评分。
5. 保持 v1.14 的执行桥接行为不变：建议行仍只聚焦，不执行命令。

## 非目标

- 不改变 `enemyThreatCountermeasureSort` 排序规则和 `score` 计算公式。
- 不新增一键执行、自动命令队列、确认弹窗或 AI 决策。
- 不改变敌方意图生成、反制建议生成、limit 策略、战斗数值、移动规则、整补成本、胜负条件或回合流程。
- 不引入第三方库、网络依赖、真实美术资产、动画或复杂资产管线。
- 不把收益解释做成冗长说明页；它必须服务侧栏快速扫描。

## 当前架构依据

- `GameModels.swift` 已定义 `EnemyThreatCountermeasurePreview`、地图标记、执行桥接预览等共享模型。收益解释若被 UI 和测试共同使用，优先放在这里，尽量用 computed property 避免改动现有构造调用。
- `GameState.swift` 已在生成反制建议时写入 `projectedDamage`、`projectedEnemyHPAfterDamage`、`willDestroyEnemy`、`projectedFriendlyHPAfterAction`、`projectedRecoveredHP`、`destination`、`routeCost`、`score` 和 `reason`。本轮解释应复用这些字段，不重新跑规则。
- `ContentView.swift` 的 `EnemyThreatCountermeasureRow` 已展示类型、执行单位、目标、威胁、效果、路线和原因；本轮应在该行内补紧凑收益说明，不让 View 推导规则有效性。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已覆盖四类反制建议、点选聚焦、地图标记和执行桥接预览；本轮应复用同一 fixture 增加收益解释断言。

## 实现步骤

1. 在 `GameModels.swift` 新增轻量收益解释模型：
   - 枚举例如 `EnemyThreatCountermeasureBenefitKind`，包含 `.damage`、`.survival`、`.objective`、`.recovery`、`.route`、`.priority` 等。
   - 结构体例如 `EnemyThreatCountermeasureBenefitMetric`，包含 kind、title、value、detail。
   - 为 `EnemyThreatCountermeasurePreview` 新增 computed property，例如 `benefitSummary`、`benefitMetrics`、`priorityText`。
2. 收益解释规则：
   - 抢先打击：首要 metric 是伤害或击毁；若有 `projectedEnemyHPAfterDamage`，显示敌军剩余耐久；若有 `projectedFriendlyHPAfterAction`，显示反击后己方耐久。
   - 撤出危险区：首要 metric 是预计保留耐久；附带目的格和路线消耗。
   - 据点防守：首要 metric 是“进驻”或“封堵”，根据 `destination == threatTargetCoordinate` 判定；附带目的格和路线消耗。
   - 整补支撑：首要 metric 是恢复耐久；附带整补后耐久和当前位置。
   - 所有类型都可显示 `score` 或“优先值”作为排序解释，但不能改变排序。
3. 更新 `ContentView`：
   - 在 `EnemyThreatCountermeasureRow` 的现有指标区下方或原因前显示 `benefitMetrics.prefix(3)`。
   - 使用现有小指标/胶囊风格，保持文本可换行或缩放，不造成移动端溢出。
   - 更新反制建议行的 accessibility label，加入 `benefitSummary`。
4. 扩展 XCTest：
   - 在 `testEnemyThreatCountermeasuresCoverActionsLimitAndReadOnly` 中断言四类建议都有非空收益解释和关键 metric。
   - 抢先打击包含 `.damage`，击毁时 value 为“击毁”或非击毁时包含预计伤害。
   - 撤出危险区包含 `.survival` 和 `.route`。
   - 据点防守包含 `.objective` 和 `.route`。
   - 整补支撑包含 `.recovery` 和整补后耐久。
   - 查询收益解释无副作用。
5. 扩展 `RulesSmokeTest.swift` 中反制建议段，加入同类轻量断言。
6. 同步更新：
   - `WW2Tactics/README.md`：说明反制建议展示收益解释。
   - `md/flow/flow.md`：记录收益解释是只读字段派生。
   - `md/flow/flowchart.md`：在反制建议到 UI 的链路中增加收益解释节点。
   - `md/test/test.md`：更新当前基线覆盖项。
   - `update_log.md`：新增 v1.15 记录。

## 验收标准

1. 反制建议行能显示紧凑收益解释，玩家可快速区分战果、生存、守点、恢复和路线成本。
2. 收益解释来自 `GameModels` 或 `GameState` 结构化字段，`ContentView` 不自行计算战斗、路线、整补或评分。
3. 不改变反制建议排序、score 公式、执行桥接、真实移动、攻击、整补、AI 或胜负规则。
4. 收益解释查询不消耗行动，不改变 HP、位置、行动状态、据点归属、指令点、战报、胜负或 `latest*Result`。
5. XCTest、规则 smoke、SwiftUI typecheck 和测试源码 typecheck 通过。
6. `main` 已推送，GitHub Actions 最新 run 成功。
7. Agent C 已下载最新 artifact，核对 manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与 `origin/main` 一致，并检查 failure summary、JUnit、rules smoke、xcodebuild log 和 `.xcresult`。

## 风险和禁止项

- 禁止把 UI 文案当作规则来源；收益解释必须由模型/状态结构化字段提供。
- 禁止改动排序公式来迁就文案。
- 禁止让反制建议行直接执行命令。
- 禁止调用真实 `move`、`attack`、`reinforce` 或 `executeFocusedCommand()` 来生成解释。
- 禁止破坏 v1.14 的 ATK/MOVE/整补入口桥接。
- 禁止扩大范围到 AI、数值、战役数据、真实资产或动画。
- 禁止跳过 `main` 直推和 Agent C artifact 验收。
