# v1.33 战线态势汇总

## 版本号

v1.33

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 已完成到 v1.32（反制复核结论定位），本轮继续在 v1 地图操作体验阶段递增为 v1.33。v1.29-v1.32 已强化 AI 复盘、路径风险、据点推进解释和反制复核定位，但玩家回合仍缺少一个把据点、待命单位、敌方意图和反制建议汇总到一起的战线总览入口。

## 背景

项目目标是持续逼近 EasyTech 风格二战回合制战棋。现有系统已经能展示 MOVE/ATK/POS、OBJ 计划、敌方意图、反制建议、AI 复盘和执行复核，但这些信息分散在地图、侧栏和具体面板里。玩家在回合开始时需要一个简短的“当前战线态势”判断：还有多少单位能行动、据点推进是否领先、哪些据点正受威胁、是否有立即可执行的反制建议，以及本回合优先做什么。

## 目标

1. 新增纯派生 `BattlefieldSituationSummary` 模型，汇总当前玩家回合态势。
2. 在 `GameState` 暴露 `battlefieldSituationSummary` computed property，不新增 `@Published` 状态。
3. 汇总内容至少包含：当前阵营、指令点、待命单位数、据点进度、敌方意图数量、可执行反制数量、受威胁据点名称、首要建议标题和说明。
4. 在 `ContentView` 侧栏靠前位置展示紧凑态势卡，使用 `GameState` 已派生字段，不在 View 中计算规则。
5. 增加 XCTest 和 smoke test，确认汇总字段可派生、稳定且不改变单位/据点/指令点/结果摘要。

## 非目标

- 不新增新规则、AI 行为、战斗公式、据点评分或行动消耗。
- 不让态势卡按钮自动执行移动、攻击、整补或部署。
- 不引入第三方库、网络资源、美术资产或复杂新页面。
- 不重排敌方意图、反制建议、OBJ 计划的既有排序规则。

## 当前架构依据

- `GameModels.swift` 放纯数据模型和派生文本。
- `GameState.swift` 是规则状态机和只读派生入口。
- `ContentView.swift` 只展示 `GameState` 输出并转发用户意图。
- `GameStateTests.swift` 覆盖规则和派生行为。
- `RulesSmokeTest.swift` 覆盖主规则链和轻量回归。

## 实现步骤

1. 在 `GameModels.swift` 增加态势等级/重点类型和 `BattlefieldSituationSummary`，字段保持 `Equatable`，UI 可直接读取标题、短码和说明。
2. 在 `GameState.swift` 增加 `battlefieldSituationSummary`：
   - 读取 `activeFaction`、`activeCommandPoints`、`readyUnits`、`objectiveTiles`、`visibleEnemyThreatIntentPreviews`、`visibleEnemyThreatCountermeasurePreviews`。
   - 统计敌方攻击威胁、据点威胁、可执行反制、受威胁据点。
   - 根据优先级派生首要建议：可执行反制 > 受威胁据点 > 有待命单位推进 > 无待命单位结束回合/等待 > 已胜负。
   - 不写入任何状态，不调用移动、攻击、整补、部署、战术命令或 AI。
3. 在 `ContentView.swift` 的 `InspectorPanel` 靠前位置新增 `BattlefieldSituationSummaryView`，展示态势等级、指标 pill、受威胁据点和建议说明。
4. 在 `GameStateTests.swift` 增加测试，断言初始态势、反制建议统计、受威胁据点和只读不变式。
5. 在 `RulesSmokeTest.swift` 增加 smoke 断言，覆盖默认战役态势汇总主字段。
6. 同步更新 `WW2Tactics/README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`md/prompt/README.md` 和 `update_log.md`。

## 验收标准

- 侧栏能看到战线态势卡，字段来自 `GameState.battlefieldSituationSummary`。
- 态势汇总能指出指令点、待命单位、据点进度、敌方意图、可执行反制、受威胁据点和首要建议。
- 查询态势汇总不会改变单位位置/HP、行动状态、据点归属、指令点、战报、最新结果或胜负。
- 本地轻量检查通过。
- `origin/main` 最新 commit 的 GitHub Actions artifact 通过 Agent C 核对。

## 风险和禁止项

- 禁止把规则统计写进 SwiftUI View。
- 禁止新增不可测试的 `@Published` 态势状态。
- 禁止在态势汇总中调用真实执行方法。
- 禁止用旧 run、旧 artifact 或本地输出冒充云端验收。
- 禁止在未通过 Agent C artifact 验收时宣布本轮通过。
