# v1.18 反制建议执行回放

## 版本号

v1.18

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.17（反制建议执行前后对照）。v1.17 已经让玩家看到采纳建议前后的预计变化，但真实执行后还没有把“预计”与“实际执行结果”关联起来。本轮继续推进同一条反制建议体验链，因此递增为 v1.18。

## 背景

v1.10 到 v1.17 已经形成敌方意图、反制建议、点选聚焦、地图标记、执行入口桥接、收益解释、排序对比解释和预计执行前后对照。玩家现在能按建议聚焦到现有 ATK、MOVE 或整补入口，但执行完成后只能看到普通战斗、移动或整补结果，无法确认刚才的反制建议是否达成预期。EasyTech 风格参谋提示需要把“建议 -> 执行 -> 结果反馈”串起来，让玩家知道这一手是否达到预期。

## 目标

1. 新增最近一次反制建议执行回放模型，记录玩家通过既有 ATK、MOVE 或整补入口执行当前聚焦反制建议后的预计与实际对照。
2. 只在真实成功执行后记录回放：
   - 抢先打击：执行现有普通攻击后，记录预计伤害/敌 HP/击毁与实际 `latestCombatResult`。
   - 撤出危险区：执行现有 MOVE 后，记录预计目的地/HP 与实际位置/HP。
   - 据点防守：执行现有 MOVE 后，记录预计进驻或封堵与实际位置、据点归属或封堵动作。
   - 整补支撑：执行既有整补按钮后，记录预计恢复/整补后耐久与实际 `latestReinforcementResult`。
3. 在侧栏反制建议面板附近展示最近一次“预计 / 实际 / 结果”摘要，并加入无障碍文本。
4. 补充 XCTest 和规则 smoke test，覆盖四类执行回放、普通聚焦或无关执行不生成回放、重开/切战役清理回放。
5. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不新增一键执行反制建议按钮。
- 不改变 ATK、MOVE、POS、整补、战术命令、AI、排序、score、战斗数值、移动规则或整补成本。
- 不让反制建议行直接执行命令；仍必须走现有地图 ATK/MOVE 或单位详情整补按钮。
- 不记录未执行的预计结果为实际结果。
- 不把核心判定或回放生成写入 `ContentView`。
- 不新增第三方库、网络依赖、图片资产或复杂资产管线。
- 不重构 `ObservableObject/@Published` 架构。

## 当前架构依据

- `GameModels.swift` 已定义 `EnemyThreatCountermeasurePreview`、`impactComparisons`、执行桥接预览和所有真实结果摘要模型；本轮应新增轻量回放模型放在这里。
- `GameState.swift` 已保存 `focusedEnemyThreatCountermeasurePreview`，并在 `executeMapCommand`、`attack`、`move`、`reinforce` 成功路径中生成真实结果摘要；本轮应在执行前捕获仍有效的聚焦反制建议，执行成功后基于已有真实结果或单位状态生成回放。
- `ContentView.swift` 只能展示 `GameState` 暴露的回放字段，不能自行计算战斗、路线、整补或据点结果。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已有 `enemyThreatIntentScenario(axisSpent: true)`，可复用四类反制建议执行链测试。

## 实现步骤

1. 在 `GameModels.swift` 新增模型：
   - `EnemyThreatCountermeasureExecutionResultKind`，建议覆盖 `damage`、`survival`、`objective`、`recovery`、`route`。
   - `EnemyThreatCountermeasureExecutionResultComparison`，字段建议为 `kind`、`title`、`expected`、`actual`、`result`。
   - `EnemyThreatCountermeasureExecutionResultSummary`，字段建议包含建议类型、执行单位、目标、威胁来源、坐标、预计摘要、实际对照数组，并提供 `actualSummary`。
2. 在 `GameState.swift` 增加 `@Published private(set) var latestEnemyThreatCountermeasureExecutionResult: EnemyThreatCountermeasureExecutionResultSummary?`。
3. 在 `executeMapCommand` 执行 ATK/MOVE 前捕获当前仍聚焦且执行入口可用的 `focusedEnemyThreatCountermeasurePreview`。
   - 抢先打击只在 `.attack` 成功后记录。
   - 撤出危险区和据点防守只在 `.move` 成功后记录。
   - POS 接敌或无关移动/攻击不应误记。
4. 在 `reinforceSelectedUnit()` 或 `reinforce(unitID:)` 成功路径中，对当前聚焦的 `.reinforce` 反制建议记录整补回放。
5. 记录回放时必须使用真实成功后的状态或已有结果摘要：
   - 抢先打击用 `latestCombatResult`。
   - 撤退/防守用执行后的单位位置、HP、据点归属和原 `destination`。
   - 整补用 `latestReinforcementResult`。
6. 清理规则：
   - `loadScenario()`、重开、切战役清空回放。
   - 新的普通攻击、移动、整补若不是聚焦反制建议执行，应清空旧回放，避免过期结果误导。
   - 预览、普通聚焦、敌方意图查询、反制建议查询、点选建议本身不生成回放。
7. 在 `ContentView.swift` 的反制建议面板或结果区域展示回放：
   - 使用已有 `CombatResultMetric` 或紧凑 `Label` 展示“预计 / 实际 / 结果”。
   - 文案明确这是“最近执行回放”而非新的建议。
   - 支持换行和 VoiceOver，不只靠颜色表达结果。
8. 在 `GameStateTests.swift` 中补充测试：
   - 抢先打击聚焦后执行 ATK，生成回放，并与 `latestCombatResult` 一致。
   - 撤退聚焦后执行 MOVE，生成回放，实际位置等于目的地。
   - 据点防守聚焦后执行 MOVE，生成回放，实际位置/守点动作与建议一致。
   - 整补聚焦后调用整补，生成回放，并与 `latestReinforcementResult` 一致。
   - 普通移动/攻击/整补不带聚焦建议时不生成或会清理旧回放。
9. 在 `RulesSmokeTest.swift` 加同等轻量断言，覆盖四类回放和清理边界。
10. 更新文档：
    - `WW2Tactics/README.md`：已实现、地图交互/战斗辅助和 smoke 覆盖加入执行回放。
    - `md/flow/flow.md`：GameModels、GameState、3.13 反制建议和测试映射加入执行回放。
    - `md/flow/flowchart.md`：核心逻辑图、地图命令流和规则状态图加入执行回放节点。
    - `md/test/test.md`：当前 smoke 和 Stage Regression 基线加入反制建议执行回放。
    - `update_log.md`：追加 v1.18 记录，本地验证后写具体命令结果，Agent C 验收后补云端结果。

## 验收标准

1. 玩家点选反制建议后，通过现有 ATK/MOVE/整补入口成功执行，侧栏能看到最近一次反制建议执行回放。
2. 四类建议至少在测试中覆盖“预计 / 实际 / 结果”对照。
3. 回放只在真实成功执行后生成；预览、点选建议、无关行动不生成虚假回放。
4. 不改变排序、数值、AI、移动、攻击、整补或执行桥接语义。
5. 本地轻量检查通过。
6. Agent B 在 `main` 提交并 `git push origin main`。
7. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
8. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：移动类建议没有现成 `latestMoveResult`，应基于执行后单位位置、HP、目的地和据点归属生成轻量回放，不要新增复杂移动结果系统。
- 风险：执行后聚焦状态会被清理，必须在执行前捕获建议快照。
- 风险：回放容易被误解为自动执行建议，文案必须明确是“最近执行回放”。
- 禁止绕过 `GameState` 在 View 中实现核心规则。
- 禁止新增一键执行反制建议。
- 禁止改动战斗数值、AI、移动、整补、胜负、排序或执行桥接语义。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
