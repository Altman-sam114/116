# v1.53 战线态势复盘筛选

## 版本号

v1.53

## 版本分配依据

- 当前 `md/prompt/v1（地图操作体验）/` 最大功能提示词为 v1.52。
- v1.52 已让据点压力行显示当前/应对态势对照，本轮继续强化战线态势卡里的“复盘影响”入口。
- 本轮仍属于 v1 地图操作体验，不开启新大版本。

## 背景

战线态势卡已经能显示全局 AI 关键复盘入口，但当前只回退到 `AIPhaseReplayConclusion.keyEvents.first`。当玩家面对据点压力或刚执行过反制/占点/战斗响应时，更有价值的是优先看到与当前压力、威胁来源或响应坐标相关的 AI 事件。EasyTech 风格战棋需要玩家快速读懂“上一敌方回合影响了哪里”，因此本轮补一个只读筛选层。

## 目标

1. 为 `BattlefieldSituationReplayTarget` 增加结构化来源等级，区分压力关联、响应关联和全局关键事件。
2. `GameState.battlefieldSituationReplayTarget()` 优先选择当前据点压力的 `replayTarget`，其次匹配当前态势响应坐标，最后回退全局 key event。
3. `ContentView` 在战线态势卡“复盘影响”按钮中展示来源短码/标题，说明当前入口为什么相关。
4. 增补 XCTest 和 rules smoke，覆盖优先压力线索、响应坐标匹配、全局回退和只读边界。

## 非目标

- 不改变 AI、AI 时间线、复盘结论 keyEvents 排序、据点压力排序、反制建议排序或战斗规则。
- 不新增未来模拟、因果归因、自动防守、自动移动、自动攻击或新地图标记。
- 不在 SwiftUI 中扫描 timeline、匹配坐标或选择复盘事件。
- 不新增第三方库、资产管线、workflow、Xcode project 或长期分支策略。

## 架构依据

- `BattlefieldSituationReplayTarget` 已是战线态势卡和压力线索按钮共同使用的定位模型，适合承载来源解释。
- `GameState` 已集中拥有 `latestAIPhaseSummary`、当前压力 id、态势响应历史和地图有效性校验，应在这里完成筛选。
- `ContentView` 只能展示 `BattlefieldSituationReplayTarget` 字段并转发 `focusBattlefieldSituationReplayTarget()`。

## 实现步骤

1. 在 `GameModels.swift` 增加 `BattlefieldSituationReplayTargetSource`：
   - `.objectivePressure`、`.responseCoordinate`、`.globalKeyEvent`。
   - 提供 `title`、`shortTitle`。
2. 给 `BattlefieldSituationReplayTarget` 增加 `source` 字段，并纳入 `id`。
3. 在 `GameState` 中改造 `battlefieldSituationReplayTarget()`：
   - 若当前点选压力仍存在且有 `replayTarget`，返回其线索并标记 `.objectivePressure`。
   - 否则若当前查看态势响应有合法坐标，从最新 timeline 里优先选择 `to/from == response.coordinate` 的事件，标记 `.responseCoordinate`。
   - 否则回退原 `replayConclusion.keyEvents.first`，标记 `.globalKeyEvent`。
4. 保持 `focusBattlefieldSituationReplayTarget()` 只按 target order 调用既有 AI 时间线定位，不直接执行命令。
5. 更新 `BattlefieldSituationSummaryView` 的复盘按钮，展示 source 短码和来源标题。
6. 扩展测试和 smoke：
   - 无压力/响应匹配时保持全局回退。
   - 当前压力有线索时，summary replay target 优先使用压力线索。
   - 当前响应坐标匹配 timeline 且无压力线索时，summary replay target 使用响应关联。
   - 聚焦复盘影响仍不改变单位、指令点、AI summary、地图复盘 marker 或战报。
7. 更新 README、flow、flowchart、test、prompt README 和 update_log。

## 验收标准

- 战线态势“复盘影响”优先显示当前压力线索，其次响应坐标匹配，最后全局关键事件。
- 复盘 target source 在模型、UI 和测试中可见。
- SwiftUI 不计算 timeline 匹配或来源优先级。
- 点选复盘影响只切换 AI 复盘焦点，不改变核心战局状态。
- Agent B 在 `main` 以 `v1.53: 增强战线态势复盘筛选` 提交并 `git push origin main`。
- GitHub Actions 生成未加密 CI artifact，Agent C 下载并核对最新 `origin/main` commit、run id、run attempt 和 manifest 后才能判定通过。

## 风险和禁止项

- 禁止把筛选结果写成 AI 因果归因；只能表达“关联/匹配/关键事件”。
- 禁止为了筛选改变 `AIPhaseReplayConclusion.keyEvents`、AI timeline 或战局状态。
- 禁止在 SwiftUI 中重新匹配 timeline 或保存独立复盘 target 状态。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端验收。
