# v1.30 路径风险对比

## 版本号

v1.30

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.29（AI复盘结论关键事件定位）。`origin/main` 最新验收 commit `cf52154d5dcea33fd334cdd1ede8f0d90662fa6f` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.29-main-cf52154-run28771408489-attempt1` 核对，run id `28771408489`，attempt `1`。v1.29 已完成 AI 复盘结论关键事件定位；下一步回到玩家回合地图操作体验，增强 POS 接敌时对默认路线与安全候选之间的风险差异解释，因此本轮递增为 v1.30。

## 背景

当前 POS 接敌已经能显示默认攻击位、路线步序、终点火力暴露和最多 3 条安全接敌候选。玩家能看到候选本身的风险，但缺少“为什么它比当前默认路线更好或代价是什么”的结构化对比。EasyTech 风格战棋需要在玩家下令前快速说明战术取舍：少承伤、少敌火、是否绕路、是否进入控制区。本轮要把这些对比做成 `GameState` 纯派生数据，再由 SwiftUI 展示，避免 View 自己重算规则。

## 目标

1. 新增安全接敌路径对比模型：
   - 能比较当前 POS 默认路线与安全接敌候选路线。
   - 至少包含候选目的地、参考路线目的地、风险等级变化、潜在承伤变化、最高单源伤害变化、敌火来源数量变化、路线受威胁步数变化、移动力消耗变化、控制区惩罚变化和是否当前聚焦候选。
   - 提供稳定的短文案或指标字段，供 UI、XCTest 和 smoke test 共享。
2. 在 `GameState` 暴露纯派生对比：
   - 只在选中己方单位、聚焦射程外敌军且存在 POS 候选时返回。
   - 默认参考路线应是当前未切换安全候选前的首选 POS 路线；如果当前已聚焦某安全候选，也仍能说明该候选相对默认路线的差异。
   - 不改变 `safeEngagementOptions(for:against:)` 的排序语义，不改变默认 POS 执行路线，除非玩家点选候选。
3. 更新 UI 展示：
   - `SafeEngagementOptionsPanel` 每条候选显示相对默认路线的“少承伤 / 多承伤 / 同等风险 / 多走 N 移动力 / 少 N 步路线暴露 / 少进控制区”等紧凑信息。
   - 地图命令预览中的安全接敌建议应优先使用同一对比数据，避免字符串自行重算。
   - UI 只展示模型数据和转发点选意图，不写核心规则。
4. 补充测试：
   - XCTest 覆盖默认路线仍不变、最优安全候选有明确承伤降低，路线受威胁步数、风险等级或敌火来源差异可解释，移动力差异可解释。
   - XCTest 覆盖聚焦安全候选后，对比仍相对默认路线，且聚焦只切换预览不移动、不攻击、不生成结果摘要。
   - Smoke test 在现有安全接敌场景中覆盖对比字段、文案或指标，并验证只读不变性。
5. 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不改变移动、攻击、战术命令、补给、士气、经验、控制区、胜负、AI 或反制建议数值。
- 不改变 `safeEngagementOptionSort` 排序规则，除非发现当前测试不能稳定表达既有语义；如需调整必须在 prompt 外明确说明原因。
- 不新增真实反应射击、伏击、火力压制行动、动画、音效、真实美术资产或第三方库。
- 不让 SwiftUI 直接计算火力暴露、路线代价或安全候选排序。
- 不做大规模 UI 重排，不新增卡片嵌套卡片。

## 当前架构依据

- `RouteStepPreview` 已表示路线每步消耗、控制区惩罚和敌火来源。
- `PostMoveFireExposurePreview` 已表示终点潜在承伤、最高单源伤害、敌火来源、风险等级和击毁风险。
- `SafeEngagementOption` 已绑定候选路线、终点暴露、目标单位和目标名称。
- `GameState.focusedAttackPositionRoute` / `focusedAttackPositionRoutes` / `focusedSafeEngagementOptions` 已在 POS 聚焦时提供默认路线和候选集合。
- `focusSafeEngagementOption(_:)` / `focusSafeEngagement(targetID:destination:)` 已负责点选候选后重新校验并切换预览路线。
- `ContentView.SafeEngagementOptionsPanel` 是安全候选展示入口；地图命令预览中已有安全接敌建议文本，可改为消费新对比模型。

## 实现步骤

1. 修改 `GameModels.swift`：
   - 新增 `SafeEngagementComparisonPreview` 或同等命名的 `Equatable` 模型。
   - 字段保持纯值类型，避免闭包或 UI 类型。
   - 增加必要的派生属性，例如 `damageDeltaText`、`riskDeltaText`、`routeThreatDeltaText`、`movementDeltaText` 或 `summaryText`，确保测试能断言关键文案。
2. 修改 `GameState.swift`：
   - 新增 `focusedSafeEngagementComparisons` 纯派生属性。
   - 新增 helper 从默认 POS 路线和候选 option 生成对比。
   - 参考路线必须稳定：不因 `focusedSafeEngagementDestination` 已切换就把当前候选当成参考路线。
   - 不写入 `@Published` 状态，不移动单位，不改变候选排序。
3. 修改 `ContentView.swift`：
   - `SafeEngagementOptionsPanel` 遍历对比模型或把 option 与 comparison 配对展示。
   - `SafeEngagementOptionRow` 增加紧凑对比信息，文字必须能在侧栏窄宽度下换行，不挤出容器。
   - 命令预览中的安全接敌建议复用同一对比模型生成简短建议。
4. 修改 `GameStateTests.swift`：
   - 扩展现有安全接敌测试，断言对比数量与候选数量一致。
   - 断言第一候选相对默认目的地减少潜在承伤，风险等级不高于默认路线，移动消耗差异稳定。
   - 聚焦候选后断言对比仍以默认目的地为参考，且单位、结果摘要和默认执行语义边界不被破坏。
5. 修改 `RulesSmokeTest.swift`：
   - 在现有 `safeEngagementScenario()` smoke 段增加对比字段断言。
   - 断言安全接敌对比不会改变单位位置、行动状态、指令点和焦点执行预览。
6. 文档同步：
   - README 已实现、地图交互、战斗辅助和 smoke 描述加入“安全接敌候选显示相对默认路线的风险/承伤/移动代价对比”。
   - `md/flow/flow.md` 的 POS / THR 段落加入对比模型的纯派生边界。
   - `md/flow/flowchart.md` 在 MOVE/POS 风险链路中加入安全接敌对比。
   - `md/test/test.md` 更新当前 smoke / XCTest 基线。
   - `md/prompt/README.md` 当前建议递进到 v1.30 后续方向。
   - `update_log.md` 追加 v1.30 记录。

## 验收标准

1. POS 安全接敌候选能显示相对默认路线的风险、承伤、敌火来源、路线受威胁步数、移动力和控制区差异。
2. 对比数据由 `GameState` / `GameModels` 纯派生，SwiftUI 不自行计算规则。
3. 默认 POS 执行路线不因候选对比生成而改变；只有玩家点选候选后才切换预览路线。
4. 聚焦候选不移动、不攻击、不消耗行动、不生成攻击/战术/占领/后勤结果。
5. XCTest 和 smoke test 覆盖对比字段、排序边界和只读不变性。
6. 本地轻量检查通过。
7. Agent B 在 `main` 提交并 `git push origin main`。
8. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
9. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：如果参考路线随当前安全候选焦点变化，对比会退化成“自己和自己比”；必须保持默认 POS 路线为稳定参考。
- 风险：如果 View 自行计算风险差异，会和 `GameState` 排序依据失步；必须消费模型层结果。
- 风险：新增文案过长可能挤压侧栏；保持短句并允许换行。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负、播放控制和反制建议语义。
- 禁止新增不可测试的 UI-only 对比状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
