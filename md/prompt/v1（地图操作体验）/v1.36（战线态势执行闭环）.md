# v1.36（战线态势执行闭环）

## 版本号

v1.36

## 版本分配依据

当前 `v1（地图操作体验）` 已完成战线态势汇总、首要定位入口和下一步提示，最新版本为 v1.35。本轮继续在同一阶段递增为 v1.36，目标是把真实执行后的结果反馈回战线态势卡，形成“态势建议 -> 定位入口 -> 既有命令执行 -> 执行反馈”的小闭环。

## 背景

项目目标是持续逼近 EasyTech 风格二战回合制战棋体验。现有态势卡能告诉玩家当前优先处理什么，并指出 ATK、MOVE、整补或 OBJ 推进入口；但玩家通过既有命令链完成反制或占点后，态势卡本身缺少对“刚才响应是否有效”的紧凑反馈，需要玩家在多个结果卡之间理解上下文。

## 目标

- 新增一个只读派生模型，概括最近一次与战线态势相关的真实执行反馈。
- 优先复用 `latestEnemyThreatCountermeasureExecutionResult`，展示反制建议执行后的预计、实际和结果摘要。
- 其次复用 `latestObjectiveCaptureResult`，展示 OBJ/据点推进后的占点、奖励和进度摘要。
- 在战线态势卡中显示紧凑“执行反馈/态势响应”行，让玩家能直接看到建议闭环结果。
- 补充 XCTest 和规则 smoke test，证明反馈由真实结果派生、读取不改变状态、无关普通行动不会伪造态势响应。
- 同步更新 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不新增自动执行按钮。
- 不改变敌方意图、反制建议、OBJ 计划、AI、移动、攻击、整补、部署、据点、胜负或战斗数值。
- 不重新设计反制建议排序或 OBJ 排序。
- 不把核心规则写入 `ContentView`。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `GameModels.swift` 定义可测试的数据模型，应新增态势响应摘要类型。
- `GameState.swift` 是规则状态机，应通过 computed property 从既有 `latest*Result` 派生响应摘要，不新增 `@Published` 状态，避免复制结果状态。
- `ContentView.swift` 只展示 `GameState` / `GameModels` 已提供字段，不计算战斗、路线、反制、占点或奖励逻辑。
- `latestEnemyThreatCountermeasureExecutionResult` 和 `latestObjectiveCaptureResult` 已由真实成功路径写入，且普通行动、重开和切战役已有清理语义。

## 实现步骤

1. 在 `GameModels.swift` 新增 `BattlefieldSituationResponseKind` 和 `BattlefieldSituationResponseSummary`。
2. 在 `GameState.swift` 新增 `battlefieldSituationResponseSummary` computed property：
   - 若存在 `latestEnemyThreatCountermeasureExecutionResult`，生成 `.countermeasure` 摘要；
   - 否则若存在 `latestObjectiveCaptureResult`，生成 `.objectiveCapture` 摘要；
   - 否则返回 `nil`。
3. 摘要字段应包含标题、细节、结果短码、坐标、主结果文本和补充说明，全部来自既有结果对象。
4. 在 `ContentView.swift` 的 `BattlefieldSituationSummaryView` 中读取并展示响应摘要，样式保持紧凑，不增加嵌套卡片和业务判断。
5. 在 `GameStateTests.swift` 增加：
   - 反制建议真实执行后能派生 `.countermeasure` 响应；
   - OBJ 真实占点后能派生 `.objectiveCapture` 响应；
   - 读取响应摘要不会改变单位、据点、指令点、战报或最新结果；
   - 无当前相关结果时返回 `nil`，普通无关行动不伪造响应。
6. 在 `RulesSmokeTest.swift` 补充轻量断言，覆盖反制执行响应和据点占领响应。
7. 更新文档记录当前真实逻辑和测试基线。

## 验收标准

- 战线态势响应摘要只从既有真实结果派生，不新增 `@Published` 状态。
- 反制执行后，态势卡能显示执行反馈，且内容包含执行类型和结果对照。
- 据点占领后，态势卡能显示占点响应，且内容包含据点、奖励和进度。
- 无相关结果时不显示响应摘要；普通无关行动不伪造响应。
- XCTest、规则 smoke 和 SwiftUI typecheck 通过。
- Agent B 在 `main` 提交并推送，Agent C 下载最新 run 的 artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。
