# v1.37（战线态势敌方回合影响）

## 版本号

v1.37

## 版本分配依据

当前 `v1（地图操作体验）` 最新功能版本为 v1.36，已把战线态势卡和真实反制/占点执行结果连接起来。本轮继续沿同一方向递增为 v1.37，把敌方回合后的反制复核结果纳入战线态势卡，形成“执行反馈 -> 敌方回合影响”的下一段闭环。

## 背景

现有系统已能在反制建议真实执行后显示即时回放，并在结束回合后生成 `latestEnemyThreatCountermeasureFollowUpResult`，说明目标是否存活、威胁源是否仍在、据点是否守住和 AI 总览。但这些信息主要显示在反制复核卡中，战线态势卡只显示即时执行或占点反馈。玩家需要在侧栏多张卡之间理解“刚才的态势响应最终有没有影响敌方回合”。

## 目标

- 将 `latestEnemyThreatCountermeasureFollowUpResult` 纳入战线态势响应摘要。
- 当敌方回合复核存在时，`battlefieldSituationResponseSummary` 应优先展示敌方回合影响，而不是旧的即时执行回放。
- 响应摘要应显示复核等级、反制类型、执行单位/目标、关键对比和敌方回合坐标。
- 保持纯派生：不新增 `@Published` 状态，不模拟 AI，不改变复核生成、反制建议、AI、移动、攻击、整补、部署、据点、胜负或战斗数值。
- 补充 XCTest 和规则 smoke test，覆盖复核响应、优先级、只读无副作用和无基线敌方回合不伪造响应。
- 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不新增自动执行按钮。
- 不新增新的复核算法或 AI 行动归因。
- 不改变 `EnemyThreatCountermeasureFollowUpSummary` 的生成时机和清理语义。
- 不改变战线态势首要建议、OBJ 计划、反制建议排序或执行桥接。
- 不把核心判断写进 SwiftUI View。

## 当前架构依据

- `GameModels.swift` 已有 `BattlefieldSituationResponseSummary`，本轮可扩展 response kind 表达敌方回合影响。
- `GameState.swift` 已有 `battlefieldSituationResponseSummary` computed property，应继续作为唯一派生入口。
- `latestEnemyThreatCountermeasureFollowUpResult` 是敌方回合后真实状态对比，不模拟 AI，且已有 `outcomeLevel`、`outcomeTitle`、`detailSummary` 和 `comparisons`。
- `ContentView.swift` 的战线态势卡只渲染 response summary，不应计算复核等级、坐标或对比内容。

## 实现步骤

1. 在 `BattlefieldSituationResponseKind` 增加敌方回合影响类型，例如 `countermeasureFollowUp`。
2. 更新该类型的短码和 icon。
3. 在 `GameState.battlefieldSituationResponseSummary` 中优先从 `latestEnemyThreatCountermeasureFollowUpResult` 派生响应摘要，其次才是即时反制执行回放和占点结果。
4. 新增私有映射函数，把 follow-up summary 转成 `BattlefieldSituationResponseSummary`：
   - `title` 表达复核等级和反制类型；
   - `detail` 表达执行单位、目标和结论；
   - `resultTitle` 优先显示关键 comparison 的结果；
   - `resultDetail` 显示关键 comparison 的敌方回合前后变化；
   - `coordinate` 复用 follow-up 的执行坐标，缺失时使用威胁目标坐标。
5. 更新 `ContentView` 的 response color 映射，只消费 response kind，不计算复核规则。
6. 增加 XCTest：
   - 反制执行后、敌方回合前显示即时 `.countermeasure`；
   - 结束回合生成 follow-up 后显示 `.countermeasureFollowUp`，且旧即时回放已清理；
   - 读取 follow-up response 不改变单位、据点、指令点、AI summary、follow-up summary、消息或战报；
   - 无反制基线的敌方回合不伪造 `.countermeasureFollowUp` 响应；若 AI 真实占点触发既有占点响应，不应被本轮逻辑压掉。
7. 更新规则 smoke test 中的 follow-up 验证，断言敌方回合影响响应。
8. 同步文档和日志。

## 验收标准

- `battlefieldSituationResponseSummary` 在 follow-up 存在时优先显示敌方回合影响。
- 无 follow-up 但有即时反制执行时仍显示即时反制响应；无相关结果时返回 `nil`；无反制基线的 AI 回合不得伪造 follow-up 响应。
- 读取响应摘要不改变任何规则状态。
- UI 不计算 follow-up 规则，只展示 `GameState` 给出的字段。
- 本地轻量检查通过，提交并 push 到 `origin/main`。
- Agent C 下载最新 run artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。
