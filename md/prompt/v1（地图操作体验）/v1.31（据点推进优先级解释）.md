# v1.31 据点推进优先级解释

## 版本号

v1.31

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.30（路径风险对比）。`origin/main` 最新验收 commit `67b4d5fa92e40345a9176491f5830878ea59750c` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.30-main-67b4d5f-run28773954686-attempt1` 核对，run id `28773954686`，attempt `1`。v1.30 已增强 POS 安全接敌的路径风险对比；下一步继续强化大地图战棋的夺点决策，让 OBJ 目标计划说明为什么某个据点优先，因此本轮递增为 v1.31。

## 背景

当前 OBJ 目标计划已经能列出最多 3 条据点推进候选，并展示目的地、距离、移动消耗、控制区和终点火力风险。玩家仍需要从多个数字里自行理解“为什么首项是首选”：它是最近、可本回合占领/夺取、能推进多少距离、还有多少距离、是否有火力风险。EasyTech 风格战棋强调据点推进的即时取舍，本轮把这些排序输入转成结构化解释，供 UI 和测试共享。

## 目标

1. 增强 `ObjectiveAdvancePreview`：
   - 新增纯派生字段或 computed 属性，解释目标计划优先级。
   - 至少表达据点归属、当前距离、推进后剩余距离、是否本回合可占领/夺取、推进距离、路线消耗、控制区惩罚和终点火力风险。
   - 提供短文案，例如“本回合可夺取”“推进 4 格，剩 1 格”“终点 HIGH -12”“指令奖励见据点规则”等，供 UI 和测试断言。
2. 保持规则边界：
   - 不改变 `objectiveAdvancePlanSort`、`objectiveAdvancePlans(for:)`、`nearestObjectiveTarget(for:)` 或 OBJ 快捷按钮顺序。
   - 不新增 `@Published` 状态。
   - 不改变移动、占点、奖励、胜负、AI 或反制建议语义。
3. 更新 UI：
   - `ObjectiveAdvancePlanRow` 在现有密集行内展示优先级解释短句。
   - 无障碍 label 加入同一解释。
   - UI 只展示 `ObjectiveAdvancePreview` 字段，不自己重算排序或风险。
4. 补充测试：
   - XCTest 覆盖直接可夺取计划、远距推进计划、带火力风险计划和多目标排序计划的解释字段。
   - Smoke test 在现有 OBJ 段覆盖解释字段，确认首项仍匹配 OBJ 快捷目标，点选候选仍只聚焦不执行。
5. 同步 README、flow、flowchart、test、prompt README 和 update_log。

## 非目标

- 不改变目标计划排序，不引入新的据点评分公式。
- 不改变据点奖励数值、占点触发条件、移动路线、火力风险计算或胜负判定。
- 不新增生产队列、科技、外交、战役分支、动画、音效、真实美术资产或第三方库。
- 不让 SwiftUI 直接计算 OBJ 排序依据、路线风险或火力暴露。

## 当前架构依据

- `ObjectiveAdvancePreview` 已包含 `objectiveName`、`owner`、`route`、`reachesObjective`、`currentDistance`、`remainingDistance` 和 `fireExposure`。
- `objectiveAdvancePlanSort` 已按当前距离、本回合是否直达、推进后剩余距离、路线消耗、步数和名称稳定排序。
- `ObjectiveAdvancePlanPanel` 已消费 `objectiveAdvancePreviews(for:)` 并只通过 `focusObjectiveAdvancePreview(_:)` 转发点选。
- `focusObjectiveAdvancePreview(_:)` / `focusObjectiveAdvanceTarget(coordinate:)` 会重新查当前计划，避免 UI 使用过期路线。
- 已有 `objectiveAdvanceScenario`、`distantObjectiveAdvanceScenario`、`objectiveAdvanceFireRiskScenario` 和 `multipleObjectiveAdvancePreviewScenario` 可复用。

## 实现步骤

1. 修改 `GameModels.swift`：
   - 给 `ObjectiveAdvancePreview` 增加 computed 属性或轻量值字段，如 `distanceClosed`、`priorityReasonText`、`riskSummaryText`、`routeCostSummaryText`、`prioritySummaryText`。
   - 文案必须短，避免侧栏窄宽度溢出。
2. 修改 `ContentView.swift`：
   - `ObjectiveAdvancePlanRow` 在现有距离/消耗行与指标行之间或之后展示 `prioritySummaryText`。
   - `accessibilityLabel` 加入同一解释。
   - 不新增卡片嵌套，不重算 OBJ 排序。
3. 修改 `GameStateTests.swift`：
   - 直接可夺取计划断言“本回合可夺取/占领”、剩余 0、终点 SAFE。
   - 远距推进计划断言“推进 N 格，剩 N 格”。
   - 火力风险计划断言风险文案包含风险等级和潜在承伤。
   - 多目标计划断言首项仍与 OBJ 快捷目标一致，解释字段不改变排序。
4. 修改 `RulesSmokeTest.swift`：
   - 在现有 OBJ smoke 段加入相同解释字段断言。
   - 保持点选候选只聚焦、不移动、不生成虚假占领摘要。
5. 文档同步：
   - README 加入目标计划显示优先级解释。
   - `md/flow/flow.md` 的 OBJ 段加入解释字段纯派生边界，并修正该段编号。
   - `md/flow/flowchart.md` 在 OBJ 计划链路加入优先级解释。
   - `md/test/test.md` 更新 smoke/XCTest 基线。
   - `md/prompt/README.md` 当前建议移除本轮已做事项。
   - `update_log.md` 追加 v1.31 记录。

## 验收标准

1. OBJ 目标计划行能说明计划优先原因：直达/推进、距离变化、风险、路线代价和归属。
2. 解释数据由 `ObjectiveAdvancePreview` 纯派生，SwiftUI 不自行重算排序。
3. OBJ 首项仍与 `nearestObjectiveTarget(for:)` 和 `focusNearestObjectiveTarget()` 一致。
4. 点选目标计划仍只聚焦路线，不移动、不占点、不生成虚假结果摘要。
5. XCTest 和 smoke test 覆盖直接、远距、火力风险和多目标排序场景。
6. 本地轻量检查通过。
7. Agent B 在 `main` 提交并 `git push origin main`。
8. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
9. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：若解释字段暗示新排序或新评分，但代码实际仍按旧排序，会误导玩家；文案必须说明现有排序输入，而不是虚构分数。
- 风险：若 View 直接重算火力风险或路线信息，会与 `GameState` 失步；必须消费模型字段。
- 禁止改变 OBJ 排序、占点奖励、移动和胜负规则。
- 禁止新增不可测试的 UI-only 计划状态。
- 禁止新增动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
