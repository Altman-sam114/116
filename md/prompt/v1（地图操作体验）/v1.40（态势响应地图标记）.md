# v1.40（态势响应地图标记）

## 版本号

v1.40

## 版本分配依据

当前 `v1（地图操作体验）` 最新功能版本为 v1.39，已让战线态势卡从普通攻击、战术命令、部署和整补结果派生最近响应。下一步应继续把“行动结果 -> 态势解释”落到地图上，让玩家不用只看侧栏，也能直接在六边形地图中看到最近响应发生的位置。因此本轮递增为 v1.40。

## 背景

现有 `BattlefieldSituationResponseSummary` 已包含 `coordinate`，但 `ContentView.HexMapView` 只显示 CAP、INT、反制聚焦和 AI 复盘等地图标记，没有把战线态势响应投射到地图格。EasyTech 风格战棋强调“战报结果和地图位置同步”，所以最近一次战斗、战术、占点、部署、整补或反制影响应在地图上有轻量可读的 RSP/ATK/CMD/DEP/REP/CAP/CTR/AI 标记。

## 目标

- 新增战线态势响应地图标记，让最近 `battlefieldSituationResponseSummary.coordinate` 对应地图格显示响应短码和图标。
- 标记必须从 `GameState.battlefieldSituationResponseSummary` 纯派生，不新增真实执行状态，不改变 `latest*Result` 生成或清理语义。
- 地图标记应复用 `BattlefieldSituationResponseKind.shortTitle/iconName`，并能区分 follow-up、反制、占点、普通攻击、战术命令、部署和整补。
- `HexTileView` 边框、标记槽位和无障碍文案应包含态势响应；颜色只由响应类型映射，不在 View 中推导战斗/后勤规则。
- 若响应没有合法坐标，地图不显示标记；侧栏响应仍照常显示。
- 普通移动、预览聚焦、失败命令仍不生成响应标记。
- 补充 XCTest 和规则 smoke test，覆盖 marker 派生、只读无副作用、优先级和无坐标/无响应退化。

## 非目标

- 不新增历史响应列表、动画、声音、镜头移动、自动滚动或自动执行。
- 不改变战斗、战术命令、部署、整补、据点占领、反制执行、敌方回合复核、AI 或胜负规则。
- 不改变 v1.39 的 response 文案和优先级，除非为 marker 纯派生所需做最小模型补充。
- 不让 SwiftUI 反查单位、计算伤害、恢复、指令点、坐标或响应优先级。
- 不引入第三方库、图片资源或复杂资产管线。

## 当前架构依据

- `BattlefieldSituationResponseSummary` 已有 `kind/title/detail/resultTitle/resultDetail/coordinate`，足够作为 marker 来源。
- `BattlefieldSituationResponseKind` 已有 `shortTitle` 和 `iconName`，可复用为地图短码和系统图标。
- `HexMapView` 已按坐标分组反制 marker、AI marker，并逐格传给 `HexTileView`。
- `HexTileView` 已集中处理地图标记、边框优先级和 `accessibilityLabel`，适合增加一个响应 marker 参数。
- 反制聚焦 marker 和 AI 复盘 marker 已提供紧凑胶囊标记样式，可作为视觉密度参考。

## 实现步骤

1. 在 `GameModels.swift` 新增只读 marker 模型，例如 `BattlefieldSituationResponseMapMarker`：
   - `kind: BattlefieldSituationResponseKind`
   - `coordinate: HexCoordinate`
   - `shortTitle: String`
   - `iconName: String`
   - `title/detail/resultTitle/resultDetail` 或 `summary` 字段，用于无障碍和测试断言
   - `id` 基于 kind、coordinate 和响应内容稳定生成
2. 在 `GameState.swift` 新增 computed property，例如 `battlefieldSituationResponseMapMarker: BattlefieldSituationResponseMapMarker?`：
   - 读取 `battlefieldSituationResponseSummary`
   - 要求 `response.coordinate` 非空且 `tile(at:)` 仍存在
   - 纯派生 marker，不写入 `@Published`，不改变 `focusedCoordinate`、消息、战报、单位、据点、AI summary 或 latest result
3. 在 `ContentView.HexMapView` 中读取该 marker，按坐标传给对应 `HexTileView`。
4. 在 `HexTileView` 新增 `battlefieldSituationResponseMarker` 参数：
   - 渲染 `BattlefieldSituationResponseMapMarkerView`
   - 为 marker 预留不遮挡 INT、反制、OBJ、CAP、AI 复盘、行动按钮的槽位
   - 边框优先级低于当前选中/命令/反制聚焦/focused AI，高于普通 AI marker、普通据点边框
   - `accessibilityLabel` 加入“态势响应，<短码>，<标题>，<结果>”
5. 新增小型 SwiftUI marker view：
   - 使用 `Label` 或 `Image + Text`，保留可读文字，不依赖颜色作为唯一信息
   - 使用既有系统图标，不新增资源
   - `accessibilityHidden(true)`，由 tile 总 label 统一播报
6. 更新 `BattlefieldSituationSummaryView` 如有必要，只做微小文案补充；不得把 marker 逻辑写进侧栏 view。
7. 补充 `GameStateTests.swift`：
   - 普通攻击生成 `.combat` marker，coordinate 等于 response coordinate，并且读取 marker 无副作用
   - 战术命令、部署、整补至少覆盖 marker kind/coordinate/shortTitle
   - 反制执行 marker 仍优先于普通攻击 marker；follow-up marker 仍最高优先级
   - 普通移动、预览聚焦和失败部署不生成 marker
   - 如构造无效坐标或无 response，marker 为 nil
8. 补充 `RulesSmokeTest.swift`：
   - 在普通攻击、战术命令、部署、整补、反制执行、follow-up 现有断言旁追加 map marker 断言
   - 保留普通移动不生成 response/marker 的断言
9. 同步文档和日志。

## 验收标准

- 最近一次战线态势响应有合法坐标时，地图对应格显示态势响应标记。
- 标记 kind、shortTitle、iconName、coordinate 和 response summary 一致。
- 反制/follow-up/占点/普通行动响应优先级不变，marker 跟随当前最高优先级响应。
- 普通移动、预览聚焦、失败命令、无坐标响应不生成 marker。
- 读取 marker 不改变单位、据点、指令点、消息、战报、AI summary、latest result、focusedCoordinate 或胜负。
- SwiftUI View 不推导核心规则，只展示 `GameState` 给出的 marker。
- 本地轻量检查通过，提交并 push 到 `origin/main`。
- Agent C 下载最新 run artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。

## 风险和禁止项

- 禁止新增独立 `@Published` marker 状态，避免与 response summary 不一致。
- 禁止让普通移动或单纯聚焦生成 marker。
- 禁止在 View 中回查单位、重算 response、重排优先级或修改规则状态。
- 禁止遮挡核心命令标记导致 MOVE/ATK/POS 可读性下降。
- 禁止跳过 Agent C artifact 验收。
- 若 GitHub 权限、账号、远端或 artifact 下载失败，必须停止并说明阻塞，不能伪装云端验证完成。
