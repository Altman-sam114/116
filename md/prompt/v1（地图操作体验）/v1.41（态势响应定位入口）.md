# v1.41（态势响应定位入口）

## 版本号

v1.41

## 版本分配依据

当前 `v1（地图操作体验）` 最新功能版本为 v1.40，已将最近一次战线态势响应投射为地图标记。下一步应补齐“侧栏响应反馈 -> 对应地图格”的定位入口，让玩家能从战报式结果直接跳到六边形地图上的发生位置。因此本轮递增为 v1.41。

## 背景

`BattlefieldSituationResponseSummary` 和 `battlefieldSituationResponseMapMarker` 已能描述最近一次反制、占点、攻击、战术命令、部署、整补或敌方回合影响的坐标，地图也能显示 RSP/ATK/CMD/DEP/REP/CAP/CTR/AI 标记。但侧栏响应卡目前只是展示结果，玩家需要手动在地图上找标记。EasyTech 风格战棋强调战报、战场焦点和地图反馈互相联动，所以响应卡应提供一个只定位、不执行命令的入口。

## 目标

- 在 `GameState` 增加战线态势响应定位入口，例如 `focusBattlefieldSituationResponseTarget()`。
- 入口必须只使用当前 `battlefieldSituationResponseMapMarker` 的合法坐标作为定位来源。
- 有合法响应坐标时，更新 `focusedCoordinate`，清理临时 OBJ/SAFE/反制引导，并写入明确消息。
- 无响应、无合法 marker 或坐标已不在当前地图时，只写提示，不改变焦点和任何规则状态。
- 在 `BattlefieldSituationSummaryView` 的响应卡中增加一个紧凑定位按钮，点击后调用上述 `GameState` 方法。
- 响应定位必须保留 `battlefieldSituationResponseSummary`、`battlefieldSituationResponseMapMarker` 和最新结果摘要，不执行 MOVE/ATK/POS/整补/部署/AI。
- 补充 XCTest 和规则 smoke test，证明定位入口只改焦点/消息，不改单位、据点、指令点、战报、结果摘要或 AI 摘要。

## 非目标

- 不新增历史响应列表、上一条/下一条响应浏览、自动播放、动画、镜头缩放或声音。
- 不改变 v1.40 marker 派生模型、响应优先级或响应文案。
- 不改变战斗、战术命令、部署、整补、据点占领、反制执行、敌方回合复核、AI 或胜负规则。
- 不让 SwiftUI 反查单位、计算坐标、判断响应优先级或执行命令。
- 不引入第三方库、图片资源或复杂资产管线。

## 当前架构依据

- `GameState.battlefieldSituationResponseMapMarker` 已从 `battlefieldSituationResponseSummary` 纯派生，并已校验 `tile(at:)`。
- `GameState` 已有 `focusBattlefieldSituationPrimaryTarget()` 和 `focusBattlefieldSituationReplayTarget()`，可作为“侧栏按钮只定位、不执行”的入口范式。
- `BattlefieldSituationSummaryView` 已读取 `game.battlefieldSituationResponseSummary` 并渲染响应卡，适合加入按钮；按钮只转发到 `GameState`。
- 地图自动滚动已复用 `focusedCoordinate`，所以本轮不需要新增 UI 状态或滚动逻辑。

## 实现步骤

1. 在 `GameState.swift` 新增 `focusBattlefieldSituationResponseTarget()`：
   - 若 `battlefieldSituationResponseMapMarker == nil`，写入“战线态势暂无可定位的响应标记。”一类提示并返回。
   - 否则再次校验 marker 坐标仍在当前地图。
   - 清理 `guidedObjectiveCoordinate`、安全接敌焦点、反制聚焦等临时引导，避免旧路线/反制标记和响应定位混用。
   - 设置 `focusedCoordinate = marker.coordinate`。
   - 写入包含响应短码、标题和坐标的消息，例如“态势响应定位：ATK ... @ qX,rY。”
   - 不改变 `selectedUnitID`，不执行任何命令。
2. 在 `ContentView.swift` 的 `BattlefieldSituationSummaryView` 响应卡内：
   - 当 `game.battlefieldSituationResponseMapMarker` 非空时显示定位按钮。
   - 按钮使用系统图标，例如 `location.magnifyingglass`，文案保持短，避免挤压响应结果。
   - 按钮 `action` 直接调用 `game.focusBattlefieldSituationResponseTarget`。
   - 无 marker 时不显示按钮，避免给出无效入口。
   - 无障碍文案包含“定位态势响应”和响应短码/标题。
3. 补充 `GameStateTests.swift`：
   - 对普通攻击响应执行定位，断言 `focusedCoordinate == marker.coordinate`、消息包含短码/坐标。
   - 定位前后单位、据点、指令点、战报、`latestCombatResult`、`battlefieldSituationResponseSummary`、`battlefieldSituationResponseMapMarker`、AI summary 不变。
   - 普通移动/失败部署没有 marker 时调用定位入口，只写提示且不改变旧焦点。
   - 至少覆盖一个非 combat 响应，例如部署或整补，证明入口不依赖攻击结果。
4. 补充 `RulesSmokeTest.swift`：
   - 在已有普通攻击或反制响应 marker 断言后调用定位入口。
   - 断言焦点切到 marker 坐标，结果摘要和 marker 保留。
   - 在普通移动无响应场景调用定位入口，断言不生成 marker、不执行命令、不改变焦点。
5. 同步文档和日志。

## 验收标准

- 最近一次战线态势响应有合法 marker 时，侧栏响应卡显示定位入口。
- 点击定位入口后，地图焦点切到 marker 坐标，响应 marker 仍保留。
- 定位入口不改变单位、据点、指令点、战报、latest result、AI summary、胜负或响应优先级。
- 无响应、无合法 marker 或坐标失效时，入口不执行命令、不改变旧焦点，并给出明确提示。
- 普通移动、预览聚焦、失败命令仍不生成响应或定位入口。
- SwiftUI View 不推导核心规则，只展示 `GameState` 给出的响应和 marker。
- 本地轻量检查通过，提交并 push 到 `origin/main`。
- Agent C 下载最新 run artifact，核对 manifest、run attempt、JUnit、规则 smoke 日志、Xcode build 日志和 `.xcresult`。

## 风险和禁止项

- 禁止新增独立 `@Published` 响应定位状态，避免与 marker/summary 不一致。
- 禁止让按钮调用 `executeFocusedCommand()`、`move`、`attack`、`deploy`、`reinforce`、`endTurn` 或 AI 方法。
- 禁止在 View 中回查单位、重算 response、重排优先级或修改规则状态。
- 禁止把无 marker 的响应卡渲染成可点击入口。
- 禁止跳过 Agent C artifact 验收。
- 若 GitHub 权限、账号、远端或 artifact 下载失败，必须停止并说明阻塞，不能伪装云端验证完成。
