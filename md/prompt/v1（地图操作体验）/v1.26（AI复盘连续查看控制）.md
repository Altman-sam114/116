# v1.26 AI复盘连续查看控制

## 版本号

v1.26

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.25（AI复盘事件选中态和地图标记强调）。`origin/main` 最新验收 commit `b070614274ffed0db1b0da05bb7b445ee406e19e` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.25-main-b070614-run28746321293-attempt1` 核对。v1.25 已能点选单条 AI 时间线并显示当前事件选中态；下一步应让玩家能按上一条/下一条连续浏览敌方回合行动，降低大地图复盘成本。本轮递增为 v1.26。

## 背景

当前 AI 时间线支持点击任一可见事件定位地图，但玩家想顺序复盘敌方回合时，仍需要逐行点选。EasyTech 风格战棋的战报复盘更强调“连续看完敌方行动”，因此本轮加入只读的上一条/下一条复盘控制：它复用现有 `focusAIPhaseTimelineEvent(order:)` 成功路径，只改变当前复盘焦点、选中 order、地图强调和提示，不执行任何命令。

## 目标

1. 在 `GameState` 增加 AI 复盘相邻导航能力：
   - 新增 `canFocusPreviousAIPhaseTimelineEvent` 和 `canFocusNextAIPhaseTimelineEvent` 或等价只读属性，供 UI 禁用边界按钮。
   - 新增 `focusPreviousAIPhaseTimelineEvent()` 和 `focusNextAIPhaseTimelineEvent()` 或等价入口。
   - 若已有当前选中 order，则上一条/下一条按最新 `AIPhaseSummary.timeline` 顺序定位相邻事件。
   - 若没有当前选中 order，下一条从第一条事件开始，上一条从最后一条事件开始。
   - 在第一条点上一条、最后一条点下一条时保留旧焦点和旧 order，并发布明确边界消息。
   - 无 summary 或 timeline 为空时只发布提示，不改变旧状态。
2. 保持只读复盘边界：
   - 相邻导航只复用 AI 时间线事件定位，不调用移动、攻击、战术命令、部署、整补、AI 或回合推进。
   - 不改变单位 HP/位置、行动状态、据点归属、指令点、`latestAIPhaseSummary`、timeline 或 `latestAIPhaseMapMarkers`。
3. 强化 SwiftUI 控制：
   - 在 `AIPhaseSummaryView` 的“行动时间线”标题行加入上一条/下一条 icon 按钮。
   - 按钮使用有语义的 `Label`、`accessibilityLabel` / `accessibilityHint`，并在不可用边界禁用。
   - UI 只调用 `GameState` 的相邻导航入口，不自行计算目标 order 或修改状态。
   - 不新增自动播放、动画、计时器或历史回合列表。
4. 补充 XCTest 和规则 smoke test：
   - 覆盖无当前 order 时 next 聚焦第一条、previous 聚焦最后一条。
   - 覆盖有当前 order 时 previous/next 依次切换到相邻事件。
   - 覆盖边界请求保留当前 order / focusedCoordinate 并发布边界消息。
   - 覆盖无 summary 时相邻导航不改变单位、指令点、summary 和 marker。
   - 覆盖导航后 `focusedAIPhaseMapMarkers` 仍等于 `latestAIPhaseMapMarkers` 对当前 order 的过滤结果。
5. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变 AI 决策、行动顺序、时间线事件生成、地图复盘 marker 生成规则或 `latestAIPhaseMapMarkers` 的纯派生语义。
- 不新增逐帧播放、自动播放、暂停/继续、时间轴滑块、历史 AI 回合列表、动画、音效或真实美术资产。
- 不让 SwiftUI 自行计算下一个有效 order；有效性和边界处理必须在 `GameState`。
- 不改变战斗伤害、反击、士气、经验、补给、控制区、移动力、部署、整补、据点奖励、胜负或反制建议语义。
- 不新增第三方库、网络依赖或复杂资产管线。

## 当前架构依据

- `GameState.focusAIPhaseTimelineEvent(order:)` 是 AI 时间线点选定位的唯一入口，已负责查找事件、校验坐标、更新 `focusedCoordinate`、`focusedAIPhaseTimelineEventOrder` 和消息。
- `focusedAIPhaseMapMarkers` 已从 `latestAIPhaseMapMarkers` 按当前 order 过滤派生，适合随相邻导航自动更新。
- `AIPhaseSummaryView` 已持有 `@EnvironmentObject GameState`，时间线行已用 `Button` 转发点选意图。
- `ContentView` 只展示 `GameState` 状态，不应承载核心导航合法性。

## 实现步骤

1. 修改 `GameState.swift`：
   - 增加只读边界属性，基于 `latestAIPhaseSummary?.timeline` 和 `focusedAIPhaseTimelineEventOrder` 判断上一条/下一条是否可用。
   - 增加 `focusPreviousAIPhaseTimelineEvent()`、`focusNextAIPhaseTimelineEvent()`，内部共用私有 helper。
   - helper 只从最新 timeline 选择目标事件 order，并调用既有 `focusAIPhaseTimelineEvent(order:)`；边界和空 timeline 只写 message，不改变旧 order / focus。
2. 修改 `ContentView.swift`：
   - 在 AI 时间线标题行加入两个小型 icon 按钮，分别调用 previous / next。
   - 按钮禁用状态读取 `GameState` 的 can 属性。
   - 按钮可访问性说明“只切换 AI 复盘定位，不执行命令”。
3. 修改 `GameStateTests.swift`：
   - 扩展移动后火炮弹幕 AI 复盘测试，覆盖 next/previous 起始、相邻切换、边界保留和只读不变性。
   - 扩展无 summary 测试，覆盖 previous/next 不改变状态。
4. 修改 `RulesSmokeTest.swift`：
   - 在现有 post-move barrage 复盘 smoke 中加入 previous/next 断言。
5. 文档同步：
   - README AI、地图交互、战斗辅助和 smoke 描述加入“上一条/下一条连续复盘”。
   - `md/flow/flow.md` AI 回合摘要段落加入相邻导航和边界规则。
   - `md/flow/flowchart.md` AI 时间线链路加入 previous/next 控制节点。
   - `md/test/test.md` smoke / XCTest 基线加入 AI 复盘连续查看控制。
   - `update_log.md` 追加 v1.26 记录，并修正顶部当前状态。

## 验收标准

1. 无当前复盘 order 时，下一条聚焦第一条 AI 事件，上一条聚焦最后一条 AI 事件。
2. 有当前复盘 order 时，上一条/下一条会切到相邻事件并复用现有地图定位、选中态和 marker 强调。
3. 第一条点上一条、最后一条点下一条时保留当前 order 和地图焦点，并发布边界消息。
4. 无 summary 或空 timeline 时不会改变单位、指令点、AI summary、timeline、`latestAIPhaseMapMarkers`、当前 order 或地图焦点。
5. SwiftUI 按钮有可访问名称和 hint，禁用状态来自 `GameState`。
6. View 不计算下一个 order，不绕过 `GameState`。
7. 本地轻量检查通过。
8. Agent B 在 `main` 提交并 `git push origin main`。
9. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。

## 风险和禁止项

- 风险：边界按钮若由 UI 自行计算，容易与 `GameState` 真实 summary 失步；必须由 `GameState` 给出 can 状态和导航入口。
- 风险：若相邻导航直接修改 order 而不复用 `focusAIPhaseTimelineEvent(order:)`，可能绕过坐标有效性和消息逻辑；应复用既有成功路径。
- 风险：若按钮只靠颜色显示禁用或方向，不利于无障碍；必须使用有语义 Label 和 hint。
- 禁止改变 AI、战斗、移动、后勤、据点、胜负和反制建议语义。
- 禁止新增不可测试的 UI-only 复盘状态。
- 禁止新增自动播放、计时器、动画、音效、资产管线、第三方库或网络依赖。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
