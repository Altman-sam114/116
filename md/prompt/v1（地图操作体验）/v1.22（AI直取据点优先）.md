# v1.22 AI直取据点优先

## 版本号

v1.22

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.21（AI行动地图复盘标记）。`origin/main` 最新验收 commit `6ad18b834b85c879f5dfa1b29da5f1686d0dbc9b` 已通过 GitHub Actions artifact `ww2tactics-ci-v1.21-main-6ad18b8-run28739112169-attempt1` 核对。v1.21 已能把敌方 AI 行动复盘到地图；下一步应让敌方 AI 更贴近 EasyTech 式“大地图多据点攻防”的行为。本轮递增为 v1.22。

## 背景

当前 `runAxisAI()` 的行动顺序是后勤、可击毁攻击、战术命令、普通攻击、推进移动、移动后攻击。这个顺序能保证敌方会攻击和推进，但在没有可击毁目标时，AI 会先做普通消耗攻击，可能错过本回合可直接夺取的中立或盟军据点。EasyTech 风格战棋的敌方压力很大一部分来自据点归属变化和战线推进；当敌方本回合能直接进驻据点时，应优先夺点，再尝试移动后攻击。

## 目标

1. 新增 AI 直取据点优先规则：
   - 仅在该单位没有可击毁目标时触发。
   - 在普通非击杀攻击前，若该单位可移动到中立或非己方据点且该据点未被占据，先移动占点。
   - 移动后仍复用现有移动后攻击逻辑；若不能攻击，则按现有 AI 行动完成语义处理。
2. 保持现有高优先级行为：
   - 后勤仍先于作战行动。
   - 可击毁目标仍优先于直取据点。
   - 战术命令逻辑、指令点消耗、伤害、士气和状态规则不变。
3. 将直取据点行为纳入 `AIPhaseSummary.timeline` 和 v1.21 地图复盘标记：
   - 应产生 `move -> objectiveCapture` 时间线。
   - `latestAIPhaseMapMarkers` 应包含移动终点和占点坐标。
4. 补充 XCTest 和规则 smoke test，覆盖：
   - AI 在无击杀机会但可直取据点时优先占点，而不是原地普通攻击。
   - AI 在有可击毁目标时仍先击毁，再按既有机动追击逻辑推进。
   - 直取据点产生 AI summary、timeline、地图复盘标记和指令点奖励。
5. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变伤害公式、反击、兵种克制、地形、控制区、补给、士气、经验、军衔和胜负规则。
- 不改变战术命令选择、战术命令数值或后勤整补/部署优先级。
- 不新增概率 AI、搜索树、全局目标分配、生产队列、撤退 AI 或复杂战略规划。
- 不修改 SwiftUI 地图视觉、侧栏布局、图例、动画、音效或资源。
- 不新增第三方库、网络依赖或复杂资产管线。
- 不重构当前 `ObservableObject/@Published` 架构。

## 当前架构依据

- `GameState.runAxisAI()` 是轴心国 AI 行动顺序的唯一落点。
- `bestAttackTarget(for:requiringKill:)` 已区分可击毁攻击和普通攻击。
- `bestAdvanceDestination(for:)` / `advanceScore(for:movingTo:tile:)` 已能为后续推进选择据点、攻击位和接敌路线，但调用时机在普通攻击之后。
- `move(unitID:to:)` 已会更新据点控制，并通过 `applyObjectiveCaptureReward()` 记录占点结果和 AI timeline。
- v1.20/v1.21 的 AI summary、timeline 和 map markers 会自动记录真实移动与占点，因此本轮不需要新增展示层状态。
- 现有 `axisManeuverPursuitScenario()` 已覆盖“击毁优先后追击占点”；本轮需要新增一个“不可击毁但可直取据点”的小场景。

## 实现步骤

1. 在 `GameState.swift` 增加一个私有 helper，例如：
   - `bestImmediateObjectiveCaptureDestination(for unit: BattleUnit) -> HexCoordinate?`
   - 从 `reachableTiles(for:)` 中筛选未被占据、`tile.isObjective == true` 且 `tile.owner != unit.faction` 的坐标。
   - 排序建议：敌方据点优先于中立据点，其次可复用 `nearestAdvanceTargetDistance(from:for:)` 或坐标稳定排序。
2. 调整 `runAxisAI()` 行动顺序：
   - 保持后勤、可击毁攻击、战术命令在前。
   - 在普通 `bestAttackTarget(for:)` 之前插入直取据点移动。
   - 直取据点移动后复用现有“移动后若能攻击则攻击，否则结束行动并 Entrench”的逻辑，避免重复代码可抽取私有 helper。
3. 不改变 `bestAdvanceDestination(for:)` 的泛用推进逻辑；它仍负责普通推进。
4. 新增 XCTest 场景 helper，例如 `axisImmediateObjectiveCaptureScenario()`：
   - 小地图 3-4 格。
   - 一个盟军或中立据点在轴心单位本回合可达。
   - 同时放置一个轴心单位当前可攻击但不能击毁的盟军目标，验证 AI 选择先占点而不是普通消耗攻击。
   - 避免给轴心足够指令点触发部署/整补/战术命令干扰。
5. XCTest 断言：
   - `endTurn()` 后据点 owner 变为 `.axis`。
   - 轴心单位移动到据点坐标。
   - 原可攻击目标 HP 不下降，证明普通非击杀攻击被占点优先级压过。
   - `latestAIPhaseSummary.objectivesCaptured == 1`。
   - timeline 顺序包含 `move -> objectiveCapture`。
   - `latestAIPhaseMapMarkers` 包含 `.destination` 和 `.objective` 在据点坐标。
   - 保留或扩展现有机动追击测试，确认可击毁目标仍先于直取据点。
6. `RulesSmokeTest.swift` 增加同类小场景和断言，保证命令行 smoke 覆盖本轮规则。
7. 文档同步：
   - `WW2Tactics/README.md` AI 描述加入“无击杀机会时优先直取可达据点”。
   - `md/flow/flow.md` AI 执行流和 AI 回合摘要段落加入直取据点优先。
   - `md/flow/flowchart.md` 在 AI 状态图中标出可击毁、直取据点、普通攻击的顺序。
   - `md/test/test.md` 当前 smoke / XCTest 基线加入 AI 直取据点优先。
   - `update_log.md` 追加 v1.22 记录，本地验证后写具体命令结果，Agent C 验收后补云端结果。

## 验收标准

1. 轴心 AI 在没有可击毁目标且本回合能移动到非己方据点时，会先移动占点，而不是先做普通非击杀攻击。
2. 可击毁目标仍优先于直取据点，现有机动追击占点链不退化。
3. 直取据点通过现有 `move` / `updateObjectiveControl` / `applyObjectiveCaptureReward` 路径落地，不绕过 `GameState`。
4. 直取据点会生成 AI summary、`move -> objectiveCapture` timeline、占点奖励和 v1.21 地图复盘标记。
5. 不改变战术命令、后勤、战斗数值、移动范围、反制建议和 UI 语义。
6. 本地轻量检查通过。
7. Agent B 在 `main` 提交并 `git push origin main`。
8. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
9. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：插入直取据点优先级可能影响既有攻击顺序；必须限定在“无可击毁目标”之后，避免破坏击杀优先和机动追击测试。
- 风险：移动到据点后可能还能攻击，应复用现有移动后攻击逻辑，不新增第二套行动完成规则。
- 风险：后勤或战术命令可能干扰新测试场景；测试应控制指令点、单位类型和距离。
- 禁止在 View 中实现 AI 决策。
- 禁止绕过 `move`、`updateObjectiveControl` 或 `applyObjectiveCaptureReward` 直接改据点归属。
- 禁止改变伤害、移动、补给、士气、战术命令、部署、整补、胜负和反制建议语义。
- 禁止新增不可测试的 AI 状态或全局策略缓存。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
