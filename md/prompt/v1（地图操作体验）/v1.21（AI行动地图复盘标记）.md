# v1.21 AI行动地图复盘标记

## 版本号

v1.21

## 版本分配依据

当前 `md/prompt/v1（地图操作体验）/` 最新功能版本为 v1.20（AI回合行动时间线），且 `origin/main` 最新验收 commit `55f6a3a728b097e7cdb63cb7c23c08b0b6339d14` 已通过 GitHub Actions artifact 核对。v1.20 已能在侧栏列出敌方回合真实行动顺序，但地图本身还不能显示这些行动发生在哪里。本轮沿着敌方回合反馈链继续推进，因此递增为 v1.21。

## 背景

EasyTech 风格战棋里，玩家需要在地图上快速理解敌方刚才的推进、攻击、部署和占点位置。v1.20 的 `AIPhaseSummary.timeline` 已记录真实成功动作，但这些信息只在侧栏显示；玩家仍要把文本坐标和地图位置对应起来。本轮应把最近一次 AI 行动时间线派生为地图只读复盘标记，让玩家能直接在地图上看到敌方回合关键行动位置，同时保持 `GameState` 为唯一规则来源。

## 目标

1. 新增 AI 行动地图复盘标记模型，基于 `AIPhaseTimelineEvent` 派生，不新增独立可失步状态。
2. `GameState` 提供只读 `latestAIPhaseMapMarkers` 或等价 computed property：
   - 从 `latestAIPhaseSummary?.timeline` 生成。
   - 覆盖部署、整补、战术命令、攻击、移动和占点的关键坐标。
   - 每条标记包含事件顺序、动作类型、坐标、角色、短码、摘要和排序字段。
3. 地图上显示最近一次 AI 回合的复盘标记：
   - 移动应至少显示起点和终点，终点更醒目。
   - 攻击/战术命令应显示攻击者位置和目标位置，目标更醒目。
   - 部署/整补/占点显示对应坐标。
   - 标记最多显示最近若干条事件或全部事件的紧凑标记，不能遮挡现有单位、据点和命令标记。
4. 地图 tile 的 VoiceOver 文案加入 AI 复盘信息，不能只靠颜色或短码表达。
5. 图例加入 AI 复盘标记说明。
6. 补充 XCTest 和规则 smoke test，覆盖标记派生、事件顺序、角色坐标、重开/切战役清理和玩家预览/失败命令不生成标记。
7. 同步 README、flow、flowchart、test 和 update_log。

## 非目标

- 不改变 AI 决策顺序、目标选择、移动规则、攻击数值、整补成本、部署逻辑、战术命令、胜负条件或指令点收入。
- 不新增动画、逐帧播放、镜头自动移动、暂停/播放控制、音效或真实美术资产。
- 不把 AI 时间线做成通用事件总线。
- 不让 `ContentView` 从 battleLog、单位 HP 或地图状态反推 AI 行动。
- 不复用反制建议 ACT/SRC/CTR/TGT 标记、敌方意图 INT 标记或据点 CAP 标记表达 AI 复盘，以免语义混淆。
- 不新增第三方库、网络依赖或复杂资产管线。
- 不重构当前 `ObservableObject/@Published` 架构。

## 当前架构依据

- `GameModels.swift` 已有 `AIPhaseTimelineEventKind`、`AIPhaseTimelineEvent` 和 `AIPhaseSummary.timeline`，可在这里新增轻量 `AIPhaseMapMarkerRole` / `AIPhaseMapMarker` 类型。
- `GameState.swift` 已有 `latestAIPhaseSummary`，应从该 summary 派生地图标记，避免新增 `@Published latestAIPhaseMapMarkers` 造成失步。
- `ContentView.swift` 的 `HexMapView` 已把多种 marker 按坐标分组传给 `HexTileView`；本轮应仿照 `focusedEnemyThreatCountermeasureMapMarkers` 增加独立 AI 复盘 marker 参数。
- `HexTileView` 已有 CAP、INT、反制聚焦、路线、威胁、火力风险等 marker 位置和 `accessibilityLabel`，本轮应选择不冲突的位置并更新图例。
- `GameStateTests.swift` 和 `RulesSmokeTest.swift` 已有 v1.20 的 AI timeline 场景，可复用 `axisTacticalCommandScenario`、`axisFullAdvanceScenario`、`axisDeploymentResultScenario`、`axisReinforcementResultScenario` 和 `axisManeuverPursuitScenario`。

## 实现步骤

1. 在 `GameModels.swift` 新增：
   - `AIPhaseMapMarkerRole`，建议覆盖 `actor`、`target`、`origin`、`destination`、`objective`。
   - `AIPhaseMapMarker`，字段建议为 `eventOrder`、`eventKind`、`role`、`coordinate`、`shortCode`、`summary`。
   - `id` 应稳定，例如 `AI-\(eventOrder)-\(role.rawValue)-\(coordinate.id)`。
   - role 应有 `title`、`shortTitle`、`sortOrder`，用于 UI 和测试。
2. 在 `GameState.swift` 新增只读派生：
   - `var latestAIPhaseMapMarkers: [AIPhaseMapMarker]`。
   - 从 `latestAIPhaseSummary?.timeline` flatMap 生成标记。
   - 对 `.move`：`from` -> `.origin`，`to` -> `.destination`。
   - 对 `.attack` / `.tacticalCommand`：`from` 或 actor 当前坐标 -> `.actor`，`to` -> `.target`。若 actor 坐标缺失，可只输出目标标记，不做 View 推断。
   - 对 `.deployment` / `.reinforcement`：`to` -> `.destination` 或 `.actor`。
   - 对 `.objectiveCapture`：`to` -> `.objective`。
   - 标记顺序按 `eventOrder` 再按 role `sortOrder` 稳定排序。
3. 在 `ContentView.swift`：
   - `HexMapView` 中按坐标 group `game.latestAIPhaseMapMarkers`。
   - `HexTileView` 新增 `[AIPhaseMapMarker]` 参数。
   - 新增 `AIPhaseMapReplayMarker` 视图，显示 `AI` 或事件短码与角色短码；保持紧凑，不要做大卡片或覆盖单位主体。
   - 标记建议放在 bottom/center 或动态偏移位置，避免顶端已被 INT、CAP、OBJ、火力风险、反制建议占用。
   - `borderColor` / `borderWidth` 可用一条低优先级橙紫或青紫强调，但不要覆盖当前可执行 MOVE/ATK/POS 的边框优先级。
   - `accessibilityLabel` 加入 “AI复盘，第 N 步，动作，角色，摘要”。
   - `MapLegendView` 增加 “AI复盘” 图例项。
4. 测试：
   - XCTest 增加或扩展 v1.20 AI timeline 测试：
     - tactical command 标记包含 actor 和 target。
     - full advance 标记包含 move origin/destination 和 attack target，顺序稳定。
     - logistics 标记包含 deployment/reinforcement 坐标。
     - maneuver pursuit 标记包含 attack target、move destination、objective marker，且 objective 坐标为 q0,r0。
     - restart/selectScenario 后 `latestAIPhaseSummary == nil` 且 `latestAIPhaseMapMarkers.isEmpty`。
     - 玩家预览、失败部署不生成 AI summary 或 markers。
   - Smoke test 在现有 AI summary/timeline block 后追加 marker 断言，覆盖顺序、坐标、角色和不与 `totalActions` 误绑定。
5. 文档更新：
   - `WW2Tactics/README.md`：AI 和战斗辅助加入“AI 行动地图复盘标记”。
   - `md/flow/flow.md`：GameModels、GameState、AI 回合摘要和测试映射加入 map markers。
   - `md/flow/flowchart.md`：AI timeline 后加入 map replay marker 节点。
   - `md/test/test.md`：当前 smoke / XCTest 基线加入 AI 行动地图复盘标记。
   - `update_log.md`：追加 v1.21 记录，本地验证后写具体命令结果，Agent C 验收后补云端结果。

## 验收标准

1. 结束玩家回合后，最近一次 AI 行动时间线除侧栏列表外，还能在地图上以只读复盘标记显示关键坐标。
2. 地图标记来自 `GameState.latestAIPhaseSummary.timeline` 派生，不由 `ContentView` 推断战斗、移动、后勤或占点。
3. 标记至少覆盖整补、部署、战术命令、攻击、移动和占点事件；移动起点/终点、攻击 actor/target、占点 objective 坐标可测试。
4. 玩家预览、失败行动、普通聚焦和被动休整不生成 AI 地图复盘标记。
5. 重开、切战役和下一次 AI 回合会清理或覆盖旧标记。
6. 不改变 AI、战斗、移动、整补、部署、战术命令、胜负和反制建议语义。
7. 本地轻量检查通过。
8. Agent B 在 `main` 提交并 `git push origin main`。
9. GitHub Actions 生成未加密 artifact，Agent C 下载并核对 `ci-artifact-manifest.json`、`ci-failure-summary.md`、`junit.xml`、`rules-smoke.log`、`xcodebuild.log` 和 `WW2Tactics.xcresult`。
10. Agent C 确认 artifact manifest 的 `branch=main`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致。

## 风险和禁止项

- 风险：地图已有 CAP、INT、反制聚焦、MOVE/ATK/POS、THR、火力风险等标记；AI 复盘标记必须有独立语义和低优先级布局，避免遮挡当前可执行命令。
- 风险：`AIPhaseTimelineEvent` 有些事件可能缺少 `from` 或 `to`；派生 marker 时应容忍缺失字段，不让 UI 推断。
- 风险：AI 回合可能因胜负提前结束；已经记录的事件应仍可派生标记，重开或切战役再清理。
- 风险：事件过多会挤压地图；UI 应紧凑展示，必要时每格聚合同坐标事件为短码组合。
- 禁止绕过 `GameState` 在 View 中推断 AI 行动。
- 禁止复用反制建议 marker 或敌方意图 marker 来表达 AI 复盘。
- 禁止改变 AI 决策、数值、移动、整补、部署、战术命令、胜负或反制建议语义。
- 禁止伪造本地测试、云端 run 或 artifact 验收。
